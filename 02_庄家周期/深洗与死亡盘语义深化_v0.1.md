# 深洗与死亡盘语义深化 v0.1

> 分线：B庄家周期 / D风险管理 / C语义概念  
> 版本：v0.1  
> 状态：初版 / 已吸收部分现有资料 / 待二次检索  
> 目的：明确 meme 第一控盘区策略中“深洗”和“死亡盘”的区别，避免把死亡盘误判成洗盘。  

---

## 1. 本文要解决的问题

在 meme 市场中，第一波拉升后出现深度回撤非常常见。问题是：

```text
深度回撤到底是洗盘，还是已经变成死亡盘？
```

这直接决定策略是否还有研究价值。

如果误判：

```text
把深洗当死亡盘 -> 可能错过二次启动
把死亡盘当深洗 -> 可能接住长期归零/流动性枯竭风险
```

对本策略而言，最重要的不是“跌了多少”，而是：

```text
深跌之后是否还能重新回收第一控盘成本区，并重新出现需求。
```

---

## 2. 当前定义

### 2.1 深洗

[暂定定义]

```text
深洗是指 meme 第一波拉升后，价格大幅回撤并长期冷却，通过跌破关键低点、扫流动性、打掉浮筹、降低市场热度来完成筹码再分配；但后续仍能重新接近并回收第一控盘区成本锚，说明该结构尚未彻底死亡。
```

英文对应：

```text
washout
shakeout
spring
liquidity sweep
stop hunt
```

---

### 2.2 死亡盘

[暂定定义]

```text
死亡盘是指第一波拉升后深度回撤，价格长期无法重新回收第一控盘区 POC / AVWAP，反抽持续被卖，成交、关注、叙事和流动性持续衰减，成本锚从支撑变为压力，二次启动结构长期无法形成。
```

英文对应：

```text
dead market
dead coin
failed recovery
dead cat bounce
breakdown
no demand
```

---

## 3. 它不是什么

### 3.1 深洗不等于所有大跌

错误理解：

```text
跌得深 = 洗盘
回撤到0.618/0.786 = 洗盘
庄家还会拉 = 洗盘
```

正确理解：

```text
深洗必须有后续重新回收和需求确认。
```

---

### 3.2 死亡盘不等于短期没涨

错误理解：

```text
洗盘时间长 = 死亡盘
横盘很久 = 死亡盘
热度下降 = 死亡盘
```

正确理解：

```text
死亡盘的核心是成本锚长期无法回收，反抽持续被卖，需求没有恢复。
```

---

## 4. 阶段/结构拆解

### 4.1 深洗路径

```text
P1 第一控盘区形成
-> P2 第一波拉升
-> P3 高位派发或部分派发
-> P4 深度回撤/扫流动性
-> P5 长时间冷却/浮筹清理
-> P6 重新接近第一控盘成本区
-> P7 回收 POC / AVWAP
-> P8 第一次拉起
-> P9 回撤不破成本锚
-> P10 二次拉升确认
```

### 4.2 死亡盘路径

```text
P1 第一控盘区形成
-> P2 第一波拉升
-> P3 高位派发严重
-> P4 深度回撤
-> P5 反抽无力
-> P6 POC / AVWAP 变压力
-> P7 每次接近成本区都被卖
-> P8 成交和关注持续衰减
-> PX 死亡盘 / 尾仓派发 / 无二次启动
```

## 5. 正向表现：更像深洗

更像深洗的表现：

```text
1. 跌破关键低点后能快速收回
2. 深跌后不再持续创新低
3. 接近第一控盘区时卖压减弱
4. 价格能重新接近 POC / AVWAP
5. 站回 POC 后不是立刻跌回
6. AVWAP 从压力逐步变成支撑
7. 第一次拉起后回撤不破关键成本锚
8. 二次拉升时有结构推进
9. 成交量出现从衰竭到恢复的变化
10. 社区/叙事/流动性没有完全死亡
```

Wyckoff 映射：

```text
Spring / Shakeout -> Sign of Strength -> Last Point of Support
```

重要启发：

```text
Spring 不是单独的跌破，而是处在区间中的假跌破，并需要后续 SOS 和 LPS 确认。
```

---

## 6. 反向表现：更像死亡盘

更像死亡盘的表现：

```text
1. 跌破第一控盘区后长期无法收回
2. POC 长期变成上方压力
3. AVWAP 长期压在价格上方
4. 每次反抽接近成本区就被卖
5. 放量不涨，说明供应持续出现
6. 反抽缩量，说明需求不足
7. 二次拉升无法突破第一次反抽高点
8. 成交量、流动性、社群热度持续枯竭
9. 第一波高位出现明显派发痕迹
10. 后续上涨更多像 dead cat bounce
```

对应关键词：

```text
failed recovery
no demand
dead cat bounce
supply coming in
failed reclaim
```

## 7. 常见误判

### 7.1 把“跌够深”当成洗盘

错误：

```text
回撤到 0.786，所以是洗盘。
```

修正：

```text
Fib 只说明跌幅，不说明是否还有需求。
```

---

### 7.2 把“第一次反抽”当成二次启动

错误：

```text
价格从低点拉了一段，所以庄家回来了。
```

修正：

```text
第一次拉起只能算 SOS 候选，必须等待回撤不破和二次拉升确认。
```

---

### 7.3 把“触碰 POC / AVWAP”当成重新接受

错误：

```text
价格碰到成本锚，所以成本区回收。
```

修正：

```text
重新接受必须观察站回、停留、回撤不破、再次推进。
```

---

### 7.4 把“洗盘很久”当成庄家还在

错误：

```text
盘得越久，庄家洗得越干净。
```

修正：

```text
长时间冷却如果没有需求恢复，可能只是死亡盘。
```

## 10. 已吸收的资料观点

### 10.1 Wyckoff 文章观点

来源：

```text
meme庄家控盘语义补充资料/EN_Wyckoff吸筹派发自动化_适合深洗与二次确认语义.md
```

可吸收观点：

```text
Spring 不是孤立跌破，而是区间内带有上下文的假跌破。
Spring 后需要 Sign of Strength。
SOS 后的 Last Point of Support 才是更保守的低风险位置。
必须用状态机保证顺序，不能看到跌破就直接判断。
```

对本策略影响：

```text
你的保守上车位置更接近 LPS，而不是 Spring 低点或第一次 SOS。
```

---

### 10.2 Liquidity Sweep + BOS 文章观点

来源：

```text
meme庄家控盘语义补充资料/EN_BOS流动性扫荡策略_适合诱多与洗盘识别.md
```

可吸收观点：

```text
流动性扫荡通常表现为影线扫过前高/前低，但收盘重新回到结构内部。
扫流动性需要结合 Break of Structure 来判断方向。
单独扫低点不能说明马上看多，需要后续结构确认。
```

对本策略影响：

```text
深洗如果只是扫流动性，需要观察是否收回并出现后续结构推进。
```

---

### 10.3 风险管理语义文档观点

来源：

```text
知识库迭代机制_v0.1/风险管理语义概念与专属关键词_v0.1.md

可吸收观点：

本阶段风险管理不是仓位管理，而是结构风险识别。

```

## 11. 对当前策略的影响

本文对策略的影响：

```text
1. 深洗不能只看跌幅，必须看回收。
2. 死亡盘不能只看时间，必须看需求是否恢复。
3. POC / AVWAP 是否重新接受，是区分深洗和死亡盘的关键。
4. 第一次拉起不是稳妥位置，最多是 SOS 候选。
5. 回撤不破 + 二次拉升确认，才更接近保守风格所要的位置。
```

---

## 12. 本版结论

当前最重要定义：

```text
深洗 = 深跌后仍能重新回收成本区并恢复需求。
死亡盘 = 深跌后成本区长期变压力，反抽持续被卖，需求无法恢复。
```

对你的策略而言：

```text
只有从深洗进入“成本锚回收 + 二次确认”的结构，才有资格继续研究。
如果表现为死亡盘，则直接进入 no_trade / observe_only。
```
