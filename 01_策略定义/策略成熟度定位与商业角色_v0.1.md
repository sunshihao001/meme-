# 策略成熟度定位与商业角色 v0.1

> 版本：v0.1  
> 分线：策略定义线 / 商业诊断线 / 风险审计线  
> 目的：把最新的质疑审计结论回填到策略定义层，明确当前“第一控盘区成本中枢回收模型”是什么、不是什么、当前成熟度、商业角色与禁止推进边界。  
> 状态：策略定位修正版 / 覆盖“无条件深化策略”的旧工作倾向。  

---

## 1. 当前策略一句话定位

当前策略不是成熟自动交易系统，而是：

```text
围绕 meme 早期第一控盘区、深洗、成本锚重新接受与二次启动确认的结构性观察模型。
```

更商业化的说法是：

```text
meme 早期结构候选筛选器 + 反证样本标注框架 + 风险审查器。
```

---

## 2. 当前策略不是什么

当前不是：

```text
1. 已验证稳定盈利策略。
2. 无需修改的成熟交易系统。
3. 可以直接自动下单的 EA。
4. 只要出现二次启动就买入的信号规则。
5. 能保证胜率或盈亏比的完整商业产品。
6. 可以脱离样本外验证直接部署的策略。
```

如果后续出现上述表述，必须降级修正。

---

## 3. 当前策略是什么

当前是：

```text
1. 一个结构性策略假设。
2. 一个可被状态机标注的观察框架。
3. 一个用于识别二次启动候选的研究模型。
4. 一个可以连接 POC/AVWAP/Volume Profile/Wyckoff/SMC 的知识框架。
5. 一个适合做 MQL5 观察指标 MVP 的需求源。
6. 一个需要通过反证样本和消融实验验证的候选策略。
```

---

## 4. 当前成熟度评级

```text
研究成熟度：中高
规则成熟度：中
样本验证成熟度：低
反证审计成熟度：中
商业定位成熟度：中
自动交易成熟度：低
观察器产品化成熟度：中高
```

综合评级：

```text
B-：值得验证，但不能宣称成熟。
```

## 5. 当前商业角色排序

### 5.1 最适合优先推进

```text
1. MQL5 观察指标
2. 样本标注器
3. 反证审计器
4. 人工复盘辅助工具
5. 候选结构提醒器
```

### 5.2 中期可考虑

```text
1. 多标的结构扫描器
2. 策略研究仪表盘
3. 消融实验分析器
4. 半自动交易提醒系统
```

### 5.3 暂不建议推进

```text
1. 全自动下单 EA
2. 实盘资金策略
3. 对外宣称稳定盈利的产品
4. 高杠杆交易信号服务
```

---

## 6. 策略继续推进的条件

只有满足以下条件，才允许继续向“交易策略”方向推进：

```text
1. 反证样本库至少完成 80 个样本。
2. B/C/D 类样本有足够数量，不只看成功样本。
3. 复合策略在消融实验中优于至少 2 个简单基准。
4. net_expectancy_R_after_cost > 0。
5. 样本外或时间后验验证仍保持稳定。
6. 参数小幅扰动后仍不崩溃。
7. 非发明者标注一致率达到最低标准。
```

不满足时，只能继续作为：

```text
观察器 / 样本框架 / 审计工具
```

---

## 7. 自动交易 EA 禁止推进条件

只要存在以下任一情况，不允许推进全自动 EA：

```text
1. 没有真实样本库。
2. 没有失败样本库。
3. 复合策略尚未优于简单基准。
4. 扣除滑点、手续费、冲击成本后期望值不明。
5. 规则仍依赖人工主观解释。
6. 第一控盘区边界和 AVWAP 锚点无法稳定复现。
7. 样本外结果未知。
```

当前状态：

```text
全自动 EA 暂不允许推进。
```

## 8. 当前应优先建设的产品形态

### 8.1 观察指标 MVP

目标：

```text
帮助人看结构，而不是替人做交易。
```

功能：

```text
手动框选 FCZ
自动计算 POC/VAH/VAL
绘制 AVWAP 多锚点
显示 reclaim / rejected 状态
输出 current_state
导出 CSV
```

---

### 8.2 样本标注器

目标：

```text
把主观结构转成可审计样本。
```

功能：

```text
记录 A/B/C/D/E/F 类样本
记录 MFE/MAE
记录是否事后移动边界
记录是否换锚点
记录商业可执行性评分
```

---

### 8.3 风险审查器

目标：

```text
优先减少坏交易，而不是制造更多交易。
```

功能：

```text
death_market_score
bull_trap_score
failed_reclaim_score
distribution_score
liquidity_trap_score
narrative_decay_score
```

## 9. 策略是否需要修改的判断方式

以后不再用“感觉还可以深化”作为修改理由。

必须回答：

```text
1. 修改解决哪个失败样本？
2. 修改是否能事前识别？
3. 修改是否会误杀成功样本？
4. 修改是否改善 net_expectancy_R_after_cost？
5. 修改是否通过样本外验证？
```

如果不能回答，则只能作为观察字段，不能进核心规则。

---

## 10. 当前主线优先级

从现在开始，主线优先级是：

```text
P1. 填充真实反证样本：FCZ_B_0001 / FCZ_C_0001 / FCZ_D_0001
P2. 用状态机规则检查样本是否可一致标注
P3. 完成第一批 20 个样本
P4. 输出第一批20样本审计报告
P5. 再决定是否开发 MQL5 观察指标 MVP
```

不是：

```text
继续添加更多概念
继续扩大资料检索
继续把策略包装成成熟系统
```

---

## 11. 本版结论

当前策略最合理的定位是：

```text
一个值得验证的结构性观察模型，而不是成熟交易策略。
```

最合理的商业角色是：

```text
观察器 + 样本标注器 + 风险审查器。
```

只有在反证样本库和消融实验通过后，才允许重新讨论：

```text
是否升级为半自动提醒器或自动交易 EA。
```
