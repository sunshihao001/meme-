## 1. 本文要解决的问题

本轮不是继续泛泛收集交易概念，而是围绕四个基准判断当前“第一控盘区成本中枢回收模型”到底与哪些成熟思路接近：

```text
A. 简单 POC 站回策略
B. 简单 AVWAP / VWAP 站回策略
C. 简单 Liquidity Sweep / False Breakout 回收策略
D. 当前复合策略：第一控盘区 + 深洗 + POC/AVWAP 接受 + 回撤不破 + 二次启动
```

同时补充两类验证纪律：

```text
E. 回测 / 优化 / 过拟合纪律
F. 失败案例 / 反例
```

### 3.1 POC / Volume Profile / Value Area 基准

#### POC-01

```text
标题：PivoTrack VP Pivot Anchored Volume Profile
URL：https://www.mql5.com/en/market/product/171455
来源类型：MQL5 Market / MT5 Indicator
归属基准：POC / Volume Profile
相关程度：高
是否有源码：否，Market 产品
是否有回测：否
是否有失败讨论：有基本免责声明
是否值得吸收：是
```

可吸收点：

```text
1. 自动检测 Pivot High / Pivot Low，并在相邻枢轴之间建立成交量分布。
2. 输出 POC / VAH / VAL / Value Area 背景。
3. 支持 developing profile，从最近确认 pivot 到当前 K 线持续更新。
4. 支持价格穿越 developing POC / VAH / VAL 警报。
5. 默认 Value Area 为 68%，而本项目暂定为 70%，后续需可参数化。
6. 支持 tick volume 与 real volume。
```

对当前策略的影响：

```text
它说明“枢轴锚定 Volume Profile + POC/VAH/VAL + crossing alert”已经是 MQL5 生态中存在的模块。当前策略不能把 POC 本身当作独特 edge，真正差异应放在：第一控盘区选择、深洗后重新接受、回撤不破、二次推进确认。
```

### 3.4 复合策略 / Wyckoff / SMC / BOS 基准

#### COMP-01

```text
标题：Automating Classic Market Methods in MQL5 (Part 1): Wyckoff Accumulation and Distribution
URL：https://www.mql5.com/en/articles/22628
来源类型：MQL5 Article
归属基准：复合策略 / Wyckoff / Spring / SOS / LPS
相关程度：极高
是否有源码：文章 EA 代码
是否有回测：有限，需进一步深读
是否有失败讨论：有状态失效逻辑
是否值得吸收：必须吸收
```

可吸收点：

```text
1. 用有限状态机自动化 Wyckoff accumulation/distribution。
2. Accumulation 顺序：prior downtrend -> trading range -> spring -> SOS -> LPS -> long trade。
3. Spring 不是普通跌破，而是在已定义 range 中的 false breakdown + recovery + volume signature。
4. 使用 tick volume 作为 forex 中 real volume 的代理，并强调 relative volume。
5. 任何状态可以 reset；例如 spring 后跌破 spring low，则 accumulation 失效。
```

对当前策略的影响：

```text
这是本轮最接近当前模型的资料。当前策略的“第一控盘区 -> 深洗 -> 回收 -> 二次启动”可视为 meme/crypto 语境下的 Wyckoff accumulation/re-accumulation 变体。但当前策略多了 POC/AVWAP 成本锚接受，少了完整 Wyckoff 事件命名。后续应重点对比状态机。
```

## 4. 相似度初判

### 4.1 与简单 POC 站回策略的接近程度

```text
接近程度：中
```

原因：

```text
当前模型使用 POC / VAH / VAL，但不是“价格站回 POC 就成立”。
当前模型要求：第一控盘区存在 -> 深洗后回收 -> POC/AVWAP 重新接受 -> 回撤不破 -> 二次推进。
```

结论：

```text
POC 是当前策略的一个成本锚模块，不是完整策略本身。
```

---

### 4.2 与简单 AVWAP / VWAP 站回策略的接近程度

```text
接近程度：中高
```

原因：

```text
当前模型非常依赖 AVWAP 作为成本锚，但多锚点选择和“最受尊重锚点”比普通 session VWAP 更关键。
```

结论：

```text
AVWAP 是当前策略的核心确认模块，但不能单独构成策略 edge。
```

---

### 4.3 与 Liquidity Sweep / False Breakout 策略的接近程度

```text
接近程度：高
```

原因：

```text
当前模型中的“深洗 -> sweep -> reclaim”与 false breakout / liquidity grab 高度重叠。
```

差异：

```text
普通假突破策略通常只看支撑/阻力回收；当前模型还要求回收到第一控盘区成本锚，并完成二次启动确认。
```

结论：

```text
Liquidity Sweep 是当前策略的重要入场前置事件，但不是最终确认。
```

---

### 4.4 与 Wyckoff / SMC 复合策略的接近程度

```text
接近程度：很高
```

原因：

```text
Wyckoff accumulation sequence：range -> spring -> SOS -> LPS
当前模型：第一控盘区 -> 深洗 -> 回收成本锚 -> 第一次拉起 -> 回撤不破 -> 二次启动
```

二者在结构上非常接近。

差异：

```text
1. 当前模型更强调 meme 早期第一控盘区。
2. 当前模型更强调 POC/AVWAP 成本锚重新接受。
3. Wyckoff 更强调 range、spring、SOS、LPS 的经典事件顺序。
4. SMC 更强调 OB、FVG、BOS、CHoCH、诱导等现代 price action 模块。
```

结论：

```text
当前策略不是完全独创的新交易思想，更像 Wyckoff re-accumulation / liquidity sweep / value reclaim / AVWAP cost-anchor acceptance 的复合变体。
```

## 5. 对“策略是否只是别人策略的一部分”的初步回答

第一轮检索后，当前判断是：

```text
它不是别人某一个简单策略的一部分，
但它确实是多个成熟交易思想的组合模块。
```

更准确定位：

```text
当前策略 = meme 语境下的结构性二次启动候选过滤器
```

它包含：

```text
1. Wyckoff：吸筹 / 洗盘 / Spring / SOS / LPS
2. Volume Profile：POC / VAH / VAL / Value Area
3. VWAP/AVWAP：成本锚 / fair value / institutional benchmark
4. Liquidity Sweep：假跌破 / stop hunt / reclaim
5. SMC：BOS / OB / FVG / inducement 的部分确认思想
6. 回测纪律：walk-forward / 样本外 / 过拟合控制
```

因此不能宣传为“全新策略”，应定位为：

```text
把多个常见交易常识在 meme 早期控盘结构中重新组合，并用更严格状态机过滤伪信号的观察模型。
```

## 7. 当前策略的专业性初判

### 7.1 专业性加分项

```text
1. 不是单指标信号，而是结构序列。
2. 有第一控盘区、深洗、回收、回撤不破、二次推进等阶段。

```

### 7.3 初步结论

```text
当前策略具备“专业策略研究框架”的雏形，
但尚不能称为“已验证稳定优质策略”。
```

```text
它是一个值得继续验证的结构性候选模型。

并在样本外仍保持更好的期望值和回撤控制”。
```
