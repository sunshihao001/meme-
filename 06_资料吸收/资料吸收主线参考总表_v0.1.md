# 资料吸收主线参考总表 v0.1

> 位置：`06_资料吸收/资料吸收主线参考总表_v0.1.md`  
> 分线：资料吸收总表 / 主线治理 / 理论权重控制  
> 状态：v0.1，首版归类表  
> 用途：区分哪些资料吸收文件已经进入当前主线，哪些只是辅助参考，避免 AI 把所有吸收笔记当成同等权重。

---

## 1. 为什么需要这张表

当前知识库里已经有很多资料吸收文件，但如果没有一张总表，AI 很容易出现下面的问题：

```text
1. 把主线理论、辅助理论、参考资料混在一起。
2. 把“吸收过”误解为“已经成为核心规则”。
3. 把单篇资料的局部表述误升级为项目总纲。
4. 无法判断哪些文件应该优先被总纲 / 策略定义 / 样本标注消费。
```

因此，这张表的目标是：

```text
为每份资料吸收文件分配权重、角色和去向。
```

---

## 2. 分类规则

### 2.1 主线核心

满足以下任一条件，归为主线核心：

```text
1. 直接定义项目主概念。
2. 直接决定当前阶段的研究方向。
3. 直接进入总纲、策略定义、样本标注、反证审计或指标需求。
4. 影响后续是否能写成规则或字段。
```

### 2.2 主线辅助

满足以下任一条件，归为主线辅助：

```text
1. 不直接定义主概念，但显著影响边界。
2. 主要用于确认、过滤、分型或风险审查。
3. 作为主线理论的补充证据。
```

### 2.3 参考资料

满足以下任一条件，归为参考资料：

```text
1. 只有启发意义，没有进入当前主线定义。
2. 只用于对照，不直接进入项目规则。
3. 未来可能有用，但当前没有被消费。
```

## 4. 当前主线消费路径

### 4.1 进入总纲的内容

```text
GMGN 作为数据源
第一控盘区 = 价值区域
POC / VAH / VAL = 成本区结构
HVN / POC = 成交密集证据
AVWAP = 成本锚与确认机制
Sweep / Reclaim / Confirm = 深洗三段式
Spring / SOS / LPS = 状态机确认顺序
```

### 4.2 进入策略定义的内容

```text
哪些概念构成策略本体
哪些概念只是过滤
哪些概念只能作为辅助锚点
```

### 4.3 进入样本标注的内容

```text
可计算字段
可回填状态
可反证条件
可复盘标签
```

## 5. 当前建议的阅读顺序

如果 AI 要理解项目主线，建议按这个顺序消费：

```text
1. 需求澄清后研究总纲
2. 策略成熟度定位与商业角色
3. 资料吸收主线参考总表
4. 资料吸收_GMGN作为meme市场数据源
5. MarketProfile / VolumeProfile / VWAP / AVWAP
6. Liquidity Sweep / Reclaim
7. Wyckoff Spring / Test / SOS / LPS
```

## 6. 版本结论

```text
这套知识库不是“很多资料都很重要”，而是“少数主线资料决定总纲，其余资料为主线服务”。
```

当前结论：

```text
主线已经明确，但还需要继续把主线文件反向压缩到总纲、策略定义、样本标注和指标需求里。
```
