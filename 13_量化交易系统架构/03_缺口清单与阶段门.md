## 1. 当前到完整系统的核心缺口

### 1.1 数据源与权限

缺口：

```text
1. 项目专属 GMGN adapter 设计。
2. 查询 key 与交易 key 隔离策略。
3. secrets 管理方案。
4. rate limit、失败重试、缓存、熔断。
5. 数据源版本与响应 schema 追踪。
```

当前 gate：

```text
只允许查询侧理论设计，不接入交易 key / private key / swap。
```

### 1.2 原始数据与快照

缺口：

```text
1. market / token / holders / traders / portfolio / track 查询任务。
2. raw response 保存。
3. token / market / holder / wallet snapshot。
4. fetched_at、source_capability、request_hash、schema_version。
```

当前 gate：

```text
先做 raw refs + snapshot + provenance，不做交易。
```

### 1.3 样本、反证与策略形式化

缺口：

```text
1. 成功样本、失败样本、错过大涨样本、死亡盘误判样本、不可执行样本。
2. FCZ 事前边界规则。
3. 深洗 vs 死亡盘硬边界。
4. 重新接受和二次启动确认标准。
5. 防止根据成功结果后移边界。
```

当前 gate：

```text
没有可标注、可反证、可追溯样本，不进入策略有效性判断。
```

### 1.4 特征、信号与决策

缺口：

```text
1. feature_set_version。
2. market / security / holder / smart money / FCZ / risk features。
3. reject / watch / candidate / needs_manual_review / no_trade 定义。
4. confidence score 与 reason_codes。
```

当前 gate：

```text
允许 candidate / watch / reject / needs_manual_review。
不允许 can_trade / auto_buy / auto_sell。
```

### 1.5 回测、成本与执行

缺口：

```text
1. 历史数据可用性和重建规则。
2. POC-only / AVWAP-only / Sweep-only 基准。
3. 消融实验、样本外验证、walk-forward。
4. slippage / gas / liquidity impact / failed transaction model。
5. net_expectancy_R_after_cost。
```

当前 gate：

```text
复合策略必须优于至少 2 个简单基准，且扣除成本后仍有正期望，才可讨论交易候选阶段。
```

### 1.6 资金、执行、安全与监控

缺口：

```text
1. position sizing / max risk / max daily loss。
2. stop-loss / take-profit / emergency exit。
3. GMGN quote / swap adapter。
4. transaction signing / wallet custody。
5. kill switch / audit log / post-trade reconciliation。
6. monitor / alert / incident report。
7. least privilege / disaster recovery。
```

当前 gate：

```text
任何 private key、资金、实盘交易、自动执行都必须 Owner 决策并另开阶段。
```

## 2. 阶段门

### Gate 0：需求控制面与边界

状态：进行中。

完成标准：

```text
最终目标、当前边界、禁止事项和系统层次写入 repo 真源。
```

### Gate 1：GMGN 样本标注 MVP

完成标准：

```text
GMGN 数据 → FCZ 样本字段 → 可标注 / 可反证 / 可追溯。
```

### Gate 2：只读数据探针 / Collector Spec

完成标准：

```text
定义 raw / snapshot / provenance schema；只读采集可验证；不触碰交易能力。
```

### Gate 3：状态机 Mapper

完成标准：

```text
GMGN kline + risk snapshot → FCZ state fields；allowed_mode 固定 observe_only / research_only。
```

### Gate 4：特征与风险评分

完成标准：

```text
feature_set_version、risk features、reason_codes 可审计，且通过样本反证。
```

### Gate 5：回测与消融

完成标准：

```text
历史数据重建、简单基准、复合策略消融、样本外验证、bias 检查完成。
```

### Gate 6：Paper Trading / Quote-only Dry Run

完成标准：

```text
只验证报价、滑点、延迟、成本和失败处理，不提交 swap。
```

### Gate 7：Manual Approval Trading

完成标准：

```text
系统只生成交易建议和订单草案，Owner 人工确认后才可执行。
```

### Gate 8：Limited Auto Trading

完成标准：

```text
小额度、限频、强风控、kill switch、完整审计和人工回滚能力齐备。
```

### Gate 9：Production Auto Trading

完成标准：

```text
完整自动筛选、执行、止盈止损、监控、审计、灾备和复盘体系完成。
```

## 3. 永久禁止捷径

```text
1. 不得把观察状态写成买卖信号。
2. 不得把 MQL5 编译成功写成交易系统完成。
3. 不得在没有样本反证前声明策略有效。
4. 不得绕过成本模型讨论盈利。
5. 不得把 GMGN 单一字段当作绝对真相。
6. 不得把 private key / wallet / swap 接入当前阶段。
```

---

## 4. 本版结论

```text
当前项目可以承接完整量化交易系统的架构设计，但执行阶段必须被 gate 锁住。

下一步只应推进 Gate 1-Gate 3：样本标注、只读数据、状态机 mapper。
```
