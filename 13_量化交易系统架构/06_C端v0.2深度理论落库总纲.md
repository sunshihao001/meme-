## 2. 当前项目根身份

项目当前价值不是执行，而是：

```text
可审计的观察、验证、标注与反证框架，用来测试第一控盘区成本中枢回收理论，能否从 GMGN 只读市场数据中表达成可复现的状态机字段。
```

近期正确系统身份：

```text
GMGN-powered meme-market read-only data intake
+ FCZ / First Control Zone cost-center recovery structural mapper
+ Candidate pool and risk triage engine
+ Sample library with raw references, provenance, and anti-hindsight controls
+ Falsification and ablation framework
+ Later backtest, simulation, and paper-trading framework
```

当前不是：

```text
自动交易 bot
GMGN swap executor
private-key / wallet-custody system
MQL5 / MT5 trading system
稳定盈利承诺
直接买卖信号引擎
```

## 3. C端 v0.2 的核心理论命题

```text
1. GMGN observations are candidate evidence, not truth.
2. FCZ states are hypotheses, not confirmations.
3. Scores are triage tools, not order instructions.
4. Samples are the source of validity, not isolated chart impressions.
5. Future execution must wait for data quality, sample falsification, state-machine validation, backtest reconstruction, paper ledger evidence, and explicit Owner decisions.
```

中文归纳：

```text
GMGN 字段只能作为观测证据；FCZ 状态只能作为待反证假设；评分只能用于分诊；样本库才是验证真源；任何执行层都必须位于数据、样本、反证、回测、paper、权限、安全与 Owner 决策之后。
```

## 4. 当前 observe-only MVP 闭环

C端 v0.2 明确当前 MVP 仍然是只观察闭环：

```text
1. Discover candidate token.
2. Query GMGN read-only data.
3. Run basic due diligence.
4. Calculate FCZ state-machine fields.
5. Apply risk scoring and reason codes.
6. Output candidate / watch / reject / needs_manual_review.
7. Record sample and raw data.
8. Do not auto-trade.
```

允许输出：

```text
candidate
watch
reject
needs_manual_review
```

禁止输出：

```text
can_trade
auto_buy
auto_sell
execute_swap
place_order
take_profit_order
stop_loss_order
```

## 5. 16 层系统框架承接

C端 v0.2 使用 16 层系统框架。旧文档中出现的 15 层表述，应视为历史阶段的压缩版本，不再覆盖当前 v0.2 理论主轴。

当前应按以下 16 层理解 meme 项目：

```text
1. 数据源与权限层
2. 数据采集与原始数据层
3. 数据质量与多源校验层
4. 样本标注与反证层
5. 市场宇宙 / 候选池层
6. 策略状态机 / 信号层
7. 风控过滤层
8. 仓位管理层
9. 执行层
10. 退出机制层
11. 回测与仿真层
12. Paper Trading 层
13. 监控与告警层
14. Kill Switch 层
15. 交易日志 / 审计 / 复盘层
16. 安全权限与人工确认层
```

当前阶段只允许建设前置研究、只读、样本、反证、mapper、validator 和观察闭环。第 8-16 层中涉及资金、执行、权限、安全与人工确认的部分，只能做边界设计和 future gate，不得实现实盘执行。

## 6. FCZ 状态机承接范围

C端 v0.2 给出的状态机应先作为理论设计和字段 mapper 候选，不应直接升级为交易信号。

可承接的状态集合包括：

```text
unobserved
discovered
data_pending
data_ready
fcz_candidate
fcz_locked
deep_wash_candidate
recovery_candidate
second_launch_watch
risk_rejected
needs_manual_review
archival states
observe_only_output
```

关键边界：

```text
1. discovered 不能跳过 data_ready 直接变成 candidate。
2. 任何 state 都不能直接转成 auto_buy / execute_swap / place_order。
3. state confidence 不是 profit probability。
4. FCZ lock、deep wash、recovery、second launch watch 都必须保留 invalidation 规则。
```


```text
1. GMGN fields are observations, not truth.
2. GMGN fields are not direct buy signals.
3. Trending / trenches / signal / smart money / KOL 只能作为 discovery/context。
```

## 8. 样本、反证与 ablation 承接原则

样本库是策略有效性和反证验证真源。C端 v0.2 建议样本类型至少覆盖：

```text
success
failure
ambiguous
missed-big-run
death-spiral false-positive
impossible-to-execute
random/control
```

样本记录应保留：

```text
sample_id
chain
token_address
raw_refs
provenance_refs
fcz_lock_timestamp
state_sequence
risk refs
decision_output
reason_codes
evidence_grade
anti_hindsight_lock
outcome window
counterexample_type
review_status
```

反证与消融必须避免 hindsight：

```text
1. FCZ boundary 必须早于 outcome window。
2. 不使用未来 high/low 锁定 FCZ。
3. 不使用 post-outcome POC。
4. 不混淆 state label 与 outcome label。
5. 每个核心判断字段都要能被 counterexample 和 ablation 检验。
```

## 9. Reason-code 与风险评分承接原则

C端 v0.2 明确：

```text
score 是 triage，不是 buy/sell decision。
```

D端后续拆分时，应把 reason-code 用作审计、分诊和复盘字段，而不是仓位、下单或止盈止损指令。

建议保留四类维度：

```text
structural score / reason codes
data quality score / reason codes
risk score / reason codes
review status / reason codes
```

风险处理必须区分：

```text
hard reject
soft warning
unknown-risk handling
manual review required
```

## 10. Stage gates 承接原则

当前从研究到执行的阶段门顺序是：

```text
read-only data gate
mapper gate
sample library gate
backtest gate
paper gate
quote-only gate
manual execution gate
limited auto gate
```

任何后层不能跳过前层。尤其是：

```text
没有 raw/provenance，不进入样本验证。
没有样本验证，不进入策略有效性判断。
没有反证与消融，不进入交易候选策略。
没有成本模型，不进入实盘执行。
没有安全与 Owner approval，不接入 private key / swap。
```
