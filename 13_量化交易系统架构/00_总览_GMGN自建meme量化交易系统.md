## 1. 本目录为什么存在

Owner 已纠正项目根方向：

```text
MQL5/MT5 不是最终 meme 市场交易执行平台。
最终方向是：GMGN 数据源 + 自建 meme 市场交易/扫描/验证系统。
```

因此需要把已经通过审查的理论草案从 `specs/` 与 `10_工程化交接/` 中拆出，形成长期可读的正式知识库分线：

```text

```

本目录回答：

```text
1. 最终系统是什么。
2. 当前已有资料应归位到哪里。

5. 如何从研究库迁移到自建交易系统。
```

## 2. 最终系统定义

最终目标可表述为：

```text
GMGN-powered meme-market quantitative trading system
```

中文定义：

```text
以 GMGN Skill / GMGN OpenAPI 为主要市场数据源，以第一控盘区成本中枢回收模型作为结构策略核心，逐步形成可自动发现候选 meme token、生成结构状态与风险判断、进入回测/纸上交易/灰度交易，并在完整风控和 Owner 授权后再考虑自动执行的自建量化交易系统。
```

终局能力包括但不限于：

```text
1. 自动发现候选 meme token。
2. 自动完成 token 尽调与风险过滤。
3. 自动计算 FCZ / 成本中枢 / 深洗 / 重新接受 / 二次启动结构字段。
4. 自动判断 reject / watch / candidate / needs_manual_review。
5. 在完整验证、风控、安全、Owner 授权后，才进入交易决策与执行阶段。
6. 全程记录原始数据、特征、状态、决策、订单、执行、失败原因和人工干预。
7. 支持回测、消融、纸上交易、quote-only dry run、灰度实盘、监控、回滚和复盘。
```

## 3. 当前现实定位

当前项目仍处于：

```text
GMGN-driven FCZ Structural Observer
+ Sample Labeling Framework
+ Falsification Audit System
+ Risk Reviewer
```

当前不是：

```text
完整量化交易系统
成熟自动交易策略
自动下单系统
稳定盈利模型
```

当前允许推进：

```text
1. 架构文档。
2. GMGN 数据字段映射。
3. 样本标注与 raw refs。
4. 状态机 mapper 设计。
5. 反证、消融、回测规则设计。
6. 纸上交易/quote-only dry run 的前置规格。
```

当前禁止推进：

```text
1. API key/private key 接入。
2. wallet custody。
3. GMGN swap / multi-swap。
4. 自动下单。
5. 自动止盈止损执行。
6. 资金实盘风险暴露。
```

## 5. 核心原则

```text
1. GMGN 是主要数据源，不是唯一真源。
2. MQL5 观察器是参考原型，不是最终执行环境。
3. FCZ 结构策略必须先变成可标注、可反证、可回测字段。
4. 自动交易必须经过数据、样本、反证、回测、成本、执行、安全、监控、Owner approval 多层门禁。
5. 当前阶段只允许研究闭环和只读数据闭环，不允许资金执行闭环。
```

```text
13_量化交易系统架构 是最终 GMGN 自建 meme 量化交易系统的架构承接目录。
```
