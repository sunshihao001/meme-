# 从研究库到交易系统的迁移路线 v0.1

> 分线：13_量化交易系统架构 / 迁移路线  
> 状态：v0.1 / 已审查理论草案落库  
> 目标：定义从当前 Markdown 研究库到 GMGN 自建 meme 量化交易系统的安全迁移路径。  
> 边界：只定义迁移路线与交付物，不执行交易接入。

---

## 1. 迁移总原则

```text
1. 先研究库，后工程库。
2. 先只读数据，后状态字段。
3. 先样本反证，后回测消融。
4. 先 paper / quote-only，后人工确认交易。
5. 先小额灰度，后有限自动。
6. 任何资金执行都必须 Owner approval。
```

---


## 3. 迁移阶段

### Phase A：研究库稳定化

交付物：

```text
13_量化交易系统架构/
07_样本标注/
09_规则与回测/
SOURCE_OF_TRUTH.md
validate_all 通过
```

完成标准：

```text
系统目标、边界、资料归位、MVP 和阶段门已落库。
```

### Phase B：只读数据规格化

交付物：

```text
gmgn raw schema
gmgn snapshot schema
provenance schema
fixture examples
collector failure modes
```

完成标准：

```text
可以保存和复核 GMGN 只读数据；不触碰交易权限。
```

### Phase C：状态机 Mapper 规格化

交付物：

```text
gmgn_fcz_state_mapper spec
input/output contract
state field dictionary
reason_codes dictionary
mapper fixtures
```

完成标准：

```text
GMGN kline + risk snapshot 可产出 FCZ 状态字段；allowed_mode 固定 observe_only。
```

### Phase D：样本反证与特征工程

交付物：

```text
sample dataset
feature_set_version
risk feature dictionary
positive/negative/missing evidence schema
反证样本补全
```

完成标准：

```text
状态判断可以被样本反证，不再只靠理论自洽。
```

### Phase E：回测与消融

交付物：

```text
historical reconstruction rules
baseline strategies
ablation plan
walk-forward validation
bias checklist
```

完成标准：

```text
复合策略优于至少 2 个简单基准，且没有明显 look-ahead / survivorship bias。
```

### Phase F：Paper Trading / Quote-only Dry Run

交付物：

```text
paper trade ledger
quote-only adapter spec
slippage / latency / failure report
no-swap proof
```

完成标准：

```text
系统只验证报价和执行可行性，不提交 swap。
```

### Phase G：Manual Approval Trading

交付物：

```text
trade proposal schema
owner approval record
pre-trade risk checklist
post-trade audit record
```

完成标准：

```text
系统只生成草案，Owner 人工确认后才可执行。
```

### Phase H：Limited Auto Trading

交付物：

```text
limited wallet
risk limits
kill switch
audit log
incident response
rollback plan
```

完成标准：

```text
小额度、限频、强风控、可关停、可复盘。
```

## 4. Codex / 实现 Agent 交接规则

后续交给实现 Agent 时，必须使用 bounded work order：

```text
target：本阶段唯一产物
path：允许修改的文件/目录
checks：必须运行的验证
state：当前阶段和 gate
stop gate：何时停止并交还
forbidden：private key / swap / auto order / 删除旧文件 / 重构旧目录
```

当前可交接的安全实现切片仅限：

```text
1. 只读 fixture validator。
2. schema validator。
3. state mapper prototype with fixtures。
4. markdown/docs index update。
5. backtest reconstruction spec skeleton。
```

当前不可交接：

```text
1. swap adapter。
2. wallet signing。
3. auto order。
4. take-profit / stop-loss execution。
5. private key storage。
```

---

## 5. 本版结论

```text
从研究库到交易系统不是一次跳跃，而是 Gate A-H 的渐进迁移。

当前 repo 最适合继续完成 Phase A-C：架构落库、只读数据规格、状态机 mapper 规格。

任何交易执行都必须等待样本、回测、成本、安全、监控和 Owner approval 完成后另开阶段。
```
