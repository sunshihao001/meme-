# 系统层次地图 v0.1

> 分线：13_量化交易系统架构 / 系统层次  
> 状态：v0.1 / 已审查理论草案落库  
> 上游真源：`specs/mql5-fcz-reclaim-model/full_meme_quant_trading_system_gap_brief.md`  
> 边界：只描述系统层次与依赖关系，不实现交易代码。

---

## 1. 总体分层

GMGN 自建 meme 量化交易系统按 15 层拆解：

```text
L1  数据源与权限层
L2  数据采集与原始数据层
L3  数据质量与多源校验层
L4  样本标注与反证层
L5  结构策略形式化层
L6  特征工程层
L7  信号与决策策略层
L8  回测与消融验证层
L9  交易成本与执行可行性层
L10 仓位、止盈止损与资金管理层
L11 执行系统层
L12 纸上交易与灰度实盘层
L13 监控、审计与告警层
L14 安全、权限与合规边界层
L15 工程化与质量门禁层
```

---

## 2. 当前允许建设的层

当前阶段允许建设：

```text
L1 只读数据源理论设计
L2 raw refs / snapshot / provenance 设计
L3 数据质量字段与多源校验规则设计
L4 样本标注与反证体系
L5 FCZ 状态机字段映射
L6 feature schema 草案
L7 reject / watch / candidate / needs_manual_review 研究状态
L8 回测与消融规则设计
L15 validator / CI / 文档索引质量门
```

当前不允许建设：

```text
L9 的真实交易成本执行绑定
L10 的实盘仓位与资金规则执行
L11 的 swap / signing / order adapter
L12 的真实资金灰度实盘
L14 的 private key / wallet custody 接入
```

---

## 3. 层间依赖关系

```text
GMGN 只读数据
  → raw response / snapshot / provenance
  → data quality / multi-source validation
  → sample labeling / falsification
  → FCZ state machine formalization
  → features / risk features
  → candidate decision states
  → backtest / ablation
  → paper trading / quote-only dry run
  → manual approval trading
  → limited auto trading
  → production auto trading
```

任何后层不能跳过前层：

```text
没有 raw/provenance，不进入样本验证。
没有样本验证，不进入策略有效性判断。
没有反证与消融，不进入交易候选策略。
没有成本模型，不进入实盘执行。
没有安全与 Owner approval，不接入 private key / swap。
```

## 5. MQL5 参考层归位

MQL5/MT5 只归位为：

```text
1. 结构观察器原型。
2. POC / AVWAP / retracement / state label 的可视化参考。
3. CSV 字段试验场。
4. 自建脚本状态机字段的参考实现。
```

MQL5/MT5 不归位为：

```text
1. 最终 meme 市场数据源。
2. 最终交易执行环境。
3. 自动下单主线。
4. 策略收益证明。
```

## 6. 本版结论

```text
系统层次地图的核心作用是防止“看到最终自动交易目标，就跳过中间所有研究、验证和风控层”。

当前安全建设边界停在：只读数据、状态字段、样本反证、观察状态、回测设计和质量门。
```
