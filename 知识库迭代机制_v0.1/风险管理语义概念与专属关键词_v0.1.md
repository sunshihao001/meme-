# Meme 第一控盘区深洗回收模型：风险管理语义概念与专属关键词 v0.1

> 目的：先定义这套策略真正需要规避什么风险，再用这些风险概念去论坛做二次检索。  
> 注意：这里的风险管理不是仓位和止损，而是“结构风险识别”。  

---

## 1. 本策略里的风险管理定义

传统风险管理常说：

```text
止损
仓位
盈亏比
最大回撤
```

但当前阶段，本策略更需要的是：

```text
结构风险管理
```

也就是先判断：

```text
这是不是已经从深洗变成死亡盘？
这是不是诱多反抽？
这是不是尾仓派发？
这是不是成本锚回收失败？
这是不是流动性陷阱？
```

## 3. 结构风险分类

### 3.1 死亡盘风险

定义：

```text
第一波拉升后深度回撤，后续长时间无法重新回收 POC / AVWAP，任何反抽都被卖出，成交和关注持续枯竭。
```

关键词：

```text
dead coin
dead market
failed recovery
dead cat bounce
breakdown
no demand
```

需要观察：

```text
POC长期变压力
AVWAP长期在价格上方
反抽成交量低或放量不涨
价格持续低于VAL
社区/热度消失
```

---

### 3.2 诱多反抽风险

定义：

```text
价格看似重新回到成本区，但无法稳定接受 POC/AVWAP，随后二次拉升失败，形成吸引买盘后的下跌。
```

关键词：

```text
bull trap
failed rally
liquidity trap
fakeout
false breakout
inducement
exit liquidity
```

需要观察：

```text
站上POC后快速跌回
站上AVWAP后快速跌回
二次拉升无法过第一次高点
高位成交量放大但价格不推进
```

---

### 3.3 尾仓派发风险

定义：

```text
庄家第一波已大幅派发，后续反抽只是为了处理剩余筹码，而不是重新吸筹。
```

关键词：

```text
distribution
late distribution
exit liquidity
selling pressure
supply coming in
```

观察：

```text
第一波高点巨量滞涨
连续上影线
反抽时主动卖压强
回收成本区时成交放大但价格推进不足
```

---

### 3.4 深洗 / 洗盘风险

注意：深洗本身不是风险，误判才是风险。

深洗正向语义：

```text
washout
shakeout
spring
liquidity sweep
stop hunt
```

深洗失败语义：

```text
failed washout
failed spring
breakdown
no reclaim
```

关键分界：

```text
扫完是否快速收回？
是否重新接受POC/AVWAP？
回撤后是否出现需求？
```

---

### 3.5 成本锚失败风险

POC失败：

```text
failed POC reclaim
POC rejection
POC as resistance
```

AVWAP失败：

```text
failed VWAP reclaim
VWAP rejection
VWAP as resistance
```

本策略最怕：

```text
价格只是碰到成本锚，而不是重新接受成本锚。
```

---

### 3.6 meme专属风险

meme 市场还要额外考虑：

```text
流动性薄
合约/池子深度不足
庄家撤池
叙事热度衰减
社群死亡
KOL出货
链上大钱包砸盘
CEX未上线或上线后见顶
```

关键词：

```text
liquidity drain
rug risk
whale dump
narrative decay
attention decay
community death
exit liquidity
```

## 5. 风险管理在状态机中的位置

风险不是最后才看，而是每个阶段都要看。

```text
P1 第一控盘区：防止假控盘区
P2 第一波拉升：防止拉盘出货
P4 深洗：防止死亡盘
P6 回收成本锚：防止假回收
P8 回撤不破：防止支撑失效
P9 二次拉升：防止诱多失败
```
