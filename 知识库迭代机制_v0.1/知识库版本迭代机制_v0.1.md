# Meme 第一控盘区深洗回收模型：知识库版本迭代机制 v0.1

> 目的：让这套策略知识库可以像产品/代码一样逐版迭代，而不是一次性写完。  
> 核心思想：每一版都要有“当前定义、待深挖语义、专属关键词、论坛检索、资料吸收、版本升级记录”。  

---

## 1. 知识库总目标

建立一套围绕 meme 市场的策略知识库：

```text
庄家第一控盘区
第一波拉升
深度回撤 / 洗盘
成本锚回收
二次启动确认
诱多 / 死亡盘 / 尾仓派发过滤
```

最终目标不是直接写一个买卖信号，而是逐步形成：

```text
概念清楚
阶段清楚
风险清楚
关键词清楚
资料来源清楚
样本标注清楚
最后才规则清楚
```

---

## 2. 版本迭代原则

### 2.1 先定义，再检索，再吸收，再更新

每一版遵循：

```text
1. 建立初步定义
2. 拆分语义概念
3. 提炼中文/英文/俄文关键词
4. 用关键词检索 MQL5 论坛/文章/代码库
5. 下载资料到本地
6. 阅读吸收资料中的概念
7. 反向更新策略定义
8. 记录本版新增和未解决问题
```

---

### 2.2 不急着量化

当前阶段禁止过早陷入：

```text
精确买点
止损止盈
仓位比例
参数优化
自动交易EA
胜率统计
```

原因：

```text
策略语义尚未稳定，过早量化会把模糊概念包装成伪精确。
```

---

### 2.3 每一版必须保留历史

版本不覆盖旧结论，而是：

```text
v0.1 初步定义
v0.2 补充语义
v0.3 引入论坛资料
v0.4 更新策略分类
v0.5 建立样本标注
...
```

每次更新必须记录：

```text
本版新增
本版修改
本版删除/降级
本版未解决问题
下一版目标
```

## 4. 版本号规则

### 4.1 主版本

```text
V0.x：策略研究和知识库阶段
V1.x：指标原型和人工标注阶段
V2.x：回测和规则验证阶段

V4.x：多机器人协同审查阶段
V5.x：交易执行系统阶段
```

### 4.2 当前版本定位

```text
当前：V0.1 - 语义与阶段框架初版
```

V0.1 只解决：

```text
策略是什么
属于什么类型
阶段怎么拆
需要哪些关键词
应该搜索哪些资料
风险管理语义有哪些
```

## 6. 版本更新流程

### Step 1：定义本版主题

例如：

```text
V0.1：meme 庄家周期与风险语义初版
V0.2：深洗 vs 死亡盘语义深化
V0.3：POC/AVWAP成本锚回收语义深化
V0.4：二次启动 vs 诱多反抽语义深化
```

### Step 2：列出本版问题

例如 V0.1：

```text
什么是庄家第一控盘区？
什么是深洗？
什么是死亡盘？
什么是二次启动？
什么是诱多反抽？
风险管理在此策略里到底管什么？
```

### Step 3：提炼关键词

每个问题转成：

```text
中文关键词
英文关键词
俄文关键词
MQL5常用关键词
SMC/Wyckoff关键词
```

### Step 4：检索资料

检索目标：

```text
MQL5 Articles
MQL5 CodeBase
MQL5 Forum
英文资料
俄文资料
必要时外部资料
```

### Step 5：吸收资料

每篇资料需要记录：

```text
标题
链接
语言
本地路径
相关概念
可吸收内容
对当前策略的影响
是否需要更新定义
```

### Step 6：更新知识库

更新类型：

```text
新增概念
修改定义
降级概念
删除不适合概念
新增风险项
新增关键词
新增待验证问题
```

## 8. 当前版本的核心原则

```text
先让策略概念站稳，再让规则细化。
先让阶段清楚，再让指标参与。
先让风险分类清楚，再谈上车位置。
先收集成功和失败样本，再谈参数。
```
