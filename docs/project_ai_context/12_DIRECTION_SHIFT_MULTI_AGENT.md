# Meme 项目方向转变决策记录

## 1. 触发

用户在研究两个多代理参考项目（CryptoAgents / Aegis Flux）后，识别出当前 meme 项目方向的核心问题，决定调整方向。

> 用户原话："感觉现在的meme方向错误了，不应该一开始就做成这个交易机器人方向的…可以设计使用多个相关Agent，根据不同的策略类型，实时变化，然后这样也不会局限于现在meme这种只做一个结构策略，买入卖出也都固定化"

## 2. 核心认知转变

| 维度 | 旧方向 | 新方向 |
|---|---|---|
| **本质** | 一个FCZ策略的Paper化 | 多Agent多策略的AI市场判断推理系统 |
| **策略** | 单一FCZ结构策略，固定链路 | 多个策略Agent，不同策略类型可组合 |
| **买卖逻辑** | 固定化（FCZ→拉升→深洗→回收→二次启动→决策） | 不预设固定路径，根据多维度证据动态形成判断 |
| **判断方式** | 单一策略视角走一条链 | 多Agent各看各的→辩论对抗→综合判断 |
| **适应性** | 市场不按FCZ路径走就失效 | 实时变化，策略可动态选择/切换 |
| **扩展性** | 加新策略要推翻框架 | 直接加一个Agent节点 |

## 3. 旧方向的三个核心问题

1. **单一策略锁定** — FCZ只是 meme 市场众多结构模式中的一种。把整个项目押在一条固定链路上，市场不按这条路径走就全部失效
2. **固定化买卖逻辑** — FCZ→拉升→深洗→回收→二次启动是预设路径。但 meme 市场结构模式远不止这一种（V型反弹/阶梯式拉升/横盘突破/砸盘吸筹…）
3. **阈值空转** — 花6轮循环精雕细琢14个阈值，但没数据验证，全是纸上推演。在打磨一个可能方向就错的单一策略

## 4. 新方向框架

```
Data Scout (链上数据采集，多源)
    ↓
多策略Agent并行（每个Agent一种策略视角/市场维度）:
  - FCZ结构Agent（已有FCZ理论，降为策略之一）
  - 资金流Agent（Smart Money/大户动向）
  - 情绪/热度Agent（X/Telegram/DexScreener）
  - 安全/风险Agent（拒绝门禁→风险视角Agent）
  - [可扩展更多策略Agent]
    ↓
多空辩论共识（不同Agent视角对抗，证据驱动，非投票）
    ↓
综合判断推理（不预设固定买卖，根据证据动态形成）
    ↓
Paper-Only判断输出 + 可审计推理链
```

**关键设计原则：**
- 不预设固定买卖路径——多Agent辩论后综合判断
- 策略可扩展——加新策略Agent不用推翻框架
- 实时变化——策略根据市场状态动态选择/组合
- Paper-Only——只产出判断推理，不碰交易执行

## 5. 已有工作的重新定位（不推翻，重新定位）

| 已有工作 | 旧定位 | 新定位 |
|---|---|---|
| FCZ策略定义(01) | 唯一主策略 | 链上结构Agent的策略知识 |
| 拒绝门禁(05) | 全局硬规则 | 风险Agent的判断逻辑 |
| 证据包要求(03) | 全局前置条件 | Data Scout的输出规范 |
| Paper-Only边界(06) | 全局约束 | HITL护栏 |

| FCZ验证方法论(01-1) | 全局验证 | FCZ Agent的独立验证计划 |
| 学习闭环(08) | 全局学习 | 系统级反馈循环 |

## 6. 对后续设计的影响

1. **meme-loop-system 分支设计基线变更** — 不再是"把meme-repo规则层装进Aegis Flux引擎"，而是"构建多Agent多策略动态判断推理系统，meme-repo规则层作为各Agent的领域知识输入"
2. **FCZ不再独占主线** — FCZ理论保留，但降为"链上结构Agent"的策略之一。后续可加入其他结构模式Agent
3. **阈值校准降级** — 不再是项目级阻塞项。各Agent的策略参数可独立验证，FCZ阈值校准只是FCZ Agent的内部任务
4. **GMGN阻塞解除** — Data Scout可用公开端点/mock先行，多Agent编排骨架不依赖单一数据源
5. **输出不再固定5种** — 综合判断可以输出 nuanced 推理（如"结构favorable但资金流恶化，建议WATCH并跟踪资金流变化"），而非固定的5种决策类型
