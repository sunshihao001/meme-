# Claude 专业风格知识库设计规范 v0.1

> 位置：`00_知识库设计规范/01_设计规范/Claude专业文档设计规范_v0.1.md`  
> 目标：让每次策略完善都形成结构清晰、可持续维护、可检索、可迭代的专业文章。  
> 说明：这里的“Claude专业设计规范”指高质量研究型文档风格：清晰标题、上下文、目标、定义、边界、流程、证据、结论、下一步。

---

## 1. 文档设计核心原则

### 1.1 单文档单主题

每篇文章只解决一个核心问题。

错误：

```text
一篇文章同时写庄家周期、POC、风险、回测、买点。
```

正确：

```text
一篇文章只写“深洗 vs 死亡盘”。
另一篇文章只写“POC重新接受”。
另一篇文章只写“二次启动 vs 诱多”。
```

---

### 1.2 先定义，再展开

每篇文章开头必须先回答：

```text
这篇文章解决什么问题？
它属于哪条分线？
当前版本是什么？
本版不解决什么？
```

---

### 1.3 每个概念必须有边界

每个概念必须写：

```text
是什么
不是什么
正例
反例
风险误判
相关关键词
后续如何验证
```

---

### 1.4 不把推测写成结论

文档中要区分：

```text
已确认定义
暂定定义
待验证假设
论坛资料观点
个人策略偏好
```

推荐标签：

```text
[已确认]
[暂定]
[待验证]
[资料观点]
[个人偏好]
```

---

## 2. 标准文章结构

每篇知识库文章使用以下结构：

```markdown
# 标题

> 版本：v0.x
> 分线：风险管理线 / 庄家周期线 / 语义概念线
> 目的：一句话说明本文解决什么
> 状态：初版 / 待补充 / 已吸收资料 / 待验证

---

## 1. 本文要解决的问题

## 2. 当前定义

## 3. 它不是什么

## 4. 阶段/结构拆解

## 5. 正向表现

## 6. 反向表现 / 风险表现

## 7. 常见误判

## 8. 专属关键词

## 9. 可检索资料方向

## 10. 可吸收的论坛/文章观点

## 11. 对当前策略的影响

## 12. 本版结论

## 13. 下一版待解决问题
```

## 3. 文件命名规范

### 3.1 正式文章

```text
主题名_v0.1.md
主题名_v0.2.md
```

示例：

```text
深洗与死亡盘语义深化_v0.1.md
POC重新接受定义_v0.1.md
二次启动与诱多反抽区分_v0.1.md
```

### 3.2 来源清单

```text
主题名_补全来源清单_v0.1.md
```

### 3.3 资料吸收笔记

```text
资料吸收_文章主题_来源简称_v0.1.md
```

### 3.4 归档文件

```text
旧主题名_v0.1_已归档_原因.md
```

## 5. 每次完善的固定动作

每次完善必须做：

```text
1. 判断属于哪条分线
2. 在对应目录创建/更新文章
3. 如果产生新关键词，更新关键词文档
4. 如果检索了资料，更新来源清单
5. 如果吸收了资料，写资料吸收笔记
6. 如果改变定义，更新总纲或分线文档
7. 更新版本日志
```

---

## 6. 高质量文档检查清单

写完后检查：

```text
[ ] 标题是否清楚？
[ ] 版本号是否清楚？
[ ] 分线是否清楚？
[ ] 本文目标是否清楚？
[ ] 定义是否和旧版本冲突？
[ ] 是否说明“不是什么”？
[ ] 是否包含正例/反例？
[ ] 是否提炼关键词？
[ ] 是否说明资料来源？
[ ] 是否写明下一步？
[ ] 是否更新版本日志？
```

## 7. 文风规范

使用：

```text
短句
明确标题
表格
代码块
阶段编号
结论先行
风险单独列出
待验证问题单独列出
```

避免：

```text
长段散文
模糊形容词
没有边界的定义
把猜测当事实
只写结论不写依据
只写中文不提英文关键词
```
