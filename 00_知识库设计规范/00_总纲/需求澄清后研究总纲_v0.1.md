本文不是新的交易规则，也不是回测结论，而是当前阶段的研究总纲。

```text
1. 策略到底是什么
2. 外部成熟理论怎么映射
3. 怎么证明策略不是自嗨

5. 怎么形成可持续的项目工作流
```

## 3. 项目总目标

本项目旨在研究并验证一个围绕以下概念的观察型策略模型：

```text
第一控盘区
成本中枢
深洗
死亡盘
重新接受
二次启动
诱多
失败反抽
```

项目目标不是证明某个主观观点一定正确，而是把该模型转化为可定义、可标注、可反证、可回测、可工程化审计的研究对象。

最终需要回答：

```text
1. 策略到底是什么？
2. 它和外部成熟理论有什么关系？
3. 它是否只是事后解释？
4. 它能否形成可标注、可回测、可执行的规则？
5. 它是否具备商业化、产品化、自动化潜力？
```

## 4. 方法论与项目分离原则

当前最需要先纠正的，不是某个单点术语，而是层级混杂：

```text
方法论层（怎么思考）
≠
项目层（具体研究什么）
```

对于本项目，AI 必须先知道三件事：

```text
1. 项目不是完整量化交易系统，而是一个结构性观察 / 样本标注 / 反证审计的研究工程。
2. 方法论文件只负责“怎么澄清、怎么推进、怎么验证”，不能替代项目目标本身。
3. 所有理论回填必须落在总纲、策略定义、样本标注、反证实验、关键词检索、资料吸收这些真源文件里，不能散在对话或工程交接碎片中。
```

这一层的目的是让 AI 能建立项目的整体概念认知：

```text
先识别项目是什么
再决定应该用什么方法推进
最后才进入具体规则、指标、回测、工程化实现
```

## 5. 量化体系分层判断：当前项目到底缺什么

这段是对“距离完整量化体系还差哪些层”的正式理论回填。

### 5.1 当前项目的真实定位

当前项目不是完整量化系统，而是：

```text
GMGN-driven FCZ Structural Observer
+ Sample Labeling Framework
+ Falsification Audit System
+ Risk Reviewer
```

更直白地说，它现在处在：

```text
研究型观察器 / 结构识别器 / 样本验证器
```

而不是：

```text
自动交易系统
自动下单系统
稳定盈利系统
```

### 5.2 与完整量化体系之间缺失的层

如果要升级为完整量化体系，至少还缺以下能力层：

```text
1. 数据源与权限层
2. 数据采集与原始数据层
3. 数据质量与多源校验层
4. 样本标注与反证层
5. 结构策略形式化层
6. 特征工程层
7. 信号与决策策略层
8. 回测层
9. 执行与下单层
10. 风控层
11. 监控与复盘层
12. 人机协同决策层
13. 合规与安全层
```

### 5.3 当前阶段允许做什么，不允许做什么

允许：

```text
- 做数据源研究
- 做样本标注
- 做结构定义
- 做反证样本库
- 做理论映射
- 做观察器 MVP
- 做验证门禁
```

不允许：

```text
- 直接跳到自动下单
- 直接宣称完整量化体系
- 直接把观察器当成交易系统
- 直接把方法论文件当成项目结果
```

### 5.4 这套判断对 AI 的意义

AI 以后看到相关资料时，不能只问：

```text
这个点能不能写进策略？
```

而要先问：

```text
这个点属于哪一层？
它是理论、样本、规则、回测、执行，还是方法论？
它是否已经有证据？
它能不能被当前阶段消费？
```

## 6. 主线 1：策略到底是什么

对应目录：

```text
01_策略定义/
02_庄家周期/
03_语义概念/
04_风险管理/
```

本主线解决模型自身定义问题。

当前不能停留在“感觉像主力洗盘”“看起来像回收成本中枢”这类语义描述上，而要把核心概念拆成可判断的结构。

### 4.1 第一控盘区到底如何锁定

需要明确：

```text
1. 第一控盘区是上涨前的最低筹码区，还是第一次明显放量建仓区？
2. 是否要求先打破原有下跌结构？
3. 是否要求形成震荡区间？
4. 是否要求后续价格多次回踩仍不跌破？
5. 是否允许多个候选控盘区？
6. 如果存在多个候选控盘区，优先级如何排序？
```

### 4.2 成本中枢到底用什么表达

需要判断成本中枢主要采用哪种表达：

```text
成交密集区
POC
Value Area
AVWAP
放量 K 线区间
结构震荡区间
多工具交叉确认区域
```

必须避免成本中枢变成事后随意画出来的框。

### 4.3 深洗和死亡盘如何区分

这是模型最关键的风险点。

需要回答：

```text
1. 跌破成本中枢后，什么情况叫深洗？
2. 什么情况叫真实破位 / 死亡盘？
3. 深洗允许跌多深？
4. 深洗允许持续多久？
5. 深洗过程中量能应该放大、缩小，还是允许分阶段变化？
6. 跌破后迟迟不能收回，是否判定失败？
7. 如果之后上涨但中间经历极长时间，是否还算原模型成立？
```

### 4.4 重新接受和普通反抽如何区分

重新接受不能只是“价格又涨回来了”。

需要定义：

```text
1. 是否要求重新站上成本中枢？
2. 是否要求重新站上 AVWAP？
3. 是否要求重新站回 Value Area？
4. 是否要求重新站上关键放量 K 高点？
5. 是否需要成交量确认？
6. 是否需要时间确认？
7. 是否需要之后不再快速跌回？
```

### 4.5 二次启动和诱多如何区分

需要明确：

```text
1. 什么是二次启动？
2. 二次启动是否必须突破前高？
3. 是否必须出现放量？
4. 是否必须出现缩量回踩不破？
5. 突破后失败如何定义？
6. 诱多样本如何收集？
7. 二次启动失败后是否直接判定模型无效？
```

## 5. 主线 2：外部成熟理论怎么映射

对应目录：

```text
05_关键词检索/
06_资料吸收/
```

本主线解决模型是否孤立、自创、不可交流的问题。

当前模型中的部分概念可以和成熟交易理论建立映射关系，但不能强行套壳。

### 5.1 Wyckoff Spring / LPS 映射

需要研究：

```text
1. 第一控盘区是否对应 Accumulation Range？
2. 深洗是否对应 Spring？
3. 重新接受是否对应 Back up / LPS？
4. 二次启动是否对应 SOS？
5. 当前模型和 Wyckoff 的差异在哪里？
```

重点不是证明“本模型就是 Wyckoff”，而是判断哪些部分能借鉴，哪些部分必须重新定义。

### 5.2 Liquidity Sweep 和深洗映射

需要研究：

```text
1. 深洗是否本质上是扫流动性？
2. 扫的是前低流动性，还是成本区下沿止损？
3. 扫完后是否必须快速收回？
4. 如果扫完后继续下跌，如何判定失败？
```

### 5.3 POC / Value Area 和成本中枢映射

需要研究：

```text
1. 成本中枢是否可以用成交量分布表达？
2. POC 是否适合作为核心成本？
3. Value Area 是否适合作为成本区间？
4. 在 A 股 / 币圈 / 美股不同市场中是否适用？
5. 数据粒度不够时如何替代？
```

### 5.4 AVWAP 锚点如何避免后视偏差

AVWAP 很容易出现后视偏差。

需要明确：

```text
1. 锚点能否只选已经发生的关键事件？
2. 是否允许事后选择最低点作为锚点？
3. 锚点是否来自第一次放量启动、结构破位点、阶段低点、第一控盘区形成点、财报 / 公告 / 事件日？
4. 如果多个锚点冲突，如何处理？
```

### 5.5 SMC OB / BOS / FVG 的角色

需要判断：

```text
1. Order Block 是核心结构还是辅助说明？
2. BOS 是否作为趋势确认？
3. FVG 是否作为回补 / 加速条件？
4. 这些概念是否会让模型过拟合？
5. 是否会让规则变得不可回测？
```

## 6. 主线 3：怎么证明策略不是自嗨

对应目录：

```text
07_样本标注/
09_规则与回测/
```

本主线最重要。

如果没有反证、失败样本、漏掉的大涨样本和消融实验，本模型很容易变成：

```text
看到涨了就说之前是控盘
看到跌了就说洗盘没结束
没有涨就说周期还没到
失败了就补一条规则
```

### 6.1 哪些样本能推翻模型

必须提前定义推翻条件。

可能的推翻样本包括：

```text
1. 出现完整控盘区、深洗、重新接受，但后续没有启动。
2. 出现重新接受后迅速跌回成本中枢下方。
3. 出现所谓二次启动后持续下跌。
4. 模型能解释上涨样本，但无法解释大量失败样本。
5. 大涨样本中大量不符合模型。
6. 图形成立但实际买点不可执行。
```

### 6.2 哪些失败不能再靠补规则解释

需要防止模型不断补丁化。

不能无限使用以下解释：

```text
这里不是标准深洗
这里主力没做完
这里周期还没结束
这里换庄了
这里是特殊情况
再加一个过滤条件就好了
```

必须设定原则：

```text
如果失败样本超过预设容忍范围，或者新增规则只能解释过去、不能提升未来样本表现，则模型需要降级或重构。
```

### 6.3 哪些大涨被模型漏掉

只研究命中的大涨股不够。

需要收集：

```text
1. 大涨但没有明显第一控盘区的样本。
2. 大涨但没有深洗的样本。
3. 大涨但没有成本中枢回收的样本。
4. 大涨来自消息、基本面、财报、政策、题材驱动的样本。
5. 纯趋势延续而非控盘回收结构的样本。
```

这些漏掉的样本用于判断：

```text
模型适用范围
模型不适用场景
模型是否只是某一类行情的子集
```

### 6.4 哪些样本图形成立但实际不可执行

需要检查：

```text
1. 买点是否出现在涨停板后无法买入？
2. 是否滑点极大？
3. 是否需要盘中判断而非收盘判断？
4. 是否止损空间过大？
5. 是否持仓时间过长？
6. 是否信号太少？
7. 是否信号集中在小盘高风险标的？
8. 是否依赖未来 K 线确认？
```

如果图形成立但交易不可执行，则不能算有效策略。

### 6.5 消融实验能否证明核心条件有效

需要做消融实验，例如：

```text
去掉深洗条件，表现是否变差？
去掉重新接受条件，表现是否变差？
去掉成交量过滤，表现是否变差？
去掉成本中枢，只用突破，表现是否变差？
只用外部理论条件，和当前模型相比如何？
```

如果去掉某个核心条件后表现没有下降，说明该条件可能只是叙事，不是策略优势来源。

## 7. 主线 4：怎么把理论变成规则、标注和回测

对应目录：

```text
07_样本标注/
08_图形模式/     # 若后续新增该目录
09_规则与回测/
10_指标与自动化/ # 若后续新增该目录；当前项目已有 08_指标需求/
```

本主线解决从理论到可执行的问题。

### 7.1 样本应该怎么标注

每个样本至少需要标注：

```text
标的代码
市场
时间区间
第一控盘区位置
成本中枢表达方式
是否出现深洗
深洗幅度
深洗持续时间
是否重新接受
重新接受确认点
是否出现二次启动
二次启动确认点
失败点
最早可观察买点
止损点
是否可执行
后续最大涨幅
后续最大回撤
持仓周期
标签置信度
标注者备注
```

### 7.2 规则应该如何拆分

规则需要分层：

```text
结构规则：判断是否存在第一控盘区、成本中枢、深洗、重新接受。
触发规则：判断什么时候可以买。
风控规则：判断什么时候失败、止损、放弃。
过滤规则：判断哪些市场环境、个股条件、成交量条件不适合。
退出规则：判断什么时候止盈、减仓、失效。
```

### 7.3 回测应该先验证什么

第一阶段不应追求复杂收益曲线，而应验证：

```text
1. 信号是否能被程序识别。
2. 识别结果是否和人工标注一致。
3. 信号是否使用未来数据。
4. 买点是否可交易。
5. 失败样本是否被记录。
6. 核心条件是否真的提高胜率或收益风险比。
```

### 7.4 自动化指标应该怎么做

建议顺序：

```text
1. 人工标注模板
2. 样本库结构
3. 规则定义文档
4. 半自动识别脚本
5. 标注一致性检查
6. 回测原型
7. 指标可视化
8. 最后才考虑交易信号系统
```

## 8. 主线 5：怎么形成可持续的项目工作流

对应目录：

```text
00_知识库设计规范/

```

当前仓库中已有：

```text

```

因此，本主线的重点不是重新发明工作流，而是把需求澄清后的研究结论补进现有工作流的前置输入层。

### 8.1 哪些内容应该沉淀为项目上下文

需要沉淀：

```text
当前策略定义
术语表
五条研究主线
反证标准
样本标注规范
外部理论映射
阶段性结论
Owner 决策
后续实验计划
```

### 8.2 哪些内容应该进入方法轮

方法轮里应该进入已经收敛后的材料：

```text
Good Question Brief
需求澄清结论
策略研究总纲
术语定义
待验证假设
反证标准
可执行任务清单
Codex handoff 输入
```

### 8.3 哪些事情不能交给 Codex 自己发挥

Codex 不应自行决定：

```text
策略核心定义
成本中枢采用哪种表达
失败样本是否算失败
是否引入某个外部理论
是否修改策略逻辑
是否扩大研究范围
是否美化结果
```

Codex 可以执行的任务包括：

```text
整理目录
生成模板
写校验脚本
读取样本 CSV
做标注一致性检查
做简单回测原型
生成统计报告
```

### 8.4 哪些必须由 Owner 决策

需要 Owner 最终决定：

```text
第一控盘区的默认定义
成本中枢优先采用哪种表达
深洗最大容忍范围
重新接受的确认标准
失败样本是否纳入
回测市场和周期
是否追求交易系统化
是否只是研究模型
是否进入工程实现阶段
```
