# Meme 第一控盘区深洗回收模型：知识库总纲 v0.1

> 位置：`00_知识库设计规范/00_总纲/知识库总纲_v0.1.md`  
> 用途：作为整个知识库的最高层入口，说明这套知识库研究什么、怎么分线、怎么更新、哪些文档应该放在哪里。  
> 原则：所有后续完善内容都必须落到文件，不依赖聊天上下文。

## 1. 知识库主题

本知识库围绕一套用于 meme 市场的保守型结构策略：

```text
meme 庄家第一控盘区深洗后二次启动确认策略
```

英文定位：

```text
Meme Market Maker Re-accumulation Reclaim Strategy
```

核心研究对象：

```text
庄家第一控盘区
第一波拉升
深度回撤 / 洗盘
成本中枢回收
POC / AVWAP 重新接受
第一次拉起
回撤不破成本锚
二次拉升确认
诱多 / 死亡盘 / 尾仓派发过滤
```

---

## 2. 知识库目标

不是直接得到一个买卖信号，而是逐步建立：

```text
1. 清晰的策略定义
2. 清晰的庄家阶段模型
3. 清晰的语义概念体系
4. 清晰的风险管理语言
5. 清晰的专属关键词体系
6. 清晰的论坛检索与资料吸收机制
7. 清晰的版本迭代机制
8. 最后才进入规则、指标、回测和交易系统
```

---

## 3. 当前策略最高定义

```text
这是一个 meme 市场中的庄家第一控盘区深洗后，基于 POC / AVWAP 成本锚重新接受与二次拉升确认的保守型 re-accumulation / reclaim 观察策略。
```

它不是：

```text
不是抄底策略
不是第一波追涨策略
不是单纯Fib买入策略
不是单纯POC支撑策略
不是单纯VWAP回踩策略
不是叙事热度追涨策略
```

它是：

```text
庄家控盘结构识别
深洗后重新吸筹识别
成本区回收确认
二次启动确认
诱多与死亡盘过滤
```

## 5. 分线体系

当前知识库分为 8 条线：

```text
A. 策略定位线
B. 庄家周期线
C. 语义概念线
D. 风险管理线
E. 关键词检索线
F. 资料吸收线
G. 样本标注线
H. 指标规则线
```

其中，资料吸收线必须有一张主线治理表，避免所有吸收笔记同权：

```text
06_资料吸收/资料吸收主线参考总表_v0.1.md
```

这张表负责把“主线核心 / 主线辅助 / 参考资料”分开，供总纲、策略定义和样本标注优先消费。

每条线必须有：

```text
分线目标
当前版本
核心问题
产出文件
待检索关键词
下一步任务
```

## 6. 版本迭代机制

每次完善都必须形成文章，不能只停留在上下文。

每次迭代遵循：

```text
提出问题
-> 拆语义
-> 提关键词
-> 检索资料
-> 下载资料
-> 形成补全来源清单
-> 写资料吸收文章
-> 更新总纲/分线/词典/风险文档
-> 更新版本日志
```

## 8. 知识库维护原则

```text
所有重要结论必须写入文件
所有新关键词必须进入关键词文档
所有检索结果必须进入来源清单
所有吸收内容必须形成吸收笔记
所有定义变化必须进入版本日志
所有旧定义不删除，放入归档或标记为降级
```

## 9. 当前最重要的下一步

建立 V0.2 专题：

```text
深洗 vs 死亡盘语义深化
```

因为 meme 市场中最危险的误判就是：

```text
把死亡盘当洗盘
把诱多反抽当二次启动
把成本锚触碰当成本锚重新接受
```
