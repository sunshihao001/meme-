## 1. Good Question

```text
在 meme / 高波动标的中，“第一控盘区成本中枢回收”结构，是否能以 GMGN 作为优先研究数据源，在事前被稳定识别为一类区别于普通反抽和诱多的观察样本，并且在经过多源校验、反证样本、消融实验、可执行性评估和观察器验证后，具备升级为可回测交易系统的可能性？
```

更短版本：

```text
第一控盘区成本中枢回收结构，能否从主观叙事变成可定义、可反证、可标注、可回测、可执行评估的策略研究对象？
```

---

## 2. Background

当前项目围绕以下交易结构展开：

```text
第一控盘区
成本中枢
深洗
重新接受
二次启动
诱多 / 失败反抽
死亡盘
```

前期讨论已经形成大量术语、外部理论映射、反证样本设计和观察指标需求，但仍存在风险：

```text
1. 策略可能只是事后解释。
2. 成本中枢和 AVWAP 锚点可能存在后视偏差。
3. 深洗和死亡盘可能无法事前区分。
4. 重新接受可能只是普通反抽。
5. 二次启动可能与诱多混淆。
6. 图形成立可能不等于交易可执行。
7. Codex 若过早介入，可能会把未确认假设固化为代码。
```

因此当前阶段先做需求澄清和研究上下文固化，而不是直接进入实现。

## 6. Non-goals

当前阶段明确不做：

```text
1. 不做自动下单系统。
2. 不承诺收益率。
3. 不把观察指标写成买卖信号神器。
4. 不直接进入 Codex 实现策略逻辑。
5. 不直接做正式回测结论。
6. 不把外部理论直接升级为核心规则。
7. 不因为失败样本而事后修改定义。
8. 不把图形成立等同于交易可执行。
```

## 7. Core Assumptions To Test

```text
H1. 第一控盘区在一部分 meme / 高波动标的中确实存在，并能代表早期主要资金成本或价值共识。

H2. 成本中枢可以通过 POC / Value Area / AVWAP / 结构区间等方式在事前近似表达。

H3. 深洗和死亡盘虽然相似，但存在可观察差异，至少可以通过样本标注逐步区分。

H4. 重新接受成本中枢比普通反抽更有可能对应后续二次启动。

H5. 二次启动、诱多、失败反抽之间可以通过结构、成交、回撤和执行结果建立可标注差异。

H6. 该模型最终是否能进入交易系统，取决于反证样本、消融实验、可执行性和回测结果，而不是理论叙事本身。

H7. 即使策略最终不能自动交易，观察器 / 标注器 / 风险审计器仍可能有独立价值。
```

## 8. Falsification Criteria

以下情况会削弱、降级或阻止模型进入交易系统阶段：

```text
F1. 符合第一控盘区 + 深洗 + 重新接受 + 二次启动的样本，后续大量失败。

F2. 所谓成功样本主要依赖事后重画第一控盘区、移动 AVWAP 锚点或修改成本中枢。

F3. 去掉深洗、重新接受或二次启动等核心条件后，表现没有明显变差，说明核心条件只是叙事。

F4. 大量大涨样本不符合该模型，却能被更简单规则捕获，说明模型过度复杂或覆盖率不足。

F5. 图形上成立的信号在真实交易中不可执行，例如滑点过大、买不到、止损过远、流动性不足。

F6. 模型需要不断新增过滤条件才能解释失败，且新增条件不能事前识别。

F7. AI Agent / Codex 无法在不自行解释策略的情况下稳定标注样本，说明定义仍不够清晰。

F8. 回测或样本验证显示收益主要来自市场 beta / 暴涨行情，而非模型条件本身。
```

## 10. Recommended Path

Owner 已确认路线：

```text
先 A，再 B，最后 C。
```

```text
A. 研究库优先
   clarification / Good Question Brief / 定义 / 术语 / 反证 / 样本模板 / Owner 决策

B. 半自动标注器 / 观察器优先
   MQL5 / 脚本观察器 / 样本标注辅助工具

C. 回测原型优先
   在规则、样本、可执行性、反证框架较稳定后进入
```

Hard gate：

```text
未完成 A，不进入 B。
未完成 B 的观察与标注验证，不进入 C。
```
