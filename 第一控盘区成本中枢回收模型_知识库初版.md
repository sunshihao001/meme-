## 1. 一句话定义

该模型不是 Fib 买入法、POC 支撑买入法或 AVWAP 回踩买入法，而是：

```text
第一控盘区成本中枢被深回撤测试后，重新接受 POC / AVWAP，并经二次拉升确认的保守观察结构。
```

短命名：

```text
第一控盘区成本中枢回收模型
```

更工程化命名：

```text
POC价值区回收 + AVWAP成本锚重新接受 + 深回撤后二次结构确认模型
```

英文研究命名：

```text
Volume Profile / Market Profile Acceptance Reclaim Strategy
VWAP / Anchored VWAP Cost Anchor Reclaim Strategy
BOS-confirmed Pullback Continuation Strategy
```

## 3. 模型标准结构路径

```text
第一波控盘区出现
-> 形成筹码峰 / 成本中枢 / POC
-> 第一波拉升到控盘派发高点或结构高点
-> 深回撤到 0.618 - 0.786 区间
-> 重新接近 / 站回 POC
-> 以第一控盘区或关键事件为锚点计算 AVWAP
-> 价格拉起一小段
-> 回撤不破关键成本锚
-> 再次拉升确认
-> 只研究二次确认后的稳定小段
```

---

## 4. 该模型真正想捕捉的内容

不是：

```text
最低点
第一波主升
单次 POC 支撑
单次 AVWAP 回踩
单次 Fib 0.618 / 0.786 买点
```

而是：

```text
控盘成本区被深回撤测试后，市场重新接受该成本区，并经过第一次拉起、回撤不破、再次拉升确认后的保守延续段。
```

---

## 5. 当前核心判断句

```text
价格是否在深回撤后重新接受了第一控盘区成本中枢，并且在第一次拉起后的回撤中没有重新跌破关键成本锚，随后出现了二次拉升确认？
```

如果答案不是明确的“是”，则不应归入该模型。

```text

```

只有进入 S8 / S9，才进入策略研究区。

## 8. 关键失效条件

### 8.1 POC 回收失败

```text
价格接近 POC 后无法站回
多次冲击 POC 被压回
POC 从支撑转为压力
```

### 8.2 AVWAP 接受失败

```text
站上 AVWAP 后快速跌回
AVWAP 上方没有持续成交
回撤跌穿 AVWAP 后无法收回
```

### 8.3 二次确认失败

```text
第一次拉起后回撤破成本锚
第二次拉升无法突破第一次反抽高点
二次拉升成交量不足
```

### 8.4 派发风险过高

```text
第一波高点巨量滞涨
连续上影线
反抽期间主动卖盘持续增强
大额钱包/资金持续流出
```

结论：

```text
降级为诱多反抽 / 尾仓派发风险模型
```
