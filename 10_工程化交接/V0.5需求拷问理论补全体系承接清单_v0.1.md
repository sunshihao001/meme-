
## 2. V0.5 核心原则

```text
业务项目：项目现实 > 方法轮。
方法论知识库：方法轮可以独立演化。
```

落地规则：

```text
1. 不把方法轮全量塞进业务项目。
2. 业务项目只保留当前推进所必需的最小控制协议、决策记录、理论补丁、样本、验证。
3. 方法轮本身的完整演化留在 dbskill_claude_platform_design_kb。
4. 当前项目用 V0.5 作为外部 control-plane / 工作协议。
5. 每轮只处理一个最高价值问题，不搞快节奏全量重构。
```

## 7. 本轮检验：V0.5 是否比普通流程更有效

本轮按 V0.5 执行后，区别是：

```text
普通快流程：会直接继续写 README / issue / validator。
V0.5 慢流程：先识别“为什么 Owner 觉得散、为什么完善过的内容没被接入体系”，再只做承接清单，不重构项目。
```

当前达成的实际效果：

```text
1. 找到了上游 V0.5 真源。
2. 明确业务项目不是方法论仓库。
3. 列出了 V0.5 每个部件在当前项目中的承接状态。
4. 找出了已完善但容易被遗漏的部件。
5. 给出最小补强动作，而不是全量重构。
```

仍未达成的效果：

```text
1. 还没有把 Project Continuation Brief 独立落地。
2. 还没有把 Concept Search Absorption Loop 做成 GitHub issue 模板。
3. 还没有 validator 检查 loop report。
```
