## 5. 旧知识体系吸收原则

### 5.1 不搬迁，先吸收

```text
旧目录继续作为 source archive。
当前主线只吸收被验证有用的语义、字段、状态、样本规则、指标需求。
```

### 5.2 旧资料不能直接覆盖真源

旧资料进入当前主线前必须经过：

```text
1. 当前真源比对。
2. 冲突标注。
3. 可吸收 / 不可照搬 / 待验证 分类。
4. 写入正式分线或降级为参考。
5. 更新索引和变更记录。
```

### 5.3 每轮只选一个主题包

避免“全库吸收”导致失控。每轮循环代理只处理一个主题包，例如：

```text
Wyckoff/Spring/LPS 旧资料包
Liquidity Sweep / BOS 旧资料包
POC / Market Profile / Volume Profile 旧资料包
VWAP / AVWAP 成本锚旧资料包
Order Block / Inducement / BOS 旧资料包
指标代码库 → MQL5 观察器字段包
根目录旧总文档 → 当前策略定义冲突包
```

### 5.4 每轮必须有反馈闭环

每轮完成定义：

```text
吸收文件存在
目标真源被更新或明确无需更新
索引引用新增文件
变更记录新增阶段
validate_all 通过
下一轮主题包被选择
```

## 7. 建议新增任务包 TP-010

### TP-010：旧知识体系吸收融入循环代理

优先级：

```text
P2.5：在 index validator 之后、MQL5 观察器 MVP 之前执行。
```

原因：

```text
如果不先把旧知识吸收路线做成循环代理任务，MQL5 MVP 可能重复旧字段、漏掉旧状态机、或重新发明已有定义。
```

可自动推进：

```text
部分可自动。Agent 可读旧资料、写差异审计、更新索引/变更记录/validator；但策略含义、样本归类、商业可用性判断需要 owner 或 checker 复核。
```

推荐首个执行切片：

```text
Loop 2：策略完善_v0.2 → 状态机 / 指标字段 / 样本模板对齐
```

理由：

```text
这是旧知识体系与当前主线最容易发生重复/漂移的位置，并且直接影响样本、validator、MQL5 观察器 MVP。
```

## 8. 最终目标路径

本项目的最终目标不应写成“生成自动交易 EA”。当前可达成的最终目标应分层：

### Final Target A：可审计策略研究知识库

```text
旧知识已吸收/降级/归档
当前真源清晰
样本和反证主线可执行
索引和变更记录可验证
```

### Final Target B：MQL5 观察器 MVP 准备完成

```text
状态机字段稳定
POC/AVWAP/FCZ/sweep/reclaim 字段稳定
风险评分字段有样本依据
MQL5 指标需求可交给实现 agent
```

### Final Target C：循环代理可持续运行

```text
每轮任务从 Demand Grilling Brief 开始
每轮有 maker/checker 和 authority boundary
每轮写入项目文件
每轮 validate_all
每轮选择下一主题包
方法缺陷回写 ai- 上游仓库
```

### 不作为当前最终目标

```text
自动下单 EA
收益承诺
全自动策略判断
无需人工复核的交易系统
```
