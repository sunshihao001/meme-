# 优先级问题清单任务包 v0.1

> 分线：10_工程化交接 / 任务包推进  
> 状态：v0.1 / 可自动推进清单  
> 目的：把 Project Continuation Brief 中的补全项转化为可排序、可自动推进、可回填、可验证的任务包，避免补全清单停留在静态建议。

---

## 1. 任务包选择原则

按 V0.5 需求拷问端，本清单不按“看起来重要”排序，而按以下 5 个问题排序：

```text
1. 是否减少 owner 口头提醒？
2. 是否能让后续 Agent 更容易找到入口？
3. 是否能被自动验证？
4. 是否不需要 owner 决策即可自主完成？
5. 是否能回填到项目理论 / 工作流文件，而不只是生成报告？
```

优先级解释：

```text
P0：入口 / 控制面 / 防假完成。先做，否则后续任务会继续散。
P1：样本与反证主线。让理论可反证。
P2：validator / CI。把规则变成自动反馈。
P3：资料吸收与旧资料桥。减少知识散落。
P4：MQL5 观察器 MVP。等语义、样本、字段更稳后再推进工程实现。
P5：GitHub / 多 Agent 协作成熟化。需要 owner 决策或外部权限。
```
