# A→B 交接卡：A端已可转B v0.1

> 分线：10_工程化交接 / A→B 交接
> 用途：把 A 端已经收束完成的内容，交给 B 端继续做证据压缩与 Source Pack。

---

## 1. 结论

A 端当前已经达到**可交接到 B 端**的状态，但还不属于“全部完工”。

A 端已完成的部分：

- 默认入口与跳过条件
- 自动提问 / 自动判断 / 自动调用技能 / 自动推理
- A/B/C 收束停机线
- 状态机与路由表
- 多路线引导
- dbskill 持续学习定位

所以现在可以进入 B 端，继续做：

- 外部资料读取
- Source Pack 压缩
- 证据整理
- 冲突 / 缺口 / unknown 归纳

---

## 2. A 端交付给 B 端的输入

B 端先接收以下内容：

```text
1. 当前想法核心
2. A 端已经确认的目标
3. A 端已经确认的边界
4. 当前知识缺口
5. 需要外部证据支撑的点
6. 暂时不要展开的点
7. 统一停机口径
```

### 当前统一停机口径

```text
1. 目标、边界、方案、验证四项中至少 3 项达到可执行级别。
2. 连续 2 轮没有新增有效信息。
3. 最多循环 2~3 轮后，强制复盘是否该转端。
4. 如果继续追问只会改细节、不改路线，就停止研究循环。
```

---

## 3. B 端要做什么

B 端的任务不是继续问同样的问题，而是补证据。

### B 端优先任务

- 读取外部来源
- 压缩成 Source Pack
- 标出核心观点
- 标出可吸收点 / 不可照搬点
- 标出冲突 / 缺口 / unknown
- 判断是否足以进入 C 端

### B 端建议技能

```text
Agent Reach
+ dbs-learning
+ dbs-content-system
+ dbs-report
+ dbs-save
+ dbs-restore
```

如果概念噪声仍然大，可以加：

```text
dbs-deconstruct
```

---

## 4. B 端第一轮输出格式

B 端第一轮建议固定输出成：

```text
【Source Pack】
- 来源清单
- 核心观点
- 与当前问题的关系
- 可吸收点
- 不可照搬点
- 冲突 / 缺口 / unknown

【当前判断】
- 是否足以进入 C
- 还是要回 A 继续最小澄清

【下一步】
- 继续补证据
- 或转 C 端收束
```

---

## 5. 什么时候从 B 回 A

如果 B 端发现以下情况，就回 A：

- 问题定义又变模糊了
- 外部证据不足
- 来源之间冲突太大
- 需要重新收束边界
- 发现当前路线需要改

回 A 后只做最小补齐，不重新把全部问题再问一遍。

---

## 6. 什么时候从 B 转 C

如果 B 端已经满足以下条件，就转 C：

- Source Pack 已有足够证据
- 当前问题边界清楚
- 方案方向已经基本稳定
- 需要理论深化、方案治理、PRD / plan / issue

---

## 7. 对 B 端的启动口令

你可以直接复制这段给 B 端：

```text
请按 B 端处理。
当前 A 端已经完成最小澄清与收束，请继续做证据压缩与 Source Pack。
不要重复 A 端拷问，不要直接给最终方案。
先输出来源清单、核心观点、可吸收点、不可照搬点、冲突与缺口。
如果证据足够，再判断是否进入 C 端。
```

---

## 8. 一句话结论

```text
A 端已经可以交给 B 端；B 端负责补证据，C 端负责再收束，A 端只在边界变模糊或路线变化时回补。
```
