> 目的：把“旧知识体系吸收”的说法修正为“项目内散乱知识点归并”，并给出当前哪些已进入真源、哪些只是归位、哪些需样本验证、哪些应进入观察器或风险/状态机主线。  
> 边界：不批量移动目录，不把历史阶段材料视为废弃物，不把未验证阈值升级为策略结论。

## 1. 口径修正

不再把项目内历史阶段材料统一称为：

```text
旧知识 / legacy knowledge / obsolete material
```

更准确的称呼是：

```text
项目内散乱知识点
历史阶段知识碎片
source fragments
同一项目不同阶段形成的材料
```

原因：

```text
1. 这些材料不是外部废弃物，而是同一项目在不同阶段形成的来源。
2. 很多内容已经被当前 01-09 真源吸收，只是路径、命名和验证层不同。
3. 真正的问题不是“旧不旧”，而是“是否已经归并进当前真源 / validator / 样本 / 观察器 / 风险模型”。
```

后续文件和报告应优先使用：

```text
散乱知识点归并
source fragments consolidation
项目内知识归并
主题包归并
```

只有在明确指历史目录时，才可使用“历史阶段材料”。

## 2. 当前总体判断

当前不是：

```text
旧知识没有吸收。
```

而是：

```text
项目内散乱知识点已经有归位索引；
其中一部分已经进入当前真源；
一部分已经变成 validator / MQL5 MVP / 样本记录；
但仍有若干主题包没有逐包收敛成当前字段、状态机、风险模型、观察器字段或样本验证任务。
```

当前已有支撑文件：

```text

```

其中第二、三个文件的标题仍保留“旧知识/旧版”历史命名，本文件作为新的口径承接层，不立即批量重命名，避免破坏索引和历史提交。

## 3. 归并状态分类

本文件使用以下分类：

```text
integrated：已进入当前真源或工程资产。
indexed_only：已经在归位索引中登记，但尚未逐包归并。
partially_integrated：部分进入当前真源，但仍缺总字段 / 映射 / 样本验证。
needs_sample_validation：不能凭概念直接进入核心规则，需真实样本验证。
observer_candidate：适合进入 MQL5 观察器或指标字段，但不是交易信号。
reference_only：保留为来源，不进入当前主规则。
superseded_with_alias：当前模型已升级，但需保留映射以读懂历史阶段材料。
```

## 6. 当前优先级判断

现在已经完成：

```text
TP-003 index reference validator
TP-004 sample record markdown validator
MQL5 观察器 MVP 编译与安装验证
```

下一步不建议直接继续堆 MQL5 功能，因为 GUI 挂载仍是 blocker；也不建议大范围归并全部散乱知识点。

推荐下一切片：

```text
状态机 legacy_alias_map 小切片
```

原因：

```text
1. 它体积小，可验证。
2. 它直接解决历史阶段材料与当前 STATE_* 的读法不一致。
3. 它会降低后续 Agent 误读 S0/S1/SX 的概率。
4. 它不需要外部数据、不需要 GUI、不需要 owner 决策。
5. 它为后续 POC/AVWAP、Wyckoff、Liquidity Sweep 资料包归并提供统一状态语言。
```

推荐产物：

```text
09_规则与回测/状态机legacy别名映射_v0.1.md
```

或如果要保持一个主状态机文件：

```text
09_规则与回测/状态机规则整合_v0.2.md
```

更稳妥选择：

```text
先新增 `09_规则与回测/状态机legacy别名映射_v0.1.md`，不直接改 v0.1 主文件。
```

## 8. 本版结论

```text
1. 项目不是缺“旧知识吸收”，而是缺“项目内散乱知识点归并状态图”。
2. 当前已有大量内容已经进入真源和 validator，不应重复归并。
3. 真正剩余的是少数主题包级别的翻译层、字段聚合、样本验证和观察器字段差异归并。
4. 下一步最小可验证切片是状态机 legacy alias map。
5. 所有后续归并都必须保持观察器/研究工具边界，不进入自动下单结论。
```
