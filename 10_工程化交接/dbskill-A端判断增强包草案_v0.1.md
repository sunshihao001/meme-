# dbskill-A端判断增强包草案 v0.1

> 分线：10_工程化交接 / 外部技能接入 / D端落库草案  
> 输入：dbskill第一批SourcePack_v0.1.md、dbskill-A端判断增强包设计_v0.1.md  
> 状态：v0.1 / 内部包装草案，尚未升级为 Hermes 正式 skill  
> 目的：把 dbskill 第一批四个外部技能包装成 A端可调用的判断增强流程草案。

---

## 1. 包装定位

```text
名称：dbskill-A端判断增强包
身份：A端辅助增强包
层级：L2/L3
动作：MERGE + BRIDGE
状态：草案 / 待验证 / 未固化
```

它不是：

```text
- 不是新的 A端主 skill
- 不是 dbskill 全量安装
- 不是 F端 owner 决策替代品
- 不是内容生产工具包
```

它是：

```text
帮助 A端生成更强 Demand Grilling Brief 的外部技能桥接包装。
```

---

## 2. 内部模块

```text
M1 问题说明书模块：来自 dbs-good-question
M2 概念拆解模块：来自 dbs-deconstruct
M3 目标审计模块：来自 dbs-goal
M4 商业问题消解模块：来自 dbs-diagnosis
```

---

## 3. 触发规则

### M1 问题说明书模块

触发：

```text
用户输入模糊、边界不清、想自动化但不知道能否交给 Agent。
```

必须输出：

```text
对象 / 目标 / 冲突 / 约束 / 反馈 / Agent 可解性
```

---

### M2 概念拆解模块

触发：

```text
用户使用未定义概念、伪概念、行业黑话、商业抽象词。
```

必须输出：

```text
模糊词 / 使用场景 / 大白话改写 / Question or Problem / 隐含假设
```

---

### M3 目标审计模块

触发：

```text
用户说“我想做…”“我要完善…”“目标是…”但没有完成态。
```

必须输出：

```text
原始目标 / 空转词 / 可检查目标 / 验收标准 / 下一步动作
```

---

### M4 商业问题消解模块

触发：

```text
涉及商业模式、变现、增长、项目价值、外部技能是否值得接入。
```

必须输出：

```text
问题类型 / 语言陷阱 / 假设错误 / 逻辑错误 / 事实前提 / 信息缺口
```

---

## 4. 调用顺序

默认：

```text
M1 → M2 → M3 → M4 → 路由建议
```

可跳过条件：

```text
1. 如果没有模糊概念，可跳过 M2。
2. 如果目标已有验收标准，可跳过 M3。
3. 如果不是商业/业务判断，可跳过 M4。
4. 如果输入很简单，不启动增强包。
```

## 6. 权限边界

```text
1. A端仍由 meme-demand-control-port 主控。
2. 本增强包只提供判断辅助。
3. dbskill 外部语气和商业公理不能直接写成项目结论。
4. 需要 E端验证后，才能考虑固化为正式 skill。
5. 若样例验证不能提升判断质量，应保留为 referenced_only。
```

---

## 7. 验证入口

验证样例：

```text
我现在内部技能不够完善，想从 GitHub 外部技能仓库添加技能，
但不知道哪些该接入、怎么判断、怎么放进我的 A/B/C/D/E/F 多端口循环里。
```

通过标准：

```text
1. 能识别为外部技能接入问题。
2. 能拆清内部技能、外部技能、桥接、合并、拒绝。
3. 能输出增强版 Demand Grilling Brief。
4. 能给出端口路由。
5. 不越权替 F 端做最终固化决策。
```
