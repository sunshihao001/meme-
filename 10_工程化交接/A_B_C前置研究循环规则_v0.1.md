# A/B/C前置研究循环规则 v0.1

> 分线：10_工程化交接 / 多端口工作流重构  
> 用途：把“想法形成 → 初步定义 → 定向搜索 → 资料回流 → 理论深化”固定成可复用规则，避免 A/B/C 空转。  
> 状态：v0.1 / 正式规则草案  
> 目标：让外部资料搜索基于 A 端初步框架，而不是无边界乱搜。

---

## 1. 核心结论

```text
A 端先把想法变成可研究、可搜索、可判断的初步框架。
B 端根据 A 端框架去搜索外部来源，并压缩成 Source Pack。
A 端对 B 端资料做吸收判断，再决定是否交给 C 端生成长理论。
C 端基于 A 端框架 + B 端 Source Pack 做理论深化。
```

一句话：

```text
先定义，再搜索；先压缩，再深化；先判断，再吸收。
```

---

## 2. 这个循环解决什么问题

### 要解决的痛点

```text
1. 只凭聊天空想，工作流容易空转。
2. 资料搜索没有依据，容易搜偏。
3. C 端直接长理论，容易先入为主。
4. 搜索结果太散，无法进入知识库。
5. 外部知识没有被结构化吸收，无法提升方法轮智能度。
```

### 目标

```text
把外部知识变成可追溯、可判断、可沉淀的研究输入。
```

## 3. 总流程

```text
L0：用户自然语言想法
  ↓
A0：想法保留与语义解析
  ↓
A1：初步框架定义
  ↓
A2：搜索任务定义
  ↓
B1：搜索策略简报
  ↓
A3：批准 / 修改搜索策略
  ↓
B2：外部来源搜索
  ↓
B3：Source Pack 压缩
  ↓
B4：Knowledge Fit Report
  ↓
A4：吸收判断
  ↓
C1：长理论生成
  ↓
D：落库
  ↓
E：验证
  ↓
F：Owner 决策
```

## 4. A 端职责

A 端不是直接“问一句要什么”，而是先把想法整理成可搜索任务。

### A 端要输出的内容

```text
1. 当前想法：这次到底要完善什么？
2. 初步定义：A 端已经把想法整理成什么概念？
3. 知识缺口：现在不知道什么？
4. 搜索目标：要找理论、工具、案例、代码、论文、开源项目，还是成熟工作流？
5. 搜索关键词：中文、英文、相关术语、反向关键词。
6. 优先来源：GitHub / 官方文档 / 论文 / 博客 / X / Reddit / Hacker News / 视频。
7. 排除范围：不要搜什么，避免跑偏。
8. 返回格式：B 端必须怎么回。
```

### A 端判断重点

```text
- 这个问题是否已经足够具体？
- 搜索范围是否足够小？
- 资料回来后，A 端要判断什么？
- 这次是快速模式还是深度模式？
```

---

## 5. B 端职责

B 端是外部知识储备与资料压缩端，不是随便贴链接的端口。

### B 端建议拆成 4 个子动作

#### B1：搜索策略简报

```text
先返回“我准备怎么搜”，而不是直接搜。
```

内容包括：

```text
- 搜索关键词
- 优先平台
- 过滤条件
- 预期来源类型
- 预计得到的材料形态
```

#### B2：外部来源搜索

```text
根据 A 端框架，快速检索成品资料、半成品资料、理论、案例、实现、反例。
```

#### B3：Source Pack 压缩

不是原始链接堆，而是压缩后的研究输入：

```text
- 来源名称
- 链接
- 来源类型
- 核心观点
- 可吸收点
- 不可直接照搬点
- 和当前 A 端想法的关系
- 适合进入哪个端口
- 风险与噪声判断
```

#### B4：Knowledge Fit Report

```text
说明这些资料是否真正回答了 A 端的知识缺口，是否需要回 A 或重搜。
```

---

## 6. C 端职责

C 端只在资料已经被 A 端初步确认后，再做长理论深化。

### C 端输入

```text
- A 端初步框架
- B 端 Source Pack
- A 端吸收判断
```

### C 端输出

```text
- 长理论研究文档
- 框架改造建议
- 方法轮升级建议
- skill / template / 知识库结构建议
- 风险、反例、未决问题
```

### C 端禁止事项

```text
- 不允许只靠聊天空想直接写最终理论。
- 不允许绕过 B 端外部资料输入。
- 不允许把未验证观点写成已确认结论。
```

## 7. 搜索任务单模板

A 端发给 B 端时，建议包含以下字段：

```text
1. 当前想法
2. 初步定义
3. 知识缺口
4. 搜索目标
5. 搜索关键词
6. 优先来源
7. 排除范围
8. 返回格式
```

### 示例

```text
当前想法：建立外部知识储备循环，避免方法轮空转。

初步定义：A 端先把问题定义清楚，B 端再去找外部资料，C 端基于资料做理论深化。

知识缺口：
- 如何定义搜索依据
- 如何判断资料是否适合吸收
- 如何把外部资料压缩成 C 端可用输入

搜索目标：
- 成熟工作流
- 研究型框架
- 知识管理方法
- source pack / evidence pack / knowledge fit 相关实践

搜索关键词：
- context engineering
- agent workflow
- knowledge reserve
- source pack
- evidence-based workflow
- human-in-the-loop research loop

优先来源：GitHub、官方文档、论文、方法论文章、案例复盘。

排除范围：
- 只讲概念、不讲流程的空文
- 纯营销文章
- 与当前方法轮无关的泛 AI 讨论

返回格式：Source Pack + Knowledge Fit Report
```

---

## 8. 回退规则

```text
1. 问题不对 / 方向不对 → 回 A
2. 资料不够 / 证据弱 → 回 B
3. 理论浅 / 结构乱 / 方案不专业 → 回 C
4. 资料回来但不适合吸收 → 回 B 重搜或暂存
5. Owner 仍不满意 → 回 A 重新定义想法
```

## 9. 快速模式与深度模式

### 快速模式

适合小问题、小补强。

```text
A 初步定义
→ B 快速搜 5-10 个高质量来源
→ A 判断
→ C 短理论生成
```

### 深度模式

适合方法轮升级、端口改造、知识框架更新。

```text
A 初步框架
→ C0 轻量辅助概念深化
→ A 定义搜索任务
→ B 多来源搜索
→ A 吸收判断
→ C1 长理论生成
→ D/E/F
```

---

## 10. 本版结论

```text
外部知识搜索必须由 A 端初步框架驱动。
B 端负责搜索与压缩，不负责乱开题。
C 端负责在资料输入后做理论深化，不负责凭空写结论。
```
