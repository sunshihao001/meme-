## 3. A/B/C 知识研究循环

```text
A：把自然语言想法变成专业问题
B：把资料、上下文、证据压成 Source Pack
C：把 Source Pack 生成理论框架和候选方案
Owner：不满意可随时回 A/B/C 调整
```

### A/B/C 收束停机线

```text
当目标、边界、方案、验证四项中至少三项达到可执行级别，
且继续追问只会影响细节，不会改变路线时，停止 A/B/C 循环。
```

补充门槛：

```text
- 连续 2 轮没有新增有效信息时停止
- 最多循环 2~3 轮后强制复盘是否转端
- 停机后转入 spec / issue / 执行版方案，而不是继续空转
```

什么时候回 A：

```text
方向不对 / 题目不对 / 真实意图没抓准
```

什么时候回 B：

```text
资料不够 / 证据弱 / 缺外部参考 / 上下文不完整
```

什么时候回 C：

```text
理论浅 / 结构乱 / 方案不专业 / 边界不清
```

## 4. C端双状态

```text
C-research：研究版理论生成
C-execution：执行版接口包生成
```

C-research 负责：

```text
解释为什么、比较候选、标注假设、保留修正空间
```

C-execution 负责：

```text
明确任务边界、交付物、验证标准、端口分工、回退条件
```

判断口诀：

```text
还在讲为什么 → C-research
已经讲怎么交接和怎么验 → C-execution
```

## 5. Owner 表达引导模板

每轮输出后，系统必须告诉 Owner 下一步怎么说。

模板：

```text
你现在只需要判断：______

如果满意，可以回复：______

如果不满意，可以说：
- 方向不对：______
- 资料不够：______
- 理论/方案不够好：______

暂时不要展开：______

推荐下一句输入：______
```

防偏题规则：

```text
模糊但相关 → 系统帮你重写
新主题但可能相关 → 标记为旁支候选
明显偏离 → 先拉回当前循环
不满意但说不清 → 转成回 A / 回 B / 回 C
```

## 6. L1-L6 技能治理速查

```text
L1 端口主技能：A/B/C/D/E/F 的身份核
L2 通用辅助技能：跨端可复用能力
L3 外部桥接技能：外部 skill / 方法论桥接
L4 领域专用技能：meme、FCZ、样本、MQL5 等领域能力
L5 模板交付技能：brief、source pack、report、decision 等模板
L6 状态边界标记：activated / executed / referenced_only / deprecated 等
```

判断口诀：

```text
不是 skill 越多越强，
而是层级越清楚越稳。
```

## 7. 外部 skill 接入五分类

```text
ADOPT：直接接入
BRIDGE：桥接使用
MERGE：合并进现有能力
REJECT：拒绝接入
PATTERN_ONLY：只吸收模式
```

默认原则：

```text
不全量安装，不污染主线，先判断再吸收。
```

## 8. dbskill 状态链边界

可以做：

```text
dbs-decision：记录 F端决策事件
dbs-save：保存 repo 内 workflow snapshot
dbs-restore：从 repo 内 state/queue/snapshot 恢复
dbs-report：辅助 E/F 汇总报告
```

不能做：

```text
不能替代 git
不能替代索引
不能替代变更记录
不能替代 E端验证
不能替代 F端 Owner 决策
不能成为 repo 外真源
```

一句话：

```text
状态链降低断裂成本，但不定义事实真相。
```

## 9. 什么时候进入 D/E/F

只有满足：

```text
Owner 对 C-research 基本满意，
并同意 C 生成执行版接口包。
```

才能进入：

```text
C-execution → D落库 → E验证 → F决策
```

不能进入 D/E/F 的情况：

```text
方向还不确定
资料还不够
理论还没收敛
Owner 还不知道下一步怎么判断
```
