# 第一阶段技能编组：需求拷问与结构化推进 v0.1

> 分线：10_工程化交接 / 技能编组
> 目的：把 Superpowers、Matt Pocock Skills、dbs 技能组合成一个适合“第一阶段”的默认编组。
> 适用对象：非专业 AI/工程 Owner，用自然语言输入想法，希望先被拷问、再收束成可执行规格，而不是一上来就写代码。

---

## 1. 这份编组解决什么问题

你的问题不是“我有没有很多技能”，而是：

```text
自然语言想法
→ 被拷问清楚
→ 收束成可执行规格
→ 再进入 PRD / Issue / 执行
```

所以第一阶段不该以“冲突检查”为中心，而要以：

```text
把人话拷问成生产需要的技能组合
```

为中心。

## 3. 第一阶段默认编组

### A. 需求拷问层（主入口）

这一层负责把模糊想法拷问清楚。

**主技能：**

- `dbs-good-question`
- `grill-with-docs`
- `grill-me`

**辅助技能：**

- `brainstorming`
- `dbs-deconstruct`

**作用：**

- 把“我想要一个更好的流程”变成明确问题
- 把“想法”拆成可判断、可追问、可验证的子问题
- 先看文档和现有上下文，再追问

**推荐顺序：**

```text
dbs-good-question → brainstorming → grill-with-docs → grill-me → dbs-deconstruct
```

---

### B. 规格收束层

这一层负责把拷问后的内容压成可执行规格。

**主技能：**

- `to-prd`
- `writing-plans`

**辅助技能：**

- `brainstorming`（如果还需要多路线）
- `grill-with-docs`（如果术语还没定）

**作用：**

- 把对话收束成 PRD
- 把 PRD 写成更稳定的计划
- 让后续执行不再依赖聊天记忆

**推荐顺序：**

```text
to-prd → writing-plans
```

---

### C. 任务拆分层

这一层负责把规格拆成可领取的垂直切片。

**主技能：**

- `to-issues`
- `triage`

**作用：**

- 把 PRD 拆成 issue
- 给 issue 打状态、分层、分优先级
- 给后续执行准备入口

**推荐顺序：**

```text
to-issues → triage
```

---

### D. 执行前质量层

这一层负责在真的动手前，避免“先跑再说”。

**主技能：**

- `executing-plans`
- `using-git-worktrees`
- `dispatching-parallel-agents`

**辅助技能：**

- `verification-before-completion`
- `finishing-a-development-branch`

**作用：**

- 需要并行时先开 worktree 或子代理
- 执行前先定义验收
- 做完后先验证再收尾

## 6. 对你个人工作流画像的适配

你的画像不是“专业 AI 团队自建平台”，而是：

- 非专业 Owner
- 需要自然语言入口
- 需要先被拷问
- 需要把想法沉淀进仓库
- 需要先规格化，再执行
- 需要保留人类判断

所以第一阶段最适合的不是单一 skill，而是：

```text
A 端思考拷问 + 文档化收束 + PRD 过渡
```

也就是：

```text
dbs-good-question / grill-with-docs / grill-me
→ to-prd
→ to-issues
→ 再进入执行与验证
```
