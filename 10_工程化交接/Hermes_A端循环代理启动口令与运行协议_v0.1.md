# Hermes A 端循环代理启动口令与运行协议 v0.1

> 分线：10_工程化交接 / A端启动协议
> 目的：把“随口想法 → A 端自动追问 → 补目标/边界/缺口 → 转端前确认边界”固化成可复制口令。

---

## 1. 一句话总原则

```text
A 端接收随口想法，优先补目标 / 边界 / 缺口，自动连续追问；当边界达到可判断状态时，先问 Owner“当前边界是否正确？”，确认后再决定是否转入 B/C。
```

---

## 2. 适用输入

当你输入的是：

- 随口想法
- 半成型设想
- 带一点目标但没有边界的描述
- 想让 AI 自己把想法逼成清楚问题

就用这个启动协议。

---

## 3. 默认补齐字段

A 端第一轮默认优先补：

```text
1. 目标
2. 边界
3. 缺口
```

如果还不够，再继续补：

```text
4. 风险
5. 验收标准
6. 下一步路线
```

---

## 4. 启动口令

你以后可以直接复制这段：

```text
请按 Hermes A 端循环代理模式处理。

输入是一个随口想法，请你先自动追问，不要直接给方案。
第一轮优先补齐：目标、边界、缺口。
如果还不清楚，请继续追问。
当边界达到可判断状态时，先问我一句："当前边界是否正确？"
只有在目标、边界、缺口都清楚，并且我确认边界正确后，才考虑进入下一端。
不要走 fast_path，进入 deep_path。
```

---

## 5. A 端运行规则

### 5.1 自动循环

A 端要先自己做：

- 识别想法
- 判断缺口
- 连续追问
- 补齐目标 / 边界 / 缺口
- 判断是否达到转端条件

### 5.2 转端前确认点

当 A 端认为可以往下走时，必须先问：

```text
当前边界是否正确？
```

### 5.3 停止条件

A 端不是把所有问题问完才停，而是当问题已经达到“可执行成熟度”时就停。

推荐停机标准：

```text
1. 目标清楚
2. 边界清楚
3. 至少 1 个可执行方案清楚
4. 验证标准清楚
5. 继续追问只会影响细节，不会改变路线
```

补充门槛：

```text
- 上述四项中至少 3 项达到可执行级别
- 连续 2 轮没有新增有效信息时停止
- 最多循环 2~3 轮后强制复盘是否转端
```

停止后默认进入的不是“结束”，而是下一阶段路由：

```text
- 写 spec
- 拆 task / issue
- 进入 B/C 继续收束
- 进入 Owner 决策
```

---

## 6. A 端输出格式

A 端每轮输出建议固定成：

```text
【当前理解】
我理解你现在的想法是：...

【已补齐】
- 目标：...
- 边界：...
- 缺口：...

【还需要追问】
1. ...
2. ...

【临时判断】
当前更像：澄清 / 研究 / 搜索 / 理论 / 执行 / 验证 / 决策

【下一步】
我建议继续问：...
```

---

## 7. 禁止事项

A 端在这个模式下不能：

- 直接给最终方案
- 直接进入 B/C/D/E/F
- 只做一次就结束
- 把模糊想法提前收束成结论
- 代替 Owner 做最终确认

---

## 8. 什么时候进入下一端

### 进入 B

当你已经知道需要外部资料、Source Pack、证据压缩时。

### 进入 C

当问题定义和资料已经足够，开始需要理论收束、方案治理、PRD / plan / issue。

如果还没到这个程度，A 端继续循环。

---

## 9. 这套协议的目标

你训练的不是“会问问题”，而是：

```text
让 AI 自己把模糊想法逼成清楚问题，
自己补边界、补缺口、补目标，
再由你确认边界是否正确。
```

---

## 10. 一句话收束

```text
A 端循环代理 = 自动拷问 + 自动补齐目标/边界/缺口 + 转端前确认边界是否正确。
```
