# 最新工作流技能组合速查 v0.1

> 分线：10_工程化交接 / 运行速查
> 目的：记录当前最新工作流里 A/B/C/D/E/F 各阶段实际使用的技能组合，方便快速查看和后续同步。

---

## 1. A 端：需求拷问 / 收束

```text
brainstorming
+ dbs-good-question
+ grill-with-docs
+ grill-me
+ dbs-deconstruct
+ dbs-goal
+ dbs-decision
```

用途：

- 把模糊想法问清楚
- 压出目标、边界、缺口
- 判断是否进入 B/C
- 判断是否需要继续追问

---

## 2. B 端：Source Pack / 外部知识储备

```text
Agent Reach
+ dbs-learning
+ dbs-content-system
+ dbs-report
+ dbs-save
+ dbs-restore
```

用途：

- 读取外部来源
- 压成 Source Pack
- 保留来源关系与冲突点
- 为 C 端提供可吸收输入

---

## 3. C 端：理论收束 / 方案治理

```text
to-prd
+ writing-plans
+ to-issues
+ triage
+ improve-codebase-architecture
+ executing-plans
```

用途：

- 把 A/B 收束成可执行方案
- 拆任务、定边界、做治理
- 生成 D 端交接材料

---

## 4. D/E/F 端：执行、验证、决策

```text
executing-plans
+ using-git-worktrees
+ tdd
+ improve-codebase-architecture
+ finishing-a-development-branch
+ receiving-code-review
+ verification-before-completion
```

用途：

- 落地执行
- 保持 worktree 隔离
- 验证完成度
- 收尾分支
- 回收 review
- 做完成前验证

---

## 5. 当前工作流的核心判断

```text
A/B/C 是研究与收束段；
D/E/F 是执行、验证与决策段。
```

```text
A/B/C 收束到可执行成熟度后，
停机并转入 D/E/F，
不要继续在研究段空转。
```

---

## 6. 一句话结论

```text
最新工作流 = A 端拷问 + B 端证据压缩 + C 端方案治理 + D/E/F 执行验证决策。
```
