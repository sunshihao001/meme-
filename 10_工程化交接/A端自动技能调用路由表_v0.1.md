# A端自动技能调用路由表 v0.1

> 分线：10_工程化交接 / A端自动化
> 目的：把 A 端遇到不同信号时应该自动调用哪些技能、得到什么产出、下一步该怎么走，写成更明确的路由表。

---

## 1. 路由总原则

A 端不是固定问同一套问题，而是按信号自动选技能：

```text
输入信号 → 技能组合 → 新增信息 → 当前判断 → 下一步
```

A 端要做的是：

- 自动判断当前缺什么
- 自动挑最相关的技能
- 自动把新信息收回来
- 自动决定继续追问还是转端

---

## 2. 路由表：从模糊到清楚

### 2.1 输入很散、目标不明

优先技能：

```text
brainstorming
+ dbs-good-question
+ grill-me
+ grill-with-docs
+ dbs-deconstruct
+ dbs-goal
```

得到什么：

- 目标轮廓
- 问题定义
- 概念边界
- 需要继续问的关键缺口

下一步：

- 如果仍然散，继续 A 端追问
- 如果已经收束，转入 B 或 C

---

### 2.2 需要判断“这问题到底是什么类型”

优先技能：

```text
brainstorming
+ dbs-good-question
+ dbs-deconstruct
+ dbs-goal
+ dbs-decision
```

得到什么：

- 问题类型
- 目标类型
- 是否是决策题 / 研究题 / 执行题
- 哪些概念需要拆开

下一步：

- 选对应端口
- 或继续 A 端追问

---

### 2.3 需要补背景、补证据、补外部信息

优先技能：

```text
Agent Reach
+ dbs-learning
+ dbs-content-system
+ dbs-report
+ dbs-save
+ dbs-restore
```

得到什么：

- 外部来源
- Source Pack
- 证据强度
- 可吸收点 / 不可照搬点
- 缺口和冲突点

下一步：

- 如果证据够了，转 C
- 如果证据不够，继续搜或回 A 重新定义问题

---

### 2.4 需要把想法推进成方案

优先技能：

```text
to-prd
+ writing-plans
+ to-issues
+ triage
+ improve-codebase-architecture
+ executing-plans
```

得到什么：

- PRD / plan / issue
- 方案边界
- 任务拆分
- 可执行交接包

下一步：

- 进入 D/E/F
- 或回 A/B/C 重修

---

### 2.5 需要判断“该不该继续，还是该停”

优先技能：

```text
dbs-decision
+ dbs-slowisfast
+ dbs-report
+ dbs-goal
```

得到什么：

- 当前是否值得继续
- 是否已经到达可执行成熟度
- 是否只是细节优化
- 是否应该停机转端

下一步：

- 满足停机条件就转端
- 不满足就继续 A 端循环

---

## 3. A 端每轮自动执行顺序

1. 识别当前输入信号。
2. 判断当前缺口属于哪类。
3. 选最相关的技能组合。
4. 调用技能获取新增信息。
5. 归纳当前判断。
6. 继续追问或转端。

---

## 4. 停机条件

A 端循环在以下情况停止：

```text
1. 初步想法已经收束到你满意的地步。
2. 继续追问只会改细节，不会改路线。
3. 你明确说“满意”或“先停”。
```

---

## 5. 一句话版本

```text
A 端不是固定问，而是按信号自动选技能、自动补信息、自动判断是否继续，直到你满意。
```
