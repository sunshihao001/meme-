# dbskill 第三批接入执行卡片 v0.1

> 分线：10_工程化交接 / 外部技能接入  
> 来源：dbskill 外部仓库评估 + 接入映射表  
> 目标：把第三批“工作台迁移 / 对标 / 慢变量 / 执行阻塞” 4 个 skill 写成可执行、可验证、可兼容的落地卡片。

---

## 卡 01：dbs-agent-migration

### 目标

```text
把 dbs-agent-migration 接入多端工作台迁移流程。
```

### 归属

```text
端口：A/D/E
层级：L3/L5
动作：BRIDGE
```

### 与现有体系关系

```text
它用于统一 Claude / Codex / Grok / Hermes 的工作台规则、真源、命名与 bridge。
```

### 预期输出

```text
- 规则文件审计
- 真源识别
- 命名与 bridge 统一
- 三端工作台对齐说明
```

### 接入方式

```text
- 保留 alias
- 作为外部工作台迁移桥接层
- 与 A 端控制、D 端落库、E 端审查联动
```

### 验证方式

```text
- 是否能让多个 agent 工具链保持一致命名与规则
- 是否能减少工作台碎片化
```

### 风险

```text
- 如果没有明确真源，容易把“迁移”做成“再造一套乱的东西”
```

## 卡 02：dbs-benchmark

### 目标

```text
把 dbs-benchmark 接入对标分析与外部参照筛选流程。
```

### 归属

```text
端口：B/C/F
层级：L4
动作：BRIDGE / PATTERN
```

### 与现有体系关系

```text
它用于五重过滤，帮助找值得模仿的对标，排除噪音。
```

### 预期输出

```text
- 对标候选
- 过滤理由
- 对标维度
- 是否值得模仿的结论
```

### 接入方式

```text
- 先作为 pattern only 或 bridge
- 不直接替代 A 端判断
- 用于 B 端收敛和 F 端参考
```

### 验证方式

```text
- 是否能找到真正值得模仿的对象
- 是否能排除“看起来厉害但不适合”的噪音
```

### 风险

```text
- 容易被拿来做“看热闹式对标”，没有落地价值
```

## 卡 03：dbs-slowisfast

### 目标

```text
把 dbs-slowisfast 接入关键决策的慢变量识别流程。
```

### 归属

```text
端口：A/F
层级：L4/L2
动作：BRIDGE / PATTERN
```

### 与现有体系关系

```text
它帮助识别“走捷径”的风险，找到值得慢做的摩擦点。
```

### 预期输出

```text
- 慢变量识别
- 摩擦点识别
- 是否值得慢做的建议
```

### 接入方式

```text
- 先作为桥接/模式参考
- 与 A 端问题澄清和 F 端风险判断联动
```

### 验证方式

```text
- 是否能识别当前是否在走捷径
- 是否能指出真正值得花时间的环节
```

### 风险

```text
- 如果泛化过度，会变成“什么都能慢”而失去判别力
```

## 卡 04：dbs-action

### 目标

```text
把 dbs-action 接入执行阻塞诊断流程。
```

### 归属

```text
端口：A/F
层级：L4
动作：BRIDGE / PATTERN
```

### 与现有体系关系

```text
它用于识别“知道该做什么但做不动”的阻塞原因。
```

### 预期输出

```text
- 执行阻塞类型
- 阻塞原因
- 下一步动作建议
```

### 接入方式

```text
- 保留 alias
- 作为执行力诊断参考
- 不替代任务规划与 owner 决策
```

### 验证方式

```text
- 是否能识别真实阻塞，而不是只给鼓励
- 是否能转成可执行下一步
```

### 风险

```text
- 容易把复杂问题都解释成“执行力问题”，要避免过度心理化
```
