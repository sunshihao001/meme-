# 根系需求更正：从 MQL5 观察器回到 GMGN 自建交易系统 v0.1

> 分线：10_工程化交接 / 根系需求拷问 / Owner 纠偏  
> 状态：v0.1 / Owner 明确纠偏后形成的需求根定义  
> 生成时间：2026-06-15 06:33 PDT  
> 触发：Owner 指出最终目标不是 MQL5 平台交易，而是自建 meme 市场交易脚本；MQL5 只是量化交易体系设计参考；最终数据源提供者应是 GMGN skill/data。  
> 边界：本文件修正项目方向，不否定已完成的 MQL5 观察器原型价值；但禁止后续把 MQL5 当成最终交易执行环境。

---

## 1. Owner 原始纠偏

Owner 纠偏要点：

```text
最终目标是需要自建交易的脚本。
MQL5 平台不支持 meme 市场。
数据源不是使用 MQL5。
MQL5 只是制作量化交易体系设计中做参考。
最终交易体系里数据源提供者还是要 gmgn-skill 来做。
```

本项目必须立即修正的根错误：

```text
之前的推进容易把 MQL5 观察器当成最终产品主线。
正确理解应是：MQL5/MT5 = 研究与观察器原型参考层；GMGN + 自建脚本 = 最终 meme 市场交易体系方向。
```

---

## 2. 根系需求拷问结论

### 2.1 问题世界

```text
目标市场：meme 市场 / DEX / launchpad / 链上流动性环境。
关键数据：价格 K 线、交易量、swap、流动性、持仓集中度、dev 状态、smart money、风险字段。
关键执行：最终由自建脚本或后续交易系统承接，而不是 MT5/MQL5 EA。
关键验证：反证样本、GMGN 样本字段、状态机、消融实验、商业可执行性。
```

### 2.2 解法世界

```text
MQL5 世界：适合做结构观察器、图表参考、POC/AVWAP/状态机原型验证。
GMGN 世界：适合做 meme 市场数据源、token 发现、K线、trending、trenches、signal、smart money 与风险字段。
自建脚本世界：适合承接最终策略扫描、样本生成、规则执行、风控、提醒或交易动作。
```

### 2.3 根系目标重写

错误目标写法：

```text
把第一控盘区成本中枢回收模型做成 MQL5 EA / MT5 自动交易系统。
```

更正后的目标：

```text
把第一控盘区成本中枢回收模型沉淀成一个以 GMGN 数据为主要市场数据源、以自建脚本为最终执行/扫描/验证载体的 meme 市场结构识别与交易研究系统；MQL5 只作为观察器和量化体系设计参考，不作为最终数据源或执行平台。
```

## 3. 角色边界重排

| 层级 | 正确角色 | 当前状态 | 后续处理 |
|---|---|---|---|
| MQL5 / MT5 | 原型观察器、图表参考、结构可视化 | 已有 MVP，已编译安装 | 保留，但降级为 reference/prototype，不再当最终产品主线 |
| GMGN skill/data | 最终系统的数据源主候选 | 已有 GMGN 样本字段扩展与 skill 能力 | 提升为理论研究与工程路线主数据层 |
| 自建脚本 | 最终扫描/验证/交易体系载体 | 尚未形成完整 spec | 下一阶段需求拷问和理论研究主线 |
| 样本库 | 策略有效性与反证验证真源 | 已有 CSV/Markdown validator | 继续围绕 GMGN 数据字段补样本 |
| 状态机 | 策略结构解释与规则抽象 | 已有 STATE_* 和 legacy alias map | 需要映射到 GMGN K线/链上字段，而不是 MQL5 专属字段 |

## 4. 对已完成 MQL5 工作的正确归位

已完成的 MQL5 观察器工作不是废弃，而是归位为：

```text
1. 可视化参考原型。
2. POC / AVWAP / retracement / state label 的交互式设计样机。
3. CSV 字段试验场。
4. 状态机输出字段的参考实现。
5. 后续自建脚本可借鉴的算法草稿。
```

但它不是：

```text
1. 最终交易执行环境。
2. meme 市场数据源。
3. GMGN 数据替代品。
4. 全自动交易 EA 主线。
5. 项目商业化终点。
```
