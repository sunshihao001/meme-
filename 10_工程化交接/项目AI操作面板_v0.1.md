# 项目 AI 操作面板 v0.1

> 位置：`10_工程化交接/项目AI操作面板_v0.1.md`  
> 分线：AI 操作面板 / 项目短入口 / 防迷路入口  
> 状态：v0.1，一页式入口  
> 上游：`AI技术对标与自我定位地图_v0.1.md`  
> 用途：让 Owner 和 AI 每次进入项目时，先用一页确认“现在是什么、在哪一层、下一步只做什么”。

---

## 1. 项目一句话

```text
这是一个以 GMGN 数据源为终局方向、以 FCZ 第一控盘区成本中枢回收模型为结构核心的 meme 市场研究项目；当前阶段不是自动交易系统，而是结构观察器 + 样本标注 + 反证审计 + 后续自建交易系统需求源。
```

---

## 2. 当前四层定位

```text
Layer 1：业务项目层
当前主线：GMGN / meme / FCZ / 样本 / 结构策略 / 数据源 / 后续自建系统。

Layer 2：需求控制层
当前主线：用需求澄清端防止目标漂移，区分项目现实、方法论、执行边界。

Layer 3：工程执行层
当前主线：只允许小切片进入 spec / issue / handoff / validator / CI，不直接跳完整交易系统。

Layer 4：AI 产品化层
当前主线：把已有能力压缩成短入口、稳定闭环、可展示 demo，而不是继续堆方法名。
```

---

## 3. 当前最重要目标

```text
把混乱研究项目压缩成可操作、可验证、可复用的 AI 工作流入口。
```

当前不是：

```text
继续堆资料
继续堆方法论
直接写自动交易脚本
直接让 Codex 做完整系统
```

---

## 4. 当前禁止事项

```text
1. 禁止把 MQL5 当成最终交易执行平台。
2. 禁止把观察器说成完整量化体系。
3. 禁止把资料吸收笔记全部同权处理。
4. 禁止把方法论文件当成项目目标。
5. 禁止跳过样本、反证、回测、风控，直接进入自动下单。
6. 禁止把“写了文件”报告成“能力已经稳定运行”。
```

```text

2. 这个文件应该进入总纲、策略定义、资料吸收、样本标注、规则回测，还是工程交接？
3. 这是主线核心、主线辅助，还是参考资料？
4. 当前是 executed_once、workflow_integrated，还是只是 listed_only？
5. 下一步能否用一个可验证小切片完成？
```
