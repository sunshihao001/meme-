## 2. 旧知识中仍有价值的三块

### 2.1 MQL5 input 参数表

旧版 `MQL5观察指标需求文档_v0.1.md` 有较完整参数草案，当前 `MQL5观察指标MVP开发计划_v0.1.md` 已吸收模块范围，但参数表仍需显式回填候选。

可吸收为：

```text
absorbed_candidate：旧参数是 MQL5 MVP 的初始观察参数候选，不是已验证阈值。
```

建议优先恢复的参数族：

```text

InpATRPeriod
InpUseTickVolume
InpMinControlZoneBars
InpMaxControlZoneATR
InpMinZoneVolumeRatio
```

当前处理建议：

```text
保留为 absorbed_candidate，后续在 MQL5 MVP 计划中以“旧版 absorbed_candidate 参数表”形式显式列出。
```

### 2.2 legacy 状态 alias

旧状态：

```text
S0 无结构
S1 第一控盘区形成
S2 第一波拉升完成
S3 深回撤测试
S4 重新接近 / 站回 POC
S5 重新接受 AVWAP
S6 第一次拉起
S7 回撤不破关键成本锚
S8 二次拉升确认
S9 稳定小段观察区
SX 失效 / 诱多 / 尾仓派发风险
```

当前建议：

```text
superseded_with_alias
```

原因：当前真源已升级为更明确的 STATE_* 命名，但旧 S 状态对旧样本和旧文档仍有对照价值。

建议后续在状态机真源中显式增加：

```text
legacy_alias_map
```

映射逻辑：

```text
S1/S2/S3... -> 当前 STATE_* 主状态
SX -> STATE_FAILED_RECLAIM / STATE_DEATH_MARKET_RISK / STATE_NO_TRADE
```

---

### 2.3 volume_confirmation 总字段

旧版有：

```text
volume_confirmation = none / weak / valid / excessive
```

当前问题：

```text
当前字段拆散在 volume ratios / risk / impulse quality 里，没有独立总枚举。
```

本轮分类：

```text
needs_validation
```

建议：

```text
暂不恢复为核心字段；
在 FCZ_B/C/D 真实样本和后续 validator 校准后，再判断是否需要 aggregate_volume_confirmation。
```

## 3. 可直接用于后续文件的吸收结论

### 3.1 给 MQL5 MVP 计划的结论

```text
MQL5 MVP 计划需要恢复旧版 input 参数表，但只能标注为 absorbed_candidate / semantic_reference_only，不可写成已验证规则。
```

### 3.2 给状态机的结论

```text
旧 S 状态适合在未来状态机中保留 alias 映射，而不是直接废弃。
```

### 3.3 给样本和 validator 的结论

```text
volume_confirmation 可以作为后续样本验证的候选 aggregate 字段，但不应在当前主线强制恢复。
```
