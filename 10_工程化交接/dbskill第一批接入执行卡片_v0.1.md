# dbskill 第一批接入执行卡片 v0.1

> 分线：10_工程化交接 / 外部技能接入  
> 来源：dbskill 外部仓库评估 + 接入映射表  
> 目标：把第一批最值得接入的 4 个 skill，写成可执行、可验证、可兼容的落地卡片。

---

## 卡 01：dbs-good-question

### 目标

```text
把 dbs-good-question 接入到当前 A 端 Demand Grilling / 问题说明书流程。
```

### 归属

```text
端口：A
层级：L2
动作：MERGE
```

### 与现有体系关系

```text
它应该和 A 端的“需求澄清 / 概念解析 / 路由决策”统一口径。
```

### 预期输出

```text
- Demand Grilling Brief 的问题定义部分
- Agent 可解性判断
- 验证/边界/风险/停条件
```

### 接入方式

```text
- 保留 dbs-good-question 作为 alias
- 让 A 端主 skill 可以调用它的判断框架
- 不让它替代 A 端主身份
```

### 验证方式

```text
- 输入模糊问题时，是否能产出可批评、可验证的问题说明书
- 是否能明确“这个问题适不适合交给 Agent”
```

### 风险

```text
- 如果只接问题生成，不接路由和边界，还是会变成“问得更漂亮但没决策”
```

## 卡 02：dbs-deconstruct

### 目标

```text
把 dbs-deconstruct 接入 A 端的概念解析和假设拆解流程。
```

### 归属

```text
端口：A
层级：L2/L3
动作：MERGE
```

### 与现有体系关系

```text
它补强的是“概念拆解 / 术语清晰化 / 假设核查”。
```

### 预期输出

```text
- 概念拆解视图
- 术语定义表
- 假设列表
```

### 接入方式

```text
- 保留 alias
- 用于 A 端的概念消歧
- 与 dbs-good-question 配合使用
```

### 验证方式

```text
- 面对模糊概念时，是否能拆到足够原子
- 是否能显式识别未定义词汇和混淆词汇
```

### 风险

```text
- 如果拆解后没有回到行动层，容易变成纯哲学化讨论
```

## 卡 03：dbs-goal

### 目标

```text
把 dbs-goal 接入 A 端的目标清晰化 / 问题建模流程。
```

### 归属

```text
端口：A
层级：L2/L3
动作：MERGE
```

### 与现有体系关系

```text
它补强“愿望语法 → 可检查目标”的转换。
```

### 预期输出

```text
- 可检查目标描述
- 非目标列表
- 目标验收标准
```

### 接入方式

```text
- 保留 alias
- 让 A 端主 skill 在需求不清时调用它
- 与 Demand Grilling Brief 合并输出
```

### 验证方式

```text
- 是否能把“我想做…”变成“我要在什么条件下算完成”
- 是否能识别虚目标、空目标、口号型目标
```

### 风险

```text
- 如果只做目标描述，不做任务拆分，还是无法推进执行
```

## 卡 04：dbs-diagnosis

### 目标

```text
把 dbs-diagnosis 接入 A/C/F 之间的商业问题消解与诊断流程。
```

### 归属

```text
端口：A/C/F
层级：L3/L4
动作：BRIDGE
```

### 与现有体系关系

```text
它更像是一个“商业判断引擎”的桥接层，而不是单一技能。
```

### 预期输出

```text
- 问题是否成立
- 问题属于哪种类型
- 需要哪些事实核验
- 是否该转入决策或回填
```

### 接入方式

```text
- 作为外部桥接层保留原方法论
- 不直接替代 F 端 owner 决策
- 和 A 端的需求澄清共用输入
```

### 验证方式

```text
- 面对商业问题时，是否能先消解掉错题
- 是否能明确问题类型：信息、情绪、复杂判断、事实错误、信息不足
```

### 风险

```text
- dbs-diagnosis 的商业哲学很强，不能直接覆盖你的业务 owner 规则
- 它应该是桥接意见，不是最终权限
```

---

## 第一批执行顺序

```text
1. dbs-good-question
2. dbs-deconstruct
3. dbs-goal
4. dbs-diagnosis
```

原因：

```text
这四个最直接补“判断推理能力”。
```
