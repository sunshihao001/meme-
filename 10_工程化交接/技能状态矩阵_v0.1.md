# 技能状态矩阵 v0.1

> 分线：10_工程化交接 / 技能资产盘点  
> 状态：v0.1 / 更新优先级矩阵  
> 目的：把现有技能按“已稳定 / 待拆分 / 待合并 / 待桥接 / 待弃用”做一次可执行分组，方便后续逐项更新。

---

## 1. 读法

```text
已稳定   = 结构和命名都可暂时保持
待拆分   = 一个技能承担了太多职责
待合并   = 多个技能重复表达同一能力
待桥接   = 适合先做外部接入或 pattern only
待弃用   = 长期不用、混乱或已被替代
```

## 6. 待弃用

当前暂无明确已弃用技能，但以下情况一旦出现就应该进入弃用池：

```text
- 旧名长期只作为历史兼容
- 重复模板长期不再被引用
- 某个端口职责被新主技能完全覆盖
- 外部桥接试验后没有重复价值
```

候选监控项：

```text
- 旧版字段状态机吸收相关命名
- 过时的 prompt 版本别名
- 任何只在单一旧文档里出现的历史名
```

---

## 7. 优先级建议

```text
第一优先：待拆分
第二优先：待合并
第三优先：待桥接
第四优先：待弃用
```

原因：

```text
拆分和合并会直接改善结构清晰度；
桥接解决外部方法接入；
弃用是最后一步。
```
