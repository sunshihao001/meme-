## 2. 第一控盘区

### 当前工作定义

第一控盘区是 meme / 高波动标的早期阶段中，主要资金第一次形成明显成本共识的位置区间。该区间通常表现为成交或停留密集、波动相对收敛、后续能从该区产生第一波有效拉升，并且在深洗后仍可能作为成本锚被重新测试和回收。

更短定义：

```text
第一控盘区 = 第一波有效拉升前，最能代表初始控盘成本和早期价值共识的成交 / 停留密集区。
```

### 可观察特征

```text
1. 早期出现，而不是第一波大幅拉升后才出现。
2. 有明显价格停留或成交密集。
3. 区间内能识别 POC / HVN / 价值区域。
4. 区间波动相对收敛，不是单纯暴涨暴跌。
5. 后续从该区产生过第一波有效拉升。
6. 深洗后该区仍可能产生结构反应。
7. 不能明显是高位派发区。
```

### 非该概念的情况

```text
随便画的低位箱体
价格第一次横盘的地方
任意成交密集区
普通支撑位
单根放量 K 线
第一波拉升后的高位派发区
只因为后来上涨而事后框出的低点区域
```

### 常见误判

```text
1. 把普通横盘当控盘。
2. 把第一波高位派发当第一控盘区。
3. 先看到大涨，再回头寻找任意低位框。
4. 忽视成交 / 停留密集，只按价格低点定义。
5. 多个候选区间之间没有优先级。
```

### 待 Owner 决策点

```text
1. 是否要求第一控盘区必须在第一波有效拉升前形成？
2. 是否允许多个候选第一控盘区？
3. 多个候选区间的优先级如何排序？
4. 是否要求 POC / HVN / Value Area 支持？
5. 第一控盘区最小持续时间或最小 K 线数是多少？
```

### 后续是否可量化

可量化，但需要分层：

```text
价格区间宽度
区间持续 K 线数
区间成交量占比
POC 是否位于区间内部
后续第一波拉升幅度
深洗后是否重新测试该区
```

## 3. 成本中枢

### 当前工作定义

成本中枢是第一控盘区内部或相关结构中，最能代表主要资金平均成本、成交密集、市场价值接受的位置或区间。

它可以由 POC、Value Area、HVN、AVWAP、放量 K 区间、结构震荡区间等方式表达，但不能事后任意指定。

### 可观察特征

```text
1. 与第一控盘区存在明确对应关系。
2. 价格或成交在该区域有明显集中。
3. 后续价格靠近该区域时产生支撑 / 压力 / 接受 / 拒绝反应。
4. 可通过事前规则确定，而不是事后为了适配结果移动。
```

### 非该概念的情况

```text
任意均线
任意支撑线
事后移动的箱体中线
为了让回测好看而重新选择的成本锚
没有成交或停留依据的主观价格位
```

### 常见误判

```text
1. 把成本中枢等同于单个买点。
2. POC、AVWAP、Value Area 冲突时任意选择有利者。
3. 失败后更换成本锚点解释样本。
4. 忽略数据粒度不足导致的 POC 失真。
```

### 待 Owner 决策点

```text
1. 成本中枢默认优先采用 POC、Value Area 还是 AVWAP？
2. 不同表达冲突时如何排序？
3. 数据粒度不够时是否允许结构区间替代？
4. 是否允许多个成本中枢共同参与判断？
```

### 后续是否可量化

可量化：

```text
POC level
VAH / VAL
HVN 区域
AVWAP 锚点与数值
成本中枢上下沿
价格相对成本中枢偏离率
```

## 4. 深洗

### 当前工作定义

深洗是指价格在第一控盘区或成本中枢形成并完成第一波拉升后，向下跌破或刺穿关键成本区 / 前低 / 流动性区域，以清理浮筹或扫止损为主要结构表现，随后仍存在重新接受成本中枢可能的阶段。

### 可观察特征

```text
1. 发生在第一控盘区和第一波拉升之后。
2. 价格明显跌破成本锚或区间下沿。
3. 可能伴随 liquidity sweep / shakeout。
4. 下跌后应出现回收尝试或资金回流迹象。
5. 不能无限期等待，必须有时间和幅度边界。
```

### 非该概念的情况

```text
高位出货后的持续下跌
没有第一控盘区基础的随机暴跌
跌破后长期无法收回的死亡盘
流动性枯竭导致的自然下跌
项目基本面或叙事彻底崩塌
```

### 常见误判

```text
1. 把所有大跌都解释为洗盘。
2. 跌破后迟迟不收回仍说洗盘没结束。
3. 忽略成交、流动性、叙事衰退。
4. 用后续偶然反弹证明深洗有效。
```

### 待 Owner 决策点

```text
1. 深洗最大允许跌幅是多少？
2. 深洗最大允许持续时间是多少？
3. 是否要求快速收回成本区？
4. 是否要求扫前低后回收？
5. 多深之后必须从深洗降级为死亡盘？
```

### 后续是否可量化

可量化：

```text
跌破幅度
跌破持续 K 线数
sweep 深度
收回所需 K 线数
成交量变化
重新进入 Value Area 的速度
```

## 5. 死亡盘

### 当前工作定义

死亡盘是指看似经历控盘、深洗或回收，但实际上资金持续流出、流动性衰减、高点下降、成本区从支撑转为压力，后续无法形成有效二次启动的结构。

### 可观察特征

```text
1. 反复回收成本锚失败。
2. 高点持续下降。
3. POC / AVWAP 从支撑转为压力。
4. 成交或流动性持续衰减。
5. 重新接受后不能推进，或推进后迅速砸回。
6. 叙事热度、盘口深度或资金回流明显不足。
```

### 非该概念的情况

```text
短期深洗后快速回收
低流动性噪音但结构仍完整
正常回撤不破成本锚
二次启动前的健康缩量整理
```

### 常见误判

```text
1. 把死亡盘硬解释为更深级别洗盘。
2. 忽略高点下降和资金不足。
3. 只看价格回到成本区，不看能否接受和推进。
4. 反复失败后仍继续补规则。
```

### 待 Owner 决策点

```text
1. 死亡盘的硬失效条件是什么？
2. 反复回收失败多少次后判定死亡盘？
3. 流动性衰减是否作为核心判定？
4. 死亡盘是否直接 no_trade？
```

### 后续是否可量化

可量化：

```text
回收失败次数
lower high 数量
成本锚下方停留时间
成交量 / 流动性衰减率
二次启动失败后的回撤幅度
```

## 6. 重新接受

### 当前工作定义

重新接受是指深洗后，价格重新回到第一控盘区价值区域或 POC / AVWAP 成本锚上方，并在后续一段结构中没有迅速跌回；第一次拉起后的回撤不再有效跌破关键成本锚，随后出现二次推进确认。

更短定义：

```text
重新接受 = 站回成本锚 + 停留 / 换手 + 回撤不破 + 二次推进。
```

### 可观察特征

```text
1. 发生在深洗后。
2. 价格重新进入 Value Area 或站回 POC / AVWAP。
3. 不是单根 K 线触碰，而是有停留或换手。
4. 第一次拉起后的回撤不再有效跌破成本锚。
5. 后续出现二次推进或结构确认。
```

### 非该概念的情况

```text
单次触碰 POC / AVWAP
短暂站回后快速跌回
没有后续推进的普通反抽
成本锚已经变成压力的失败回收
```

### 常见误判

```text
1. 把触碰当接受。
2. 把一根 K 线站回当接受。
3. 不等回撤确认就判定成功。
4. 忽略站回后成交和停留不足。
```

### 待 Owner 决策点

```text
1. 重新接受最少需要几根 K 线确认？
2. 是否必须回撤不破？
3. 是否必须二次推进？
4. POC 与 AVWAP 冲突时以谁为准？
```

### 后续是否可量化

可量化：

```text
站回 K 线数
成本锚上方停留比例
回撤是否跌破 POC / AVWAP
二次推进幅度
重新跌回成本区的速度
```

## 7. 普通反抽

### 当前工作定义

普通反抽是指价格下跌或跌破成本区后出现的短期反弹，但反弹未能形成稳定重新接受、未能回撤不破、未能二次推进，通常只代表短期修复或诱多风险。

### 可观察特征

```text
1. 价格短期反弹到成本锚附近。
2. 可能触碰或短暂站回。
3. 缺少停留、换手、回撤不破、二次推进。
4. 很容易再次跌回成本锚下方。
```

### 非该概念的情况

```text
强接受结构
回撤不破后的二次启动
有效成本区支撑确认
```

### 常见误判

```text
1. 把普通反抽当重新接受。
2. 只因涨幅较大就认为结构有效。
3. 忽略反抽后的快速回落。
```

### 待 Owner 决策点

```text
1. 普通反抽与重新接受的最小区分条件是什么？
2. 是否把无回撤确认的站回全部归为普通反抽？
3. 普通反抽是否只能 observe_only？
```

### 后续是否可量化

可量化：

```text
触碰后回落 K 线数
站回持续时间
二次推进是否出现
反抽成交量质量
```

## 8. 二次启动

### 当前工作定义

二次启动是指深洗后价格重新接受成本锚，并经过第一次拉起与回撤确认后，再次向上推进，形成比普通反抽更强的结构确认。

### 可观察特征

```text
1. 发生在重新接受之后。
2. 通常有第一次拉起和回撤不破。
3. 再次上攻时突破局部高点或形成 BOS。
4. 推进过程中成交、波动或结构质量改善。
5. 不应依赖未来大涨结果反推。
```

### 非该概念的情况

```text
单根冲高回落
没有回撤确认的直接拉升
突破后快速跌回的诱多
弱反抽后的随机上涨
```

### 常见误判

```text
1. 把所有第二次上涨都叫二次启动。
2. 忽略突破后是否站稳。
3. 忽略执行成本和滑点。
4. 用未来涨幅证明当时已启动。
```

### 待 Owner 决策点

```text
1. 二次启动是否必须突破前高？
2. 是否必须出现 BOS？
3. 是否必须放量？
4. 二次启动失败后的判定边界是什么？
```

### 后续是否可量化

可量化：

```text
突破幅度
突破后站稳 K 线数
二次推进成交量
回踩不破情况
突破失败回撤幅度
```

## 9. 诱多

### 当前工作定义

诱多是指价格看似完成站回、突破或二次启动，但随后快速跌回关键成本锚或区间内部，使追入者承受明显亏损的失败结构。

### 可观察特征

```text
1. 常发生在 POC / AVWAP 站回、区间突破或二次启动附近。
2. 突破后缺少停留和换手。
3. 快速跌回成本锚下方。
4. 高位成交放大但无法推进。
5. 后续进入失败回收或死亡盘风险。
```

### 非该概念的情况

```text
健康突破后的正常回踩
缩量回撤不破
流动性充足且结构继续推进
```

### 常见误判

```text
1. 把所有突破失败都叫诱多，忽略普通波动。
2. 在事后才定义诱多，无法事前识别。
3. 不记录诱多样本，导致模型只保留成功突破。
```

### 待 Owner 决策点

```text
1. 诱多的时间边界是多少？
2. 突破后跌回多少算失败？
3. 是否要求成交量异常？
4. 诱多是否直接进入反证样本库 B / D 类？
```

### 后续是否可量化

可量化：

```text
突破后回落 K 线数
跌回成本锚幅度
突破成交量与后续成交量对比
追入点最大不利波动
```

## 10. 失败样本

### 当前工作定义

失败样本是指在事前规则触发时符合模型核心条件，但后续没有产生预期二次启动、收益不足、跌破失效、进入死亡盘，或交易上不可执行的样本。

### 可观察特征

```text
1. 触发时必须按事前定义判断，而不是事后重写规则。
2. 后续结果未达到最低成功标准。
3. 必须记录失败类型和失败原因。
4. 不得因为失败而剔除样本。
```

### 非该概念的情况

```text
触发前已明显违反规则的样本
数据错误样本
无法确认关键字段的 UNKNOWN 样本
```

### 常见误判

```text
1. 失败后说它本来就不符合。
2. 用新增规则事后排除。
3. 不记录亏损或不可执行案例。
```

### 待 Owner 决策点

```text
1. 最低成功标准是什么？
2. 收益不足算不算失败？
3. 横盘耗时过长算不算失败？
4. 交易不可执行是否直接归入失败？
```

### 后续是否可量化

可量化：

```text
净 R
最大有利 / 不利波动
失败类型
触发后 N 根 K 线结果
是否跌破止损
```

## 11. 可执行买点

### 当前工作定义

可执行买点是指在事前可观察、可下单、滑点和止损空间可接受、且不依赖未来 K 线确认的入场位置。

### 可观察特征

```text
1. 信号在当时可见。
2. 价格有可成交流动性。
3. 止损位置明确。
4. 滑点、盘口、涨停 / 跌停、池子深度等执行约束可接受。
5. 不需要未来确认才能知道当时可买。
```

### 非该概念的情况

```text
未来大涨后回看最优点
涨停板后无法成交的买点
盘口太薄导致滑点不可控
止损过远导致 R 不合理
必须等待未来 K 线后才能确认的点
```

### 常见误判

```text
1. 图形成立就认为可以交易。
2. 忽略滑点、流动性和涨停限制。
3. 用未来确认反推当时买点。
4. 不计算止损距离。
```

### 待 Owner 决策点

```text
1. 最大允许止损 R 或百分比是多少？
2. 最大允许滑点是多少？
3. 哪些市场执行约束必须纳入？
4. 不可执行但图形成立的样本如何归类？
```

### 后续是否可量化

可量化：

```text
entry_price
stop_price
slippage_estimate
liquidity_depth
max_adverse_excursion_R
是否当时可观察
```

## 12. 反证样本

### 当前工作定义

反证样本是指能够挑战、削弱或推翻当前模型某个核心假设的样本，包括模型符合但失败、模型不符合但大涨、看似符合但死亡盘、图形成立但不可执行等类型。

### 可观察特征

```text
1. 明确挑战一个或多个核心假设。
2. 能被归入 A/B/C/D/E/F 等样本类别。
3. 记录事前规则是否会触发。
4. 记录是否发生边界移动或锚点更换。
```

### 非该概念的情况

```text
普通成功样本且不挑战模型
数据缺失无法判断的样本
只用于展示漂亮图形的案例
```

### 常见误判

```text
1. 只收集成功样本，不收集反证。
2. 把反证样本解释为特殊情况。
3. 不区分规则缺陷、执行缺陷、市场环境缺陷。
```

### 待 Owner 决策点

```text
1. 哪些反证足以让模型降级？
2. 反证样本最低收集数量是多少？
3. 反证结论是否优先于理论深化？
```

### 后续是否可量化

可量化：

```text
sample_class
sample_subclass
challenged_hypothesis
would_rule_have_triggered_ex_ante
was_boundary_moved_after_outcome
net_realized_R_after_cost
```

## 13. 后视偏差

### 当前工作定义

后视偏差是指使用未来已经发生的结果，反过来选择成本中枢、AVWAP 锚点、第一控盘区边界、过滤条件或样本分类，使模型看起来更有效。

### 可观察特征

```text
1. 失败后更换锚点。
2. 只在成功样本上找到完美区间。
3. 用未来高点或低点选择 AVWAP 锚点。
4. 新规则只解释过去，不能事前执行。
```

### 非该概念的情况

```text
事前明确定义的规则
独立样本外验证
固定版本规则下的失败记录
```

### 常见误判

```text
1. 认为画线合理就不是后视。
2. 不记录规则版本。
3. 忽略锚点选择时点。
```

```text
2. 样本标注是否必须记录 rule_version？
3. 失败后调整规则需要什么审计流程？
```

## 14. 消融实验

### 当前工作定义

消融实验是指在固定样本库和评估标准下，逐步移除模型中的某个条件，观察模型表现是否明显下降，以判断该条件是否真正贡献策略优势。

### 可观察特征

```text
1. 固定样本集。
2. 固定评价指标。
3. 每次只移除或替换一个条件。
4. 比较完整模型、简化模型、基准策略表现。
```

### 非该概念的情况

```text
随意改规则后观察几个案例
只在成功样本上比较
用主观解释替代统计结果
```

### 常见误判

```text
1. 条件越多越好。
2. 不和简单策略对比。
3. 不记录被移除条件对失败样本的影响。
```

### 待 Owner 决策点

```text
1. 首批消融条件有哪些？
2. 评价指标采用胜率、净 R、最大回撤还是可执行性？
3. 哪个结果说明某条件应降级为辅助叙事？
```

### 后续是否可量化

可量化：

```text
full_model_result
without_deep_washout_result
without_reclaim_result
without_volume_filter_result
baseline_breakout_result
performance_delta
```
