# 结构风险评分模型 v0.1

> 分线：D风险管理 / H指标规则 / G样本标注  
> 版本：v0.1  
> 状态：初版 / 概念评分框架 / 暂不用于自动交易  
> 目的：把前面“深洗、死亡盘、成本锚回收、诱多反抽、二次启动”等语义转成可审查的结构评分框架。  

---

## 1. 本文要解决的问题

前面已经定义了：

```text
第一控盘区
深洗 vs 死亡盘
POC / AVWAP 重新接受
诱多反抽 vs 二次启动
```

现在需要把这些概念变成一个统一审查框架：

```text
当前结构到底是 no_trade、observe_only，还是二次启动候选？
```

注意：本评分模型当前不是自动交易信号。

它只用于：

```text
结构审查
样本标注
多机器人分工
后续指标需求
人工复盘一致性
```

---

## 2. 总体评分思想

本模型分成两类分数：

```text
风险分：越高越危险
确认分：越高越接近二次启动候选
```

风险分包括：

```text
death_market_score
bull_trap_score
failed_reclaim_score
distribution_score
liquidity_trap_score
narrative_decay_score
```

确认分包括：

```text
first_control_zone_score
cost_anchor_acceptance_score
deep_washout_reclaim_score
second_start_score
structure_confirmation_score
```

---

## 3. 输出分级

最终输出不是买卖，而是：

```text
no_trade
observe_only
candidate_for_deeper_review
second_start_candidate
```

### 3.1 no_trade

含义：

```text
结构风险高，死亡盘/诱多/派发/回收失败明显，不应纳入当前策略研究。
```

典型情况：

```text
death_market_score 高
failed_reclaim_score 高
bull_trap_score 高
second_start_score 低
```

---

### 3.2 observe_only

含义：

```text
结构尚未失效，但确认不足，只观察不研究上车。
```

典型情况：

```text
第一控盘区可能存在；
深洗可能完成；
但 POC/AVWAP 尚未重新接受；
或第一次拉起还未回撤确认。
```

---

### 3.3 candidate_for_deeper_review

含义：

```text
结构进入候选，需要人工或多机器人进一步审查。
```

典型情况：

```text
第一控盘区明确；
深洗后重新回收成本区；
但二次启动确认还不完整。
```

---

### 3.4 second_start_candidate

含义：

```text
成本锚重新接受、回撤不破、二次拉升确认均较充分，可进入保守研究区。
```

注意：

```text
这仍不是自动买入信号，只是策略研究区。
```

---

## 4. 风险评分模块

### 4.1 death_market_score

衡量是否已经从深洗变成死亡盘。

加分条件：

```text
POC长期变压力
AVWAP长期在价格上方
反抽持续被卖
成交和热度持续枯竭
价格长期低于VAL
多次回收失败
```

高分含义：

```text
不应继续按洗盘/二次启动观察。
```

---

### 4.2 bull_trap_score

衡量是否是诱多反抽。

加分条件：

```text
站回成本锚后快速跌回
第一次拉起放量滞涨
回撤有效跌破成本锚
第二次拉升失败
上影线多
主动卖压强
```

高分含义：

```text
反抽可能是吸引买盘，而不是二次启动。
```

---

### 4.3 failed_reclaim_score

衡量 POC / AVWAP / Value Area 是否回收失败。

加分条件：

```text
只触碰不站回
站回但不能停留
站回后快速跌回
POC/AVWAP变压力
回撤跌破后无法收回
```

---

### 4.4 distribution_score

衡量第一波或反抽过程中是否存在派发。

加分条件：

```text
第一波高位巨量滞涨
连续上影线
反抽放量不涨
拉升中卖压明显
二次拉升阶段供应持续出现
```

---

### 4.5 liquidity_trap_score

衡量扫流动性后是否没有形成有效回收。

加分条件：

```text
扫低点后不能收回
扫完后没有BOS或higher low
扫完后反而继续下跌
假突破后快速失败
```

---

### 4.6 narrative_decay_score

meme 专属风险。

加分条件：

```text
成交活跃度下降
社区热度下降
叙事热度消失
池子/流动性变薄
大钱包持续卖出
```

---

## 5. 确认评分模块

### 5.1 first_control_zone_score

衡量第一控盘区是否真实存在。

加分条件：

```text
早期形成
成交/停留密集
存在POC/HVN/Value Area
后续第一波有效拉升验证
不是高位派发区
```

---

### 5.2 deep_washout_reclaim_score

衡量深洗是否完成并出现回收。

加分条件：

```text
有深度回撤/扫流动性
跌破后能收回
不再持续创新低
重新接近第一控盘区
卖压减弱
```

---

### 5.3 cost_anchor_acceptance_score

衡量 POC / AVWAP 是否被重新接受。

加分条件：

```text
重新进入 Value Area
站回 POC
站回 AVWAP
站回后停留/换手
回撤不破成本锚
成本锚从压力转支撑
```

---

### 5.4 second_start_score

衡量是否进入二次启动候选。

加分条件：

```text
第一次拉起后回撤不破
形成 higher low
二次拉升突破或接近第一次拉起高点
BOS / SOS / LPS continuation 出现
成交/流动性恢复
```

---

### 5.5 structure_confirmation_score

综合结构确认。

加分条件：

```text
Sweep -> Reclaim -> Confirm 完整
成本锚支撑有效
二次拉升确认
诱多风险下降
死亡盘风险下降
```

---

## 6. 推荐判定逻辑 v0.1

### no_trade

```text
death_market_score 高
或 bull_trap_score 高
或 failed_reclaim_score 高
且 second_start_score 低
```

---

### observe_only

```text
first_control_zone_score 中等以上
但 cost_anchor_acceptance_score 低
或 second_start_score 低
```

---

### candidate_for_deeper_review

```text
first_control_zone_score 高
cost_anchor_acceptance_score 中等以上
bull_trap_score 不高
death_market_score 不高
但 second_start_score 尚未强确认
```

---

### second_start_candidate

```text
first_control_zone_score 高
cost_anchor_acceptance_score 高
second_start_score 高
bull_trap_score 低
failed_reclaim_score 低
death_market_score 低
```

---

## 7. 本版结论

```text
结构风险评分模型不是为了自动买入，而是为了把“该不该继续研究”标准化。
```

最重要原则：

```text
高风险分优先否决；
确认分只负责进入候选；
任何死亡盘/诱多/回收失败高分，都可以压制二次启动候选。
```
