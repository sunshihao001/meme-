# 第一控盘区成本中枢回收模型：MQL5资料映射与准备清单

## 一、策略在MQL5资料体系中的分类

该策略最适合归入以下复合类型：

1. **市场轮廓 / 成交量分布 / 价值区域策略**：用于识别第一控盘区、POC、价值区域 VAH/VAL、价格接受/拒绝。
2. **VWAP / AVWAP 成本锚策略**：用于确认价格是否重新接受成交量加权成本线；AVWAP需在本地指标中增加“锚点时间/锚点K线”参数。
3. **斐波那契深回撤过滤策略**：0.618–0.786 只作为深回撤过滤，不作为单独买点。
4. **订单块 / 智能资金 / 机构成本区策略**：用于辅助判断第一控盘区是否具备机构承接或派发属性。
5. **BOS / CHoCH / 支撑阻力结构确认策略**：用于二次拉升确认、反抽失败、结构破坏等判定。
6. **成交量趋势确认策略**：用于过滤“无量反抽”“高位继续派发”“二次拉升无买盘”。

因此，建议命名为：

```text
市场轮廓POC + AVWAP成本锚 + 深回撤后二次结构确认模型
```

更适合MQL5自动化研究的分类：

```text
Volume Profile / Market Profile Acceptance Reclaim Strategy
VWAP/AVWAP Cost Anchor Reclaim Strategy
BOS-confirmed Pullback Continuation Strategy
```

## 二、该策略需要准备的详细信息

### 1. 行情K线数据
- symbol
- timeframe
- time/open/high/low/close
- tick_volume 或真实 volume
- spread
- bar index

### 2. 第一控盘区/POC信息
- profile_start_time / profile_end_time
- 价格分箱 tick_size 或 point_step
- 每个价格桶成交量/停留次数
- POC_price
- VAH / VAL
- value_area_ratio，默认70%
- profile_width / histogram
- 是否形成明确单峰/多峰

### 3. 第一波结构信息
- impulse_start_time / impulse_start_price
- impulse_high_time / impulse_high_price
- 第一波涨幅、持续K线数
- 拉升成交量是否放大
- 是否出现上影线、巨量滞涨、反抽卖压

### 4. 深回撤信息
- retracement_low_time / retracement_low_price
- retracement_ratio = (high - pullback_low) / (high - impulse_start)
- 是否处于0.618–0.786
- 是否跌破VAL/POC后快速收回
- 回撤期间成交量/主动卖压变化

### 5. POC / AVWAP重新接受信息
- price_cross_poc_time
- close_above_poc_bars 连续收盘站上POC数量
- poc_retest_success 是否回踩不破POC
- avwap_anchor_time：建议锚定第一控盘区起点、POC形成区或第一波启动点
- avwap_value
- close_above_avwap_bars
- VWAP/AVWAP斜率

### 6. 二次确认信息
- first_bounce_high
- retest_low_after_first_bounce
- 是否不破 POC/AVWAP/VAL
- second_push_breaks_first_bounce_high
- second_push_volume_confirmed
- BOS/CHoCH状态

### 7. 风险否决信息
- 第一波派发强度 distribution_score
- 反抽期间卖压 sell_pressure_score
- POC由支撑转压力次数
- AVWAP失守次数
- 二次拉升失败原因
- 最大可接受止损距离

## 三、建议机器人/模块分工

- 结构机器人：识别第一波、回撤、BOS、二次拉升。
- 成本锚机器人：计算POC、VAH/VAL、VWAP/AVWAP、接受/拒绝。
- 资金流机器人：分析成交量、放量突破、缩量回撤、派发风险。
- 风控机器人：基于失效条件输出 observe_only / small_size / no_trade。
