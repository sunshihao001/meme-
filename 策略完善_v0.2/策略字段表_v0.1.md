# 第一控盘区成本中枢回收模型：策略字段表 v0.1

> 目的：定义后续指标、样本标注、机器人审查、回测所需的数据字段。  
> 当前版本：字段草案，不代表最终阈值。  

---

## 1. 字段分组

```text
A. 基础行情字段
B. 第一控盘区字段
C. Market Profile / Volume Profile 字段
D. 第一波拉升字段
E. 深回撤字段
F. POC 回收字段
G. AVWAP 字段
H. 第一次拉起字段
I. 回撤不破字段
J. 二次拉升确认字段
K. 成交量确认字段
L. 风险否决字段
M. 最终分类字段
```

---

## 2. A. 基础行情字段

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| symbol | string | 是 | 交易品种 |
| timeframe | enum | 是 | 周期，如 M5/M15/H1/H4/D1 |
| bar_time | datetime | 是 | K线时间 |
| open | double | 是 | 开盘价 |
| high | double | 是 | 最高价 |
| low | double | 是 | 最低价 |
| close | double | 是 | 收盘价 |
| tick_volume | long | 是 | tick成交量 |
| real_volume | long | 否 | 真实成交量，若经纪商提供 |
| spread | double | 否 | 点差 |
| atr | double | 建议 | ATR，用于归一化区间宽度 |

---

## 3. B. 第一控盘区字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| control_zone_detected | bool | 是否识别到第一控盘区 |
| control_zone_start_time | datetime | 控盘区开始时间 |
| control_zone_end_time | datetime | 控盘区结束时间 |
| control_zone_high | double | 控盘区上沿 |
| control_zone_low | double | 控盘区下沿 |
| control_zone_mid | double | 控盘区中位价 |
| control_zone_bars | int | 控盘区持续K线数 |
| control_zone_range | double | 控盘区价格高度 |
| control_zone_range_atr | double | 控盘区高度 / ATR |
| control_zone_volume | long | 控盘区总成交量 |
| control_zone_volume_ratio | double | 控盘区成交量占观察窗口比例 |
| control_zone_quality | enum | weak / medium / strong |

建议初始阈值：

```text
control_zone_bars >= 10
control_zone_range_atr <= 2.5
control_zone_volume_ratio >= 0.30
```

---

## 4. C. Profile 字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| profile_bin_size | double | 价格分箱大小 |
| profile_type | enum | time_profile / volume_profile / hybrid |
| poc_price | double | 控制点价格 |
| poc_volume | long | POC价格桶成交量/停留次数 |
| vah_price | double | 价值区上沿 |
| val_price | double | 价值区下沿 |
| value_area_ratio | double | 默认0.70 |
| hvn_count | int | 高成交量节点数量 |
| lvn_count | int | 低成交量节点数量 |
| distribution_shape | enum | single_peak / double_distribution / flat / unclear |
| poc_position_in_zone | enum | upper / middle / lower / outside |

---

## 5. D. 第一波拉升字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| impulse_detected | bool | 是否有第一波拉升 |
| impulse_start_time | datetime | 第一波起点时间 |
| impulse_start_price | double | 第一波起点价格 |
| impulse_high_time | datetime | 第一波高点时间 |
| impulse_high_price | double | 第一波高点价格 |
| impulse_range | double | 第一波涨幅 |
| impulse_range_atr | double | 第一波涨幅 / ATR |
| impulse_bars | int | 第一波持续K线数 |
| impulse_volume | long | 第一波成交量 |
| impulse_volume_expansion | double | 相对均量放大倍数 |
| impulse_quality | enum | weak / normal / strong / climax |

建议初始阈值：

```text
impulse_range >= 1.5 * control_zone_range
impulse_volume_expansion >= 1.2
```

---

## 6. E. 深回撤字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| pullback_detected | bool | 是否检测到回撤 |
| pullback_low_time | datetime | 回撤低点时间 |
| pullback_low_price | double | 回撤低点价格 |
| retracement_ratio | double | 回撤比例 |
| in_deep_retracement_zone | bool | 是否在0.618-0.786 |
| breaks_poc | bool | 是否跌破POC |
| breaks_val | bool | 是否跌破VAL |
| fast_recovery_after_break | bool | 跌破后是否快速收回 |
| pullback_volume_ratio | double | 回撤成交量相对第一波比例 |
| pullback_quality | enum | healthy / dangerous / destructive |

建议：

```text
0.618 <= retracement_ratio <= 0.786 为理想深回撤
retracement_ratio > 0.886 且无法收回 POC/VAL，风险升高
```

---

## 7. F. POC 回收字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| poc_touched | bool | 是否重新接近POC |
| poc_reclaimed | bool | 是否重新站回POC |
| poc_reclaim_time | datetime | 回收POC时间 |
| close_above_poc_bars | int | 连续收盘在POC上方K线数 |
| poc_retest_done | bool | 是否完成POC回踩测试 |
| poc_retest_success | bool | POC回踩是否成功 |
| poc_as_resistance_count | int | POC作为压力失败次数 |
| poc_acceptance_strength | enum | none / weak / medium / strong |

建议初始规则：

```text
weak：收盘站上POC >= 1根
medium：收盘站上POC >= 3根
strong：站上后回踩不破，并再次拉起
```

## 10. K. 成交量确认字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| volume_ma_period | int | 均量周期，默认20 |
| reclaim_volume_ratio | double | 回收POC/AVWAP时成交量倍数 |
| retest_volume_ratio | double | 回撤测试成交量倍数 |
| second_push_volume_ratio | double | 二次拉升成交量倍数 |
| volume_confirmation | enum | none / weak / valid / excessive |

建议：

```text
回撤缩量：retest_volume_ratio < 1.0
二次拉升有效：second_push_volume_ratio >= 1.2
巨量但价格不推进：可能是派发，不是确认
```
