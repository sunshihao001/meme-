```text
不买最低点
不追第一波
不在 Fib 0.618 / 0.786 直接买
不在 POC 第一次触碰时买
不在 AVWAP 第一次站上时买
```

## 3. 当前最大短板

### 3.1 “第一控盘区”定义还不够机器化

当前说法：

```text
第一波控盘区出现，形成筹码峰 / 成本中枢
```

但机器需要知道：

```text
多少根K线算控盘区？
区间最大宽度是多少？
成交量集中度要达到多少？
POC是否必须位于区间中部？
单峰和双峰如何处理？
是否需要VAH/VAL？
```

需要补充概念：

```text
Volume Profile
Market Profile
TPO
POC
Value Area
VAH / VAL
High Volume Node / Low Volume Node
Acceptance / Rejection
Balanced Market / Trend Day
```

---

### 3.2 “重新接受 POC / AVWAP”需要标准化

当前说法：

```text
重新接近 / 站回 POC
重新接受 POC / AVWAP
```

但需要转成规则：

```text
连续N根K线收盘在POC上方？
是否允许影线跌破？
跌破后几根K线内必须收回？
是否要求成交量放大？
是否要求POC从阻力转支撑？
AVWAP斜率是否必须向上？
```

需要补充概念：

```text
Acceptance
Reclaim
Retest
Support-resistance flip
VWAP reclaim
Anchored VWAP
VWAP slope
Close above VWAP
Failed reclaim
```

---

### 3.3 AVWAP 锚点尚未固定

你的模型需要 AVWAP，但当前 MQL5 资料更多是日内 VWAP / 多日 VWAP。

必须明确锚点规则：

```text
锚定第一控盘区起点？
锚定POC形成时刻？
锚定第一波启动K线？
锚定深回撤低点？
锚定重新站回POC那根K线？
```

不同锚点会产生不同 AVWAP。

建议初步拆成三条线：

```text
AVWAP_ZoneStart：从第一控盘区起点锚定
AVWAP_ImpulseStart：从第一波启动点锚定
AVWAP_Reclaim：从重新站回POC那根K线锚定
```

后续用样本观察哪条更稳定。

---

### 3.4 “第一波派发严重”需要评分

当前说法：

```text
第一波派发严重且反抽中继续卖，只能归入诱多反抽 / 尾仓派发风险
```

需要变成风控分数：

```text
distribution_score = 上影线比例 + 巨量滞涨 + 高位横盘失败 + 主动卖压 + 回收失败次数
```

在加密交易里，还可以补充：

```text
大额钱包流出
CEX充值增加
链上持币地址分布恶化
资金费率过热
OI高位上升但价格不涨
```

MQL5 只能处理K线和成交量；链上/钱包数据需要外部机器人补充。

---

### 3.5 “二次确认”需要结构标准

当前二次确认为：

```text
第一次拉起 -> 回撤不破关键成本锚 -> 再次拉升确认
```

需要补充：

```text
第一次拉起必须突破什么？
回撤允许多深？
不破哪个锚：POC、AVWAP、VAL、OB上沿？
第二次拉升是否必须突破第一次拉起高点？
是否需要BOS？
是否需要成交量确认？
```

可映射关键词：

```text
Retest Validation
Break of Structure
Change of Character
Higher Low
Continuation Pattern
Volume Breakout
```

## 4. 当前策略应优先完善的方向

按重要程度排序：

### 第一优先：定义“第一控盘区”

需要从 Market Profile / Volume Profile 学：

```text
POC如何计算
VAH/VAL如何计算
价值区如何划定
高成交量节点和低成交量节点如何识别
什么时候说明市场接受该价格区
```

### 第二优先：定义 AVWAP 锚点

需要解决：

```text
锚点选择规则
多锚点并行观察
AVWAP接受/失效标准
```

### 第三优先：定义二次确认

需要解决：

```text
第一次拉起标准
回撤不破标准
第二次拉升标准
成交量确认标准
BOS确认标准
```

### 第四优先：定义风险否决

需要解决：

```text
派发评分
诱多反抽识别
尾仓派发识别
失败回收识别
```

### 第五优先：建立样本库

需要人工标注：

```text
成功案例
失败案例
诱多案例
健康回撤案例
派发后二测案例
```

没有样本库，无法进入回测阶段。

## 6. 审计结论

你的策略当前不是“买卖系统”，而是：

```text
保守型结构过滤模型
```

它距离真正自动化交易还有三层距离：

```text
概念标准化
-> 指标化检测
-> 样本回测验证
```

当前最应该做的不是直接写EA下单，而是先写：

```text
观察指标 + 状态标注器 + 风险审查表
```

也就是先让系统能回答：

```text
现在处于 S0-S9 哪个状态？
是否重新接受 POC？
是否重新接受 AVWAP？
是否完成二次确认？
是否存在派发风险？
```

只有这些问题能稳定回答，才进入交易规则设计。
