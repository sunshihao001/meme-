# 反证实验设计 v0.1

> 位置：`09_规则与回测/反证实验设计_v0.1.md`  
> 分线：规则与回测 / 反证审计 / 样本标注  
> 状态：v0.1，实验设计初版  
> 用途：在继续优化“第一控盘区成本中枢回收模型”之前，先定义什么情况会推翻、削弱或降级模型，防止策略变成不可证伪叙事。

---

## 1. 本文目的

本文用于设计反证实验，而不是证明模型有效。

核心目标：

```text
先规定什么情况算模型失败，再继续优化模型。
```

如果没有反证实验，模型容易滑向：

```text
上涨了：说明模型有效。
下跌了：说明洗盘没结束。
没启动：说明周期还没到。
失败了：说明这里不是标准形态。
```

这种模型无法被证伪，也不能进入可靠回测或商业化阶段。

---

## 2. 反证优先原则

### 2.1 先固定规则版本，再观察结果

每个样本必须记录：

```text
rule_version
would_rule_have_triggered_ex_ante
entry_rule_triggered
triggered_group
```

如果一个样本在触发时符合当时规则，后续失败，必须记录为失败，而不能事后改写为“不符合”。

---

### 2.2 失败样本不得事后剔除

禁止：

```text
后来跌了，所以它其实不是第一控盘区。
后来没启动，所以这不算重新接受。
后来失败了，所以这个锚点不对，换一个锚点。
```

允许重新分类的唯一条件：

```text
能证明在触发之前，该样本已经违反了当时文件中的事前定义。
```

---

### 2.3 新增规则必须先审计

如果为了解释失败样本而新增规则，必须先回答：

```text
1. 这个规则能否事前识别？
2. 是否会排除成功样本？
3. 是否只对当前失败样本有效？
4. 是否能在样本外提高结果？
5. 是否能被更简单规则替代？
```

未通过审计前，新增规则只能标记为：

```text
candidate_filter
observe_only
not_core_rule
```

不能直接升级为核心规则。

---

## 3. 反证对象

本模型需要被反证的核心假设包括：

```text
H1：第一控盘区能够代表主要资金初始成本。
H2：成本中枢可以用 POC / Value Area / AVWAP 等方式稳定表达。
H3：深洗不是死亡盘，而是后续重新接受的前置结构。
H4：重新接受比普通反抽有更高后续推进概率。
H5：二次启动能够过滤诱多和失败反抽。
H6：复合模型优于更简单的突破 / 均线 / VWAP / 区间回收策略。
H7：图形结构成立时，交易执行也具有商业可行性。
```

每个反证实验都要说明挑战的是哪个假设。

---

## 4. 样本类型与反证用途

### 4.1 A 类：模型符合且成功

用途：

```text
确认理想结构、收益空间和模型正例。
```

最低目标：

```text
20 个样本
```

注意：A 类不能单独证明策略有效，只能作为正例库。

---

### 4.2 B 类：模型符合但失败

用途：

```text
这是最关键反证类别，用于检验模型是否真的有过滤能力。
```

最低目标：

```text
20 个样本
```

细分：

```text
B1：回收后快速跌破成本锚
B2：二次启动后失败
B3：POC / AVWAP 站回但无量推进
B4：进入死亡盘缓跌
B5：诱多拉升后砸回区间
B6：执行成本后正 R 变负
```

反证意义：

```text
如果 B 类样本比例高，说明“重新接受 / 二次启动”不足以过滤失败。
```

---

### 4.3 C 类：模型不完全符合但大涨

用途：

```text
检验模型是否过度保守，是否错过大量高赔率机会。
```

最低目标：

```text
20 个样本
```

细分：

```text
C1：没有明显 sweep，但直接二次启动
C2：没有 POC 回收，但 AVWAP 有效
C3：没有 AVWAP 回收，但 POC 有效
C4：没有回撤不破，直接强拉
C5：第一控盘区不明显，但后续大涨
C6：符合更简单策略但不符合复合策略
```

反证意义：

```text
如果 C 类样本大量存在，说明当前复合策略可能过度复杂或覆盖率不足。
```

---

### 4.4 D 类：看似符合但死亡盘

用途：

```text
训练死亡盘过滤器，检验模型是否会误把派发当吸筹。
```

最低目标：

```text
20 个样本
```

细分：

```text
D1：高位派发伪装成第一控盘区
D2：深洗后无真实资金回流
D3：反复回收 POC / AVWAP 但高点下降
D4：二次启动失败后流动性枯竭
D5：叙事衰退导致结构失效
D6：盘口 / 池子深度恶化导致无法执行
```

反证意义：

```text
如果 D 类无法被过滤，模型会在真实交易中承受严重尾部风险。
```

---

### 4.5 E 类：图形成立但不可执行

用途：

```text
检验图形模型是否有商业可行性。
```

细分：

```text
E1：买点在涨停 / 急拉后无法成交
E2：滑点过大
E3：止损距离过远
E4：需要盘中未来确认
E5：流动性太差
E6：持仓周期过长或资金效率过低
```

反证意义：

```text
图形成立不等于策略可交易。E 类比例高说明模型只能做研究观察，不能升级为交易系统。
```

## 5. 推翻或降级模型的条件

以下情况不一定立即推翻全部模型，但必须触发降级、重构或暂停深化。

### 5.1 核心失败比例过高

如果在固定规则版本和固定样本范围内：

```text
B 类：模型符合但失败样本比例过高
D 类：看似符合但死亡盘样本比例过高
E 类：图形成立但不可执行比例过高
```

则模型不得继续只通过新增过滤条件深化，必须进入规则审计。

Owner 待决策：

```text
失败比例阈值：UNKNOWN
最小样本量：UNKNOWN
```

---

### 5.2 大量 C 类样本存在

如果大量大涨样本不满足当前复合模型，而满足更简单模型，则说明：

```text
当前模型可能过度保守。
当前模型可能只是大涨结构的一个子集。
当前模型不应被包装成通用大涨捕捉策略。
```

---

### 5.3 消融后表现不下降

如果去掉某个核心条件后，模型表现没有下降，甚至更好，则该条件必须降级。

可能降级对象：

```text
深洗条件
POC 回收条件
AVWAP 回收条件
回撤不破条件
二次启动条件
SMC 辅助条件
成交量过滤条件
```

---

### 5.4 规则只能事后解释

如果某个规则无法在触发前识别，只能在结果出来后解释，则不能进入核心规则。

典型后视风险：

```text
事后移动第一控盘区边界
事后更换 AVWAP 锚点
失败后重画 POC 区间
用未来大涨证明当时是二次启动
```

## 6. 消融实验设计

### 6.1 Baseline 0：不使用本模型

目的：

```text
比较当前模型是否真的优于简单规则。
```

基准可包括：

```text
简单突破策略
均线重新站回策略
VWAP 站回策略
区间突破策略
随机入场 / 持有基准
```

---

### 6.2 Ablation A：去掉深洗条件

问题：

```text
如果不要求深洗，只要求成本锚回收，结果是否变差？
```

判断：

```text
如果表现不变，深洗可能只是叙事条件。
如果失败样本大幅增加，深洗可能有过滤价值。
```

---

### 6.3 Ablation B：去掉重新接受条件

问题：

```text
如果只看第一控盘区和回调，不要求重新接受，结果是否变差？
```

判断：

```text
若表现显著变差，重新接受有核心价值。
若表现不变，重新接受定义可能无效或过拟合。
```

---

### 6.4 Ablation C：去掉二次启动条件

问题：

```text
如果在站回成本锚后直接入场，而不等二次启动，收益风险比如何变化？
```

判断维度：

```text
胜率
平均 R
最大回撤
错过收益
诱多比例
```

---

### 6.5 Ablation D：去掉 POC / AVWAP 其中之一

问题：

```text
POC 和 AVWAP 谁真正贡献更多？
是否二者必须共振？
```

实验组：

```text
只用 POC
只用 AVWAP
POC + AVWAP 共振
Value Area 回收
```

---

### 6.6 Ablation E：去掉 SMC 辅助条件

问题：

```text
OB / BOS / FVG 是有效过滤器，还是增加复杂度的叙事？
```

判断：

```text
如果去掉 SMC 后结果不下降，SMC 应降级为标注备注或辅助解释。
```

## 8. 不允许的解释方式

以下解释不得直接用于维护模型正确性：

```text
这里主力没做完
这里周期还没结束
这里换庄了
这里是特殊情况
这次不是标准深洗
这次不是标准二次启动
只要再加一个过滤条件就好了
```

除非它们能被转化为事前可观察、可标注、可复验的规则，否则只能写入备注，不得写入核心规则。

## 9. Owner 决策清单

进入正式实验前，Owner 需要确认：

```text
1. 第一批样本范围：市场 / 周期 / 时间段 / 标的类型。
2. 成功标准：最低净 R、最大回撤、持仓周期、可执行性。
3. 失败阈值：B / D / E 类比例达到多少必须降级。
4. C 类大涨漏报比例达到多少说明模型过度保守。
5. 首批消融条件优先级。
6. 是否允许使用 SMC 作为核心条件。
7. POC、Value Area、AVWAP 的优先级。
8. 不可执行样本是否直接判定策略失败。
```

## 10. 本版结论

当前模型必须先通过反证框架约束，才能继续进入更深的规则、指标和回测阶段。

本文件的核心要求是：

```text
先定义失败，再讨论成功。
先收集反例，再优化规则。
先做消融，再判断哪些条件是真正有效。
先判断可执行性，再谈商业化。
```
