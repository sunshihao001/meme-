# 基准策略对比实验设计 v0.1

> 版本：v0.1  
> 分线：规则与回测线 / 样本标注线 / 风险管理线  
> 目的：验证“第一控盘区深洗后 POC/AVWAP 成本锚回收 + 二次启动确认”复合策略，是否真的优于更简单的 POC、AVWAP、Liquidity Sweep 基准策略。  
> 状态：实验设计初版 / 待样本库建立 / 待回测实现  

---

## 1. 本文要解决的问题

当前策略已经形成较完整的结构链条：

```text
第一控盘区 -> 第一波拉升 -> 深洗 / Sweep -> POC/AVWAP 回收 -> 回撤不破 -> 二次拉升确认
```

但现在最大的问题不是概念是否漂亮，而是：

```text
这个复合策略是否真的比简单策略更有 edge？
```

必须回答：

```text
1. 复合条件是否提高了胜率？
2. 复合条件是否改善了平均盈亏比？
3. 复合条件是否降低了死亡盘 / 诱多盘误入率？
4. 复合条件是否只是减少交易次数，并没有改善期望值？
5. POC、AVWAP、Sweep、二次启动中，真正贡献 edge 的是哪一层？
```

---

## 2. 核心假设

### 2.1 主假设

```text
当前复合策略相对于简单 POC 站回、简单 AVWAP 站回、简单 Liquidity Sweep 回收，能够以更少交易次数换取更高质量样本，并改善期望值。
```

### 2.2 反假设

```text
当前复合策略只是把多个常见交易概念叠加在一起，降低了交易频率，但没有显著提升胜率、盈亏比或风险过滤能力。
```

### 2.3 最危险的结果

```text
复合策略回测结果看起来更稳，但只是因为样本太少、筛选过度、事后定义过强，导致过拟合。
```

## 3. 要对比的 4 组策略

### 3.1 基准 A：简单 POC 站回策略

定义：

```text
当价格从下方重新收盘站回第一控盘区 POC 上方时，记为触发。
```

最小条件：

```text
1. 已存在人工标注的第一控盘区。
2. 已计算 POC。
3. 价格曾经跌破 POC。
4. 后续收盘重新站上 POC。
```

不要求：

```text
AVWAP 站回
Sweep 结构
回撤不破
二次拉升确认
```

用途：

```text
检验“POC 站回”单独是否已经有足够 edge。
```

---

### 3.2 基准 B：简单 AVWAP 站回策略

定义：

```text
当价格从下方重新收盘站回关键 AVWAP 成本锚上方时，记为触发。
```

候选锚点：

```text
AVWAP_ZoneStart
AVWAP_ImpulseStart
AVWAP_Reclaim
```

最小条件：

```text
1. 已存在人工标注的第一控盘区。
2. 已定义至少一个 AVWAP 锚点。
3. 价格曾经跌破该 AVWAP。
4. 后续收盘重新站上该 AVWAP。
```

不要求：

```text
POC 站回
Value Area 重新进入
Sweep 结构
回撤不破
二次拉升确认
```

用途：

```text
检验“AVWAP 成本锚”单独是否有效。
```

---

### 3.3 基准 C：简单 Liquidity Sweep 回收策略

定义：

```text
价格跌破前低或第一控盘区下沿后，迅速重新收回关键水平，记为触发。
```

最小条件：

```text
1. 有明确前低或第一控盘区下沿。
2. 价格向下刺破该水平。
3. N 根 K 线内收回该水平上方。
```

建议初始参数：

```text
N = 3 到 5 根 K 线
```

不要求：

```text
POC 站回
AVWAP 站回
Value Area 接受
回撤不破
二次拉升确认
```

用途：

```text
检验“深洗 / 扫流动性后快速回收”单独是否有效。
```

---

### 3.4 实验 D：当前复合策略

定义：

```text
第一控盘区成立后，价格经历深洗 / Sweep，并重新进入 Value Area 或站回 POC/AVWAP 成本锚；随后第一次拉起，回撤不破关键成本锚，再次拉升确认。
```

最小条件：

```text
1. first_control_zone_detected = true
2. first_impulse_done = true
3. deep_washout_detected 或 sweep_detected = true
4. value_area_reentered 或 poc_reclaimed = true
5. avwap_reclaimed = true 或 avwap_confluence_count >= 1
6. first_bounce_detected = true
7. pullback_holds_anchor = true
8. second_push_confirmed = true 或 bos_after_reclaim = true
9. death_market_score 不高
10. bull_trap_score 不高
```

用途：

```text
检验“多条件复合过滤”是否真的优于简单基准。
```

## 4. 样本分组设计

为了避免只看成功样本，必须建立 4 类样本。

### 4.1 A 类：完全符合模型且成功

```text
模型条件基本全部满足，后续出现明显二次启动或高 R 扩展。
```

用途：

```text
确认模型想抓的理想结构长什么样。
```

---

### 4.2 B 类：完全符合模型但失败

```text
模型条件基本满足，但后续失败、重新跌破、诱多、死亡盘或收益不足。
```

用途：

```text
找出模型最危险的盲区。
```

---

### 4.3 C 类：不完全符合模型但大涨

```text
没有完整满足当前复合策略，但后续大涨。
```

用途：

```text
评估当前策略是否过度保守，是否错过太多高赔率机会。
```

---

### 4.4 D 类：看似早期符合但最后死亡盘

```text
早期看起来像第一控盘区、深洗或回收，但最终进入死亡盘 / 尾仓派发。
```

用途：

```text
训练死亡盘过滤器和一票否决条件。
```

## 7. 入场与出场统一口径

为了公平比较 4 组策略，必须使用统一风控框架。

### 7.1 入场价

默认：

```text
触发条件成立后的下一根 K 线开盘价。
```

如果用于人工复盘，也可记录：

```text
触发 K 收盘价
触发后回踩价
触发后突破价
```

但主比较只能选一个统一口径。

---

### 7.2 止损

默认止损优先级：

```text
1. sweep low 下方
2. 第一控盘区下沿下方
3. 回撤确认低点下方
4. 固定 ATR 止损
```

对不同策略使用同一止损逻辑，避免偏袒复合策略。

---

### 7.3 止盈

至少统计 3 种结果：

```text
1. 固定 1:2 止盈
2. 固定 1:3 止盈
3. 结构退出：跌破 POC/AVWAP 或二次启动失败
```

同时记录 MFE / MAE：

```text
MFE = 最大有利浮盈 R
MAE = 最大不利浮亏 R
```

---

### 7.4 持仓超时

必须设置超时规则：

```text
如果触发后 X 根 K 线内没有达到 1R，标记为 momentum_failure_candidate。
```

初始参数：

```text
X = 20 到 50 根 K 线，按周期调整。
```

## 9. 判定标准

### 9.1 复合策略通过最低标准

必须至少满足：

```text
1. expectancy_R > 0
2. profit_factor > 1.2
3. 相比至少 2 个简单基准，expectancy_R 更高
4. 相比简单基准，死亡盘 / 诱多误入率更低
5. 交易次数不能低到失去实际意义
```

---

### 9.2 复合策略较好标准

```text
1. win_rate >= 35%
2. average_R >= 0.5R
3. profit_factor >= 1.5
4. max_drawdown_R 可控
5. 相比 3 个简单基准均有明显改善
```

---

### 9.3 复合策略优质标准

```text
1. win_rate >= 40%
2. average_R >= 0.8R
3. profit_factor >= 1.8
4. 在不同市场阶段仍保持正期望
5. 对 B 类和 D 类失败样本有明确过滤能力
6. C 类漏掉的大涨样本不显著损害总体收益
```

## 10. 可能结果与解释

### 10.1 结果 A：复合策略显著优于所有基准

解释：

```text
当前模型的多层过滤确实创造了 edge。
```

下一步：

```text
进入 MQL5 观察指标 MVP 与更大样本回测。
```

---

### 10.2 结果 B：复合策略只优于 POC，但不优于 AVWAP / Sweep

解释：

```text
POC 不是核心 edge，AVWAP 或 Sweep 可能贡献更大。
```

下一步：

```text
降低 POC 权重，重新评估 AVWAP 与 Sweep 的主导性。
```

---

### 10.3 结果 C：复合策略胜率更高，但平均 R 更低

解释：

```text
确认太晚，过滤提高了确定性，但牺牲了赔率。
```

下一步：

```text
设计激进版入场：Sweep reclaim / POC reclaim 先观察仓，二次确认再加仓。
```

---

### 10.4 结果 D：复合策略没有优于简单基准

解释：

```text
当前模型只是复杂化，没有形成 edge。
```

下一步：

```text
回到最有效的简单基准，把当前模型降级为风险审查工具。
```

---

### 10.5 结果 E：所有策略都不稳定

解释：

```text
结构本身不是主导变量，标的筛选、叙事、资金流、流动性才是主导变量。
```

下一步：

```text
把本策略定位为更大系统中的结构模块，优先补标的筛选和链上资金流模块。
```

## 11. 防止自欺的实验纪律

```text
1. 不只选成功案例。
2. 不允许事后移动第一控盘区边界。
3. 不允许失败后重新解释锚点。
4. 不允许为了让复合策略胜出而给不同策略不同止损。
5. 不允许只比较胜率，必须比较 expectancy_R 和 profit_factor。
6. 不允许忽略滑点、流动性和手续费。
7. 不允许把样本太少的结果当成策略结论。
```

```text
   https://www.mql5.com/en/articles/16340
3. AVWAP 最受尊重锚点讨论：
   https://www.mql5.com/en/forum/455444
4. False Breakout Detector：
   https://www.mql5.com/en/market/product/145504
5. Continuous Walk-Forward Optimization：
   https://www.mql5.com/en/articles/7290
   https://www.mql5.com/en/articles/7891
6. MT5 优化与过拟合讨论：
   https://www.mql5.com/en/forum/451696
   https://www.mql5.com/en/forum/459197
7. 回测与实盘差异：
   https://www.mql5.com/en/forum/502033
```

核心回填结论：

```text
1. 当前复合策略必须按“事件顺序”测试，不能按孤立指标测试。
2. AVWAP 锚点必须从固定锚点升级为“候选锚点评分”。
3. Sweep 必须要求“刺破 + 快速回收 + 回到前结构”，不能只记录刺破。
4. BOS / OB / FVG 只能作为二次启动确认或过滤模块，不能覆盖第一控盘区成本锚逻辑。
5. 回测必须采用样本外 / walk-forward / 多窗口，而不能只做单段历史优化。
6. 优化准则不能只看 Balance / 净利润，必须记录稳定性、交易次数、回撤、期望值和失败率。
```

## 13. Wyckoff 状态机对当前实验的修正

### 13.1 可吸收的状态机思想

Wyckoff 自动化文章强调：

```text
Spring 不是普通跌破。
Spring = 已定义区间中的假跌破 + 回收 + 成交量语境 + 事件顺序。
```

因此当前复合策略不能只写成：

```text
sweep_detected = true
```

而应拆成：

```text
range_or_control_zone_locked = true
prior_impulse_or_prior_trend_confirmed = true
spring_or_sweep_below_support = true
spring_or_sweep_reclaimed = true
sos_or_first_bounce_confirmed = true
lps_or_pullback_holds_anchor = true
second_push_or_bos_confirmed = true
```

### 13.2 当前模型与 Wyckoff 的映射

```text
Wyckoff Trading Range        -> 第一控盘区 / 成本中枢
Spring                       -> 深洗 / Sweep / 假跌破
SOS                          -> 第一次拉起 / 回收后强势推进
LPS                          -> 回撤不破 POC/AVWAP 成本锚
Markup                       -> 二次启动 / 扩展段
Upthrust / Distribution       -> 诱多反抽 / 尾仓派发 / 死亡盘
```

### 13.3 对实验组 D 的修正

原实验 D 只写“复合策略”，现在应拆成两个版本：

```text
D1：当前复合策略宽松版
- 第一控盘区
- 深洗 / sweep
- POC 或 AVWAP 回收
- 回撤不破
- 二次推进

D2：Wyckoff 顺序严格版
- prior trend / prior impulse 已确认
- range / FCZ 已锁定，不能事后移动
- sweep 必须发生在已锁定区间边界附近
- reclaim 必须在 N 根 K 内完成
- first bounce / SOS 必须具有推进质量
- pullback / LPS 必须低量或不破关键锚
- second push / BOS 才能确认候选
```

目的：

```text
检验“严格事件顺序”是否真的比普通复合条件更稳定。
```

### 13.4 新增失效规则

从 Wyckoff 状态机吸收以下 reset 规则：

```text
1. Spring / sweep 后再次有效跌破 sweep low：结构重置。
2. SOS / first bounce 后无法维持在成本锚上方：降级为 failed_reclaim。
3. LPS / pullback 阶段跌破 POC/AVWAP 与 FCZ 下沿：取消 second_start_candidate。
4. 二次推进前出现放量滞涨或连续上影：提高 bull_trap_score。
```

## 14. SMC OB / BOS / FVG 对实验的修正

### 14.1 可吸收模块

SMC 文章把以下模块写成可执行 EA：

```text
Order Block
Fair Value Gap
Break of Structure
swing detection
chart drawing
strategy enum
```

对当前实验最有价值的是：

```text
1. swing high / swing low 的统一检测方式。
2. BOS 作为结构确认，而不是单独买卖信号。
3. FVG 作为推进质量或回撤效率辅助字段。
4. OB 作为潜在回踩区，但不能替代第一控盘区。
```

### 14.2 新增 SMC 辅助字段

建议在样本字段中追加：

```text
swing_high_before_reclaim
swing_low_before_reclaim
bos_level_after_reclaim
bos_direction
bos_confirmed_by_close
fvg_after_reclaim_detected
fvg_midpoint_retested
order_block_near_anchor
order_block_retest_holds
```

### 14.3 对实验组的影响

新增一个对照实验：

```text
G组：SMC确认增强版
条件：完整复合策略 + BOS 或 FVG 或 OB 回踩确认
目的：测试 SMC 确认是否能提高质量，还是只会进一步滞后入场。
```

必须记录：

```text
G组相对 D组的 trade_count 下降比例
G组相对 D组的 expectancy_R 变化
G组相对 D组的 missed_second_start_rate 变化
G组相对 D组的 average_entry_lag_bars
```

### 14.4 禁止事项

```text
1. 不允许把 OB/FVG/BOS 直接替代 FCZ/POC/AVWAP。
2. 不允许因为出现 BOS 就忽略死亡盘风险。
3. 不允许用 SMC 术语事后解释失败样本。
4. SMC 只能作为确认层或过滤层。
```

## 15. AVWAP 锚点选择对实验的修正

### 15.1 固定锚点不够

AVWAP 论坛讨论的重点是：

```text
静态 VWAP / MA / AVWAP 设置容易过拟合并快速失效。
应尝试寻找“当前最受价格尊重”的 anchored VWAP。
```

当前项目已有：

```text
AVWAP_ZoneStart
AVWAP_ImpulseStart
AVWAP_Reclaim
AVWAP_PullbackLow
```

但实验不能只比较“是否站回某一条 AVWAP”，还要比较：

```text
哪个锚点更受价格尊重？
多锚点共振是否优于单锚点？
固定锚点是否输给动态锚点评分？
```

### 15.2 新增 AVWAP 锚点评分字段


### 15.3 respect_score 暂定定义

第一版可以用简单评分，不急于复杂相关矩阵：

```text
respect_score = hold_count / max(touch_count, 1) - false_reclaim_count / max(touch_count, 1)
```

其中：

```text
touch_count：价格触及 AVWAP 容差带次数
hold_count：触及后 N 根 K 内没有有效跌破并出现反应次数
false_reclaim_count：站回后 N 根 K 内重新跌破次数
```

容差建议：

```text
AVWAP_tolerance = max(0.25 * ATR, 当前价格 * 0.001)
```

### 15.4 新增 AVWAP 对照实验

```text
B1：固定 ZoneStart AVWAP 站回
B2：固定 ImpulseStart AVWAP 站回
B3：固定 Reclaim AVWAP 站回
B4：best_respected_avwap_anchor 站回
B5：至少两条 AVWAP 共振站回
```

目的：

```text
确认 AVWAP edge 来自某个固定锚点，还是来自动态“市场尊重”的锚点。
```

## 16. False Breakout / Sweep 对实验的修正

### 16.1 False Breakout Detector 可吸收定义

该指标定义的核心是：

```text
价格暂时突破近期支撑/阻力，但快速回到前一区间。
```

### 16.2 当前 sweep 字段拆分

```text

```

### 16.3 Sweep 触发分级

```text
S0_NO_SWEEP：无扫流动性
S1_WICK_ONLY：只用影线刺破，收盘未跌破
S2_CLOSE_BREAK_RECLAIM：收盘跌破后 N 根内收回
S3_DEEP_SWEEP_RECLAIM：明显跌破后快速收回
S4_FAILED_SWEEP：刺破或跌破后无法收回
```

### 16.4 对基准 C 的修正

基准 C 不应只有一个版本，应拆为：

```text
C1：wick-only sweep reclaim
C2：close-break then reclaim within 3 bars
C3：close-break then reclaim within 5 bars
C4：sweep reclaim + bullish close strength
C5：sweep reclaim + next pullback holds reclaimed level
```

目的：

```text
确认 sweep 的有效性来自“刺破本身”，还是来自“快速收回后的保持”。
```

## 17. Walk-forward 与过拟合纪律回填

### 17.1 为什么必须 walk-forward

MQL5 walk-forward 资料强调：

```text
优化不是证明策略有效，而是稳健性测试的一部分。
```

MT5 论坛讨论进一步指出：

```text
1. Balance Max 可能掩盖长期亏损。
2. 过多参数会提高过拟合风险。
3. 策略默认参数若完全无效，靠优化救活通常不可靠。
4. forward testing 才能检查参数是否延续有效。
```

### 17.2 本项目最低测试分层

当前策略后续至少做三层：

```text
Layer 1：固定参数样本标注统计
- 不优化参数，只看规则本身是否有正期望倾向。

Layer 2：少量参数敏感性测试
- 只允许调 sweep_reclaim_bars、AVWAP_tolerance、acceptance_bars 等少数参数。

Layer 3：walk-forward / 样本外验证
- 用早期样本确定参数，在后续样本中验证。
```

### 17.3 推荐窗口

如果用于 MT5 / crypto K线回测：

```text
历史优化窗口：3 到 6 个月
forward窗口：1 个月
滚动方式：每次向前移动一个 forward 窗口
```

如果用于 meme 人工样本：

```text
训练批次：前 60% 样本
验证批次：中间 20% 样本
最终样本外：最后 20% 样本
按时间排序，不允许随机打乱后再声称样本外。
```

### 17.4 必须记录的优化元数据

```text
test_symbol
test_timeframe
data_source
broker_or_exchange
spread_model
fee_model
slippage_model
tick_mode_or_bar_mode
in_sample_start
in_sample_end
out_sample_start
out_sample_end
optimized_parameters_count
optimization_criterion
trial_count
selected_pass_rank
```

### 17.5 不允许的验证方式

```text
1. 只展示最好的一段行情。
2. 只展示最优参数组合。
3. 只用 Balance / Net Profit 排名。
4. 改完规则后回头重新标注旧样本但不记录版本。
5. 多次试验后只报告成功结果。
```

## 18. 回测实盘差异与执行成本回填

MQL5 论坛关于回测与实盘差异的讨论指出：

```text
即使使用 same EA、same settings、same period、real ticks，回测和实盘仍可能不同。
```

常见原因：

```text
slippage
variable spread
execution delay
broker liquidity
order filling behavior
margin calculation
tick sequence modeling
broker-specific stop-out rules
latency
```

对 meme / crypto 的影响更严重：

```text
1. meme 流动性薄，冲击成本可能吞噬理论 R。
2. 快速拉升/砸盘阶段，K线回测会高估可成交性。
3. sweep/reclaim 策略对滑点敏感，因为入场常发生在波动扩大时。
4. DEX/链上执行还存在 MEV、gas、交易失败、池子深度变化。
```

因此结果字段必须追加：

```text
estimated_slippage_bps
estimated_fee_bps
liquidity_depth_at_entry
entry_execution_quality
bar_backtest_fill_possible
live_fill_risk_level
```

策略通过标准中新增：

```text
扣除保守滑点和费用后 expectancy_R 仍必须 > 0。
```

## 21. 本版结论

当前策略是否专业，不应该靠概念判断，而应该靠基准对比判断。

深读资料后的关键判断是：

```text
当前策略最接近 Wyckoff re-accumulation / Spring / SOS / LPS 的 meme 成本锚版本。
```

但它不是 Wyckoff 的简单改名，因为它额外强调：

```text
第一控盘区
POC / VAH / VAL 成本中枢
多锚点 AVWAP
成本锚重新接受
诱多过滤
二次启动确认
```

最关键的问题仍然是：

```text
复合策略是否显著优于简单 POC / AVWAP / Sweep 策略？
```

如果优于，说明当前组合有价值，可以继续向指标和回测系统推进。

如果不优于，说明当前策略更适合作为：

```text
结构观察器
样本标注框架
多机器人风险审查输入
```

而不应该直接升级为自动交易策略。
