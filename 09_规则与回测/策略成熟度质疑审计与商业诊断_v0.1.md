# 策略成熟度质疑审计与商业诊断 v0.1

> 版本：v0.1  
> 分线：规则与回测线 / 商业诊断线 / 反证审计线  
> 目的：停止无条件顺从式深化，建立“质疑、反证、商业可用性诊断”模板，用于判断当前结构策略是否已经成熟、是否需要修改、是否值得继续开发为指标/EA/产品。  
> 状态：第一版审计框架 + 当前策略初审结论。  

---

## 1. 立场切换

本文件从“继续深化你的策略构思”切换为：

```text
策略审计师
反证研究员
商业诊断顾问
风险委员会
产品化负责人
```

后续不再默认：

```text
用户提出一个方向 -> 就继续深化该方向
```

而是先判断：

```text
1. 这个方向是否真的解决核心问题？
2. 是否只是让概念越来越复杂？
3. 是否能被样本、回测、商业流程证明？
4. 是否值得继续投入？
5. 是否应该停止、降级、重构或只保留为观察模块？
```

## 2. 对当前策略的直接审计结论

### 2.1 结论摘要

当前“第一控盘区成本中枢回收模型”目前不能判定为：

```text
成熟、无需修改、可直接商业化的完整交易策略。
```

更准确评级是：

```text
有专业雏形的结构性假设模型，尚未通过成熟策略审计。
```

阶段评级：

```text
研究成熟度：中高
规则成熟度：中
验证成熟度：低
商业成熟度：低到中
自动交易成熟度：低
观察器/样本标注成熟度：中高
```

### 2.2 为什么不能说已经成熟

原因：

```text
1. 目前主要是结构逻辑完整，不是统计验证完整。
2. 还没有证明它优于 POC-only、AVWAP-only、Sweep-only 等简单基准。
3. 还没有证明复杂条件带来的收益高于交易次数下降和入场滞后成本。
4. 还没有失败样本库支撑反证。
5. 还没有样本外、walk-forward 或跨市场验证。
6. 还没有执行成本、滑点、流动性冲击后的净期望验证。
7. 规则中仍有不少人工解释空间，存在事后合理化风险。
```

### 2.3 但也不能否定它

它有继续研究价值，因为：

```text
1. 不是单指标信号，而是结构序列。
2. 能与 Wyckoff、SMC、POC、AVWAP、False Breakout 建立成熟体系映射。
3. 已经意识到死亡盘、诱多反抽、回撤不破、二次启动等关键风险。
4. 适合作为观察器、样本标注体系、风险筛选模块。
5. 如果消融实验通过，可能具备策略产品化潜力。
```

## 3. 核心审计问题

后续每次用户要求“继续深化某个方向”前，必须先问以下审计问题。

### 3.1 策略必要性审计

```text
这个新增概念是否解决真实缺陷？
还是只是增加解释能力？
```

判断标准：

```text
1. 是否降低 false_positive？
2. 是否降低死亡盘误入？
3. 是否提升 expectancy_R？
4. 是否提升样本外稳定性？
5. 是否只是让失败样本更容易被事后解释？
```

如果不能明确改善上述任一项，则不得升级为核心规则，只能进入“观察字段”。

---

### 3.2 复杂度收益审计

```text
这个规则增加的复杂度，是否换来了足够边际收益？
```

必须比较：

```text
新增规则前：win_rate / expectancy_R / trade_count / max_drawdown / false_positive
新增规则后：win_rate / expectancy_R / trade_count / max_drawdown / false_positive
```

危险信号：

```text
1. 胜率上升但 average_R 下降。
2. trade_count 大幅下降导致样本不足。
3. 只过滤了明显失败样本，但也错过了多数大涨样本。
4. 需要频繁解释“这个案例特殊”。
```

---

### 3.3 反证优先审计

每个规则都必须能被反证：

```text
如果出现什么样的样本，就说明这个规则无效？
```

示例：

```text
规则：二次启动确认能过滤诱多。
反证：大量失败样本也出现二次启动确认，但随后快速跌破成本锚。

规则：AVWAP 重新接受有 edge。
反证：AVWAP-only 与随机入场或普通均线回收没有显著差异。

规则：第一控盘区有效。
反证：人工标注的第一控盘区与后续收益无统计关系，或边界变化导致结论剧烈变化。
```

## 4. 成熟策略必须通过的 10 道门

### Gate 1：定义是否不可随意移动

```text
第一控盘区边界是否能在事前确定？
POC/AVWAP 锚点是否能在触发前确定？
Sweep 参考水平是否能在发生前确定？
```

不通过表现：

```text
失败后重新解释边界、锚点、周期或结构。
```

---

### Gate 2：是否有简单基准对照

必须证明：

```text
复合策略 > POC-only
复合策略 > AVWAP-only
复合策略 > Sweep-only
复合策略 > POC+AVWAP but no second start
```

如果不成立：

```text
复合策略不能称为成熟策略，只能称为观察框架。
```

---

### Gate 3：是否有失败样本库

成熟策略必须研究失败，而不是只研究成功。

最低要求：

```text
成功样本 ≥ 20
失败样本 ≥ 20
错过大涨样本 ≥ 20
死亡盘误判样本 ≥ 20
```

---

### Gate 4：是否有样本外验证

必须至少有：

```text
时间排序样本外验证
或 walk-forward 验证
或新增样本盲测
```

不允许：

```text
先看完整历史，再回头定义规则。
```

---

### Gate 5：是否扣除了真实交易成本

必须考虑：

```text
手续费
滑点
流动性深度
冲击成本
gas / MEV / 交易失败风险，若用于链上 meme
```

若扣成本后期望值转负，则策略不能商业化。

---

### Gate 6：是否有参数稳定性

成熟策略不应只在极窄参数下有效。

必须测试：

```text
sweep_reclaim_bars = 3/5/8
AVWAP_tolerance = 0.15/0.25/0.5 ATR
pullback_holds_anchor_bars = 3/5/8
stop buffer = 0.5/1/1.5 ATR
```

如果参数轻微变化结果崩溃：

```text
策略大概率过拟合。
```

---

### Gate 7：是否能解释漏掉的大涨

如果大量 C 类样本：

```text
不完全符合模型但大涨
```

则要判断：

```text
策略是否过度保守？
是否应该降级为风险过滤器，而不是主入场策略？
```

---

### Gate 8：是否有明确停止条件

必须明确什么时候停止继续深化。

停止条件：

```text
1. 连续两轮新增规则不能提升样本外 expectancy_R。
2. 复合策略不能优于两个以上简单基准。
3. 失败样本显示核心假设被反复击穿。
4. 规则复杂度上升但商业可执行性下降。
```

---

### Gate 9：是否有商业角色定位

成熟策略不一定必须自动交易。

可选定位：

```text
1. 自动交易 EA
2. 半自动观察指标
3. 样本标注系统
4. 风险审查器
5. 多策略组合中的过滤模块
6. 交易员训练/复盘工具
```

当前更适合：

```text
半自动观察指标 + 样本标注系统 + 风险审查器
```

暂不适合直接定位为：

```text
全自动交易 EA
```

---

### Gate 10：是否能被非发明者执行

成熟商业策略必须可以被他人执行。

测试：

```text
给另一个人/Agent 同一批 K 线，是否能标出相同第一控盘区、相同 sweep、相同 AVWAP 锚点、相同入场？
```

如果一致性低：

```text
策略还不是成熟规则，只是主观经验框架。
```

## 5. 当前策略商业诊断

### 5.1 产品化方向评级

```text
全自动 EA：暂不建议
MQL5 观察指标：建议
样本标注工具：强烈建议
策略审计报告模板：强烈建议
交易提醒/候选筛选器：建议
资金实盘策略：暂不建议
```

### 5.2 最合理商业路线

当前最合理路线不是：

```text
马上写 EA 自动交易
```

而是：

```text
Phase 1：观察器
- 自动画第一控盘区候选
- 自动画 POC/VAH/VAL
- 自动画多锚点 AVWAP
- 标记 sweep/reclaim
- 标记二次启动候选

Phase 2：样本标注器
- 导出每个样本字段
- 自动记录成功/失败/错过/死亡盘
- 生成消融实验 CSV

Phase 3：策略审计器
- 自动对比 POC-only/AVWAP-only/Sweep-only/复合策略
- 输出胜率、期望值、最大回撤、漏报率、误报率

Phase 4：半自动提醒器
- 只提醒 second_start_candidate
- 不自动下单

Phase 5：只有通过样本外验证后，才讨论 EA
```

### 5.3 商业价值不在“策略神奇”

更可能的商业价值是：

```text
1. 帮助交易者不被早期诱多骗。
2. 帮助筛选二次启动候选。
3. 帮助复盘 meme 早期控盘结构。
4. 帮助沉淀可验证样本库。
5. 帮助把主观结构交易转为可审计流程。
```

## 6. 质疑审计模板

以后每次新增概念、规则或指标，都必须填写：

```text
概念名称：
提出原因：
解决哪个已知失败问题：
如果不加入会怎样：
加入后预计改善哪个指标：
可能引入什么副作用：
如何反证它无效：
需要哪些样本字段：
与现有规则是否重叠：
是否可由更简单规则替代：
是否影响商业可执行性：
结论：核心规则 / 辅助字段 / 观察字段 / 暂不纳入
```

## 8. 后续 Agent 工作方式修正

后续当用户要求继续深化某个策略方向时，Agent 必须先执行：

```text
1. 先质疑：这个方向是否必要？
2. 再审计：它解决哪个失败问题？
3. 再反证：什么样本能证明它错？
4. 再定位：核心规则、辅助字段、观察字段还是暂不纳入？
5. 最后才写入知识库。
```

禁止默认行为：

```text
用户说深化 -> 直接继续扩写概念。
```

推荐默认行为：

```text
用户说深化 -> 先给成熟度诊断 / 反证清单 / 是否值得继续的结论。
```

## 9. 本版最终判断

当前策略不是“已经成熟、无需修改”的策略。

它更像：

```text
一个有专业雏形、方向合理、但尚未经过反证和商业审计的结构性策略假设。
```

下一阶段不应继续盲目深化概念，而应进入：

```text
反证审计阶段
消融实验阶段
样本一致性阶段
商业产品定位阶段
```

最推荐的下一步不是继续加规则，而是建立：
