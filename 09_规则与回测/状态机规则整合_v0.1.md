# 状态机规则整合 v0.1

> 版本：v0.1  
> 分线：规则与回测线 / 指标需求线 / 样本标注线  
> 目的：把“第一控盘区成本中枢回收模型”从描述性结构，整理成可标注、可导出、可审计、可消融实验的有限状态机。  
> 状态：状态机初版 / 服务观察器与样本标注 / 非自动交易规则。  

---

## 1. 本文件要解决的问题

当前策略描述已经比较完整：

```text
第一控盘区 -> 第一波拉升 -> 深洗/Sweep -> POC/AVWAP 回收 -> 回撤不破 -> 二次启动
```

但描述性结构存在三个问题：

```text
1. 不同人可能对阶段理解不同。
2. 失败后容易事后移动边界或换锚点。
3. 指标、样本、消融实验无法稳定复现同一规则。
```

所以需要状态机，把策略拆成：

```text
状态
进入条件
退出条件
失败条件
可导出字段
禁止事后解释
```

---

## 2. 设计原则

### 2.1 状态机不是交易信号

本状态机不是：

```text
自动下单 EA
买卖信号神器
收益承诺规则
```

它是：

```text
观察状态标注器
样本审计框架
消融实验分组依据
MQL5 指标 MVP 的状态输出层
```

---

### 2.2 先锁定，再判断

任何后续判断必须基于已经锁定的结构：

```text
FCZ 已锁定后，才能判断 sweep 是否有效。
AVWAP 锚点已锁定后，才能判断是否重新接受。
Sweep 参考水平已锁定后，才能判断 reclaim 是否有效。
```

禁止：

```text
后续涨了再补第一控盘区。
后续跌了再说这个不是第一控盘区。
后续失败了再换 AVWAP 锚点。
```

## 4. 状态定义

### 4.1 STATE_IDLE

含义：

```text
尚未识别出可研究的第一控盘区候选。
```

进入条件：

```text
默认初始状态。
```

转移到 `STATE_FCZ_CANDIDATE`：

```text
出现早期横盘/成本中枢候选，并满足最小K线数或成交量聚集条件。
```

输出：

```text
final_classification = no_trade 或 observe_only
allowed_mode = observe
```

---

### 4.2 STATE_FCZ_CANDIDATE

含义：

```text
出现疑似第一控盘区，但尚未锁定。
```

进入条件：

```text
1. 区间高低点初步形成。
2. 价格在区间内停留一定 bars。
3. POC / HVN 初步可计算。
4. 尚未发生明确第一波拉升。
```

必要字段：

```text
first_control_zone_detected
fcz_high
fcz_low
fcz_bars
fcz_volume_share
poc_level
vah_level
val_level
```

转移到 `STATE_FCZ_LOCKED`：

```text
区间边界、POC、VAH/VAL 已可事前固定，且后续不允许随结果移动。
```

失败/降级：

```text
区间过短、成交量无聚集、边界不可复现 -> 回到 STATE_IDLE。
```

---

### 4.3 STATE_FCZ_LOCKED

含义：

```text
第一控盘区已锁定，后续所有验证必须基于该区间。
```

进入条件：

```text
1. fcz_high / fcz_low 已固定。
2. poc_level 已固定。
3. AVWAP_ZoneStart 锚点已固定。
4. 记录 rule_version。
```

转移到 `STATE_FIRST_IMPULSE`：

```text
价格从 FCZ 向上离开，形成第一波拉升。
```

转移到 `STATE_NO_TRADE`：

```text
FCZ 锁定后长期无推进，或流动性/叙事衰退明显。
```

审计要求：

```text
was_boundary_moved_after_outcome 必须默认为 false。
如果后续移动边界，样本进入 E 类定义争议。
```

---

### 4.4 STATE_FIRST_IMPULSE

含义：

```text
第一控盘区后出现第一波拉升，证明区间可能存在主动资金行为。
```

进入条件：

```text
first_impulse_done = true
first_impulse_high 已记录
first_impulse_return 达到最小阈值
```

转移到 `STATE_DEEP_WASHOUT_OR_SWEEP`：

```text
第一波后出现明显回撤、深洗或扫流动性。
```

失败：

```text
第一波后直接缓慢回落且无法回到成本锚 -> STATE_DEATH_MARKET_RISK。
```

---

### 4.5 STATE_DEEP_WASHOUT_OR_SWEEP

含义：

```text
第一波后发生深回撤、假跌破、扫流动性或 Spring 类结构。
```

进入条件：

```text
retracement_ratio 达到深洗区间
或 sweep_detected = true
或价格跌破 FCZ_low / VAL / prior_low 后尝试收回
```

必要字段：

```text
retracement_ratio
deep_washout_detected
sweep_detected
sweep_reference_level
sweep_reference_type
sweep_grade
sweep_reclaim_bars
```

转移到 `STATE_RECLAIM_TEST`：

```text
N 根 K 内收回关键水平，进入 reclaim 测试。
```

失败：

```text
S4_FAILED_SWEEP 或无法收回 -> STATE_DEATH_MARKET_RISK 或 STATE_NO_TRADE。
```

---

### 4.6 STATE_RECLAIM_TEST

含义：

```text
价格正在测试是否重新进入价值区域、站回 POC 或 AVWAP。
```

进入条件：

```text
sweep_reclaim_close_back_inside = true
或 value_area_reentered = true
或 poc_reclaimed = true
或 avwap_reclaimed = true
```

转移到 `STATE_COST_ANCHOR_ACCEPTED`：

```text
POC/AVWAP 不只是触碰，而是形成 RECLAIMED_MEDIUM 或 ACCEPTED_STRONG。
```

失败：

```text
poc_rejected = true
avwap_rejected = true
FAILED_RECLAIM
```

失败后进入：

```text
STATE_FAILED_RECLAIM
```

---

### 4.7 STATE_COST_ANCHOR_ACCEPTED

含义：

```text
成本锚重新接受成立，结构从“可能回收”升级为“候选观察”。
```

进入条件：

```text
cost_anchor_acceptance_score >= 2
poc_reclaimed = true 或 avwap_reclaimed = true
最好 avwap_confluence_count >= 1
```

输出：

```text
final_classification = candidate_for_deeper_review
allowed_mode = research_only
```

转移到 `STATE_PULLBACK_TEST`：

```text
第一次拉起后回撤，测试 POC/AVWAP 是否守住。
```

失败：

```text
重新跌破成本锚 -> STATE_FAILED_RECLAIM。
```

---

### 4.8 STATE_PULLBACK_TEST

含义：

```text
回收后第一次拉起完成，当前测试回撤是否不破关键成本锚，类似 Wyckoff LPS。
```

进入条件：

```text
first_bounce_detected = true
价格回撤到 POC/AVWAP/FCZ 上沿附近
```

转移到 `STATE_SECOND_START_CANDIDATE`：

```text
pullback_holds_anchor = true
higher_low_after_reclaim = true
failed_reclaim_score 不高
```

失败：

```text
回撤跌破 POC/AVWAP 与 FCZ 下沿 -> STATE_FAILED_RECLAIM 或 STATE_DEATH_MARKET_RISK。
```

---

### 4.9 STATE_SECOND_START_CANDIDATE

含义：

```text
二次启动候选，不等于交易确认，只表示值得重点观察。
```

进入条件：

```text
pullback_holds_anchor = true
second_start_score >= 2
bull_trap_score 不高
death_market_score 不高
```

转移到 `STATE_CONFIRMED_OBSERVATION`：

```text
second_push_confirmed = true
或 bos_after_reclaim = true
或结构推进质量达标
```

失败：

```text
二次推进失败、放量滞涨、快速跌回成本锚下方 -> STATE_FAILED_RECLAIM 或 STATE_DEATH_MARKET_RISK。
```

输出：

```text
final_classification = second_start_candidate
allowed_mode = research_only / candidate
```

---

### 4.10 STATE_CONFIRMED_OBSERVATION

含义：

```text
结构观察确认，允许进入样本统计和消融实验中的复合策略触发组。
```

进入条件：

```text
second_push_confirmed = true
或 bos_confirmed_by_close = true
且风险评分不过高
```

注意：

```text
这仍不是自动交易信号，只是观察器确认状态。
```

输出：

```text
entry_rule_triggered = true
triggered_group 包含 D/F/G 中的对应组
```

---

### 4.11 STATE_FAILED_RECLAIM

含义：

```text
成本锚回收失败或二次启动失败。
```

进入条件：

```text
poc_rejected
avwap_rejected
sweep_failed_after_reclaim
first_bounce_failed
second_push_confirmed 后快速跌破成本锚
```

样本归类倾向：

```text
B 类：模型符合但失败
D 类：看似符合但死亡盘
```

输出：

```text
failed_reclaim_score = 2 或 3
final_classification = no_trade 或 observe_only
```

---

### 4.12 STATE_DEATH_MARKET_RISK

含义：

```text
结构可能已从健康深洗转为死亡盘/派发/流动性衰退。
```

进入条件：

```text
death_market_score >= 2
或 distribution_score >= 2
或 narrative_decay_score >= 2
或多次回收失败且高点下降
```

输出：

```text
allowed_mode = no_trade / observe
primary_risk = death_market / distribution / narrative_decay
```

样本归类倾向：

```text
D 类
```

---

### 4.13 STATE_NO_TRADE

含义：

```text
禁止把当前结构作为候选策略样本，只能观察或归档。
```

进入条件：

```text
高风险评分
定义争议严重
执行不可行
样本外不满足
```

输出：

```text
final_classification = no_trade
allowed_mode = no_trade
```

## 5. 状态转移总图

```text
STATE_IDLE
  -> STATE_FCZ_CANDIDATE
  -> STATE_FCZ_LOCKED
  -> STATE_FIRST_IMPULSE
  -> STATE_DEEP_WASHOUT_OR_SWEEP
  -> STATE_RECLAIM_TEST
  -> STATE_COST_ANCHOR_ACCEPTED
  -> STATE_PULLBACK_TEST
  -> STATE_SECOND_START_CANDIDATE
  -> STATE_CONFIRMED_OBSERVATION
```

失败/风险分支：

```text
STATE_RECLAIM_TEST -> STATE_FAILED_RECLAIM
STATE_COST_ANCHOR_ACCEPTED -> STATE_FAILED_RECLAIM
STATE_PULLBACK_TEST -> STATE_FAILED_RECLAIM
STATE_SECOND_START_CANDIDATE -> STATE_FAILED_RECLAIM
STATE_FAILED_RECLAIM -> STATE_DEATH_MARKET_RISK
任意状态 -> STATE_NO_TRADE
```

## 6. 与 Wyckoff 的映射

```text
FCZ_LOCKED                  -> Trading Range
DEEP_WASHOUT_OR_SWEEP       -> Spring
COST_ANCHOR_ACCEPTED        -> Reclaim / Demand returns
FIRST_BOUNCE                -> SOS 初步表现
PULLBACK_TEST               -> LPS
SECOND_START_CANDIDATE      -> Markup 前候选
CONFIRMED_OBSERVATION       -> Markup 初步确认
DEATH_MARKET_RISK           -> Distribution / Failed Accumulation
```

---

## 7. 与样本分类的映射

```text
A 类：进入 CONFIRMED_OBSERVATION 后成功扩展
B 类：进入 SECOND_START_CANDIDATE 或 CONFIRMED_OBSERVATION 后失败
C 类：未进入完整状态链但后续大涨
D 类：进入部分状态链但最终 DEATH_MARKET_RISK
E 类：FCZ/AVWAP/Sweep 等关键状态标注争议
F 类：状态链成立但执行不可行
```

## 8. 与 MQL5 观察指标的关系

MQL5 MVP 只需先输出：

```text
current_state
final_classification
allowed_mode
primary_risk
primary_confirmation
next_expected_event
```

第一版不要求全自动识别所有状态。

推荐 MVP 人机协作方式：

```text
手动框选 FCZ
自动计算 POC/VAH/VAL
手动或半自动标记 first_impulse_high / sweep_low
自动计算 AVWAP 和 reclaim 状态
半自动输出 current_state
CSV 导出样本字段
```

## 9. 状态机审计纪律

```text
1. 状态必须按顺序推进，不允许跳过关键状态后事后补齐。
2. FCZ 锁定后不得随结果移动。
3. AVWAP 锚点必须在触发前确定。
4. 失败样本必须保留当时状态，不得改写历史状态。
5. 新增状态必须说明解决哪个失败问题，并进入质疑审计模板。
```

---

## 10. 本版结论

当前策略最适合先被实现为：

```text
状态观察器 + 样本标注器 + 反证审计框架
```

而不是：

```text
全自动交易 EA
```

状态机的价值在于：

```text
让每个样本在同一套事件顺序下被审计，减少事后解释和主观漂移。
```

```text
用 FCZ_B_0001、FCZ_C_0001、FCZ_D_0001 三个真实样本测试状态机是否可标注。
```
