# 策略反证样本库设计 v0.1

> 版本：v0.1  
> 分线：规则与回测线 / 样本标注线 / 反证审计线  
> 目的：为“第一控盘区成本中枢回收模型”建立反证样本库，不再只收集成功案例，而是系统记录失败、漏报、误报、死亡盘、边界争议与不可执行样本。  
> 状态：设计初版 / 待样本填充 / 待字段表落地。  

---

## 1. 为什么要建立反证样本库

当前策略不能继续只靠成功样本深化。

如果只记录：

```text
第一控盘区 -> 深洗 -> 回收 -> 二次启动 -> 大涨
```

会产生三个严重问题：

```text
1. 幸存者偏差：只看到成功结构。
2. 事后解释：失败案例被重新解释成“其实不符合”。
3. 复杂化陷阱：每失败一次就加一个新规则，导致无法验证。
```

反证样本库要解决的问题是：

```text
什么样的样本能证明当前策略错了？
什么样的失败说明需要修改规则？
什么样的失败说明应该停止深化？
什么样的大涨说明模型过度保守？
```

---

## 2. 反证原则

### 2.1 先记录事实，再解释原因

每个样本必须先记录：

```text
K线事实
成交量事实
POC/VAH/VAL 事实
AVWAP 锚点事实
Sweep/reclaim 事实
后续收益事实
最大回撤事实
```

然后才允许写解释。

禁止：

```text
先根据策略想证明什么，再选择性记录事实。
```

---

### 2.2 失败样本不得事后剔除

如果样本在触发时符合规则，后续失败，必须进入失败库。

禁止写成：

```text
后来失败了，所以它其实不算真正的第一控盘区。
```

除非能证明：

```text
在触发之前，它就违反了事前定义。
```

---

### 2.3 反证优先级高于深化

如果某个新增规则只是为了排除一个失败样本，则必须先进入审计：

```text
这个新增规则是否也会排除成功样本？
是否能在事前识别？
是否能被非发明者一致执行？
是否提升样本外结果？
```

未通过审计前，不允许升级为核心规则。

## 3. 样本分类体系

### 3.1 A 类：模型符合且成功

定义：

```text
模型核心条件基本满足，后续出现明确二次启动或高 R 扩展。
```

用途：

```text
确认理想结构和最大收益空间。
```

最低数量：

```text
20 个
```

---

### 3.2 B 类：模型符合但失败

定义：

```text
触发时符合模型核心条件，但后续跌破、诱多、横盘耗死或收益不足。
```

这是最重要的反证类别。

用途：

```text
验证模型是否有真实过滤能力。
寻找二次启动、AVWAP 回收、POC 接受等条件的失效模式。
```

最低数量：

```text
20 个
```

必须记录失败类型：

```text
B1：回收后快速跌破成本锚
B2：二次启动后失败
B3：POC/AVWAP 站回但无量推进
B4：进入死亡盘缓跌
B5：诱多拉升后砸回区间
B6：执行成本后正 R 变负
```

---

### 3.3 C 类：模型不完全符合但大涨

定义：

```text
未完整满足当前复合策略，但后续出现大幅上涨。
```

用途：

```text
评估模型是否过度保守，是否错过高赔率机会。
```

最低数量：

```text
20 个
```

细分：

```text
C1：没有明显 sweep，但直接二次启动
C2：没有 POC 回收，但 AVWAP 有效
C3：没有 AVWAP 回收，但 POC 有效
C4：没有回撤不破，直接强拉
C5：第一控盘区不明显，但后续大涨
C6：符合更简单策略但不符合复合策略
```

---

### 3.4 D 类：看似符合但死亡盘

定义：

```text
早期看起来像第一控盘区、深洗或回收，但最终进入死亡盘、尾仓派发或持续流动性衰减。
```

用途：

```text
训练死亡盘过滤器，检验当前模型是否会误把派发当吸筹。
```

最低数量：

```text
20 个
```

细分：

```text
D1：高位派发伪装成第一控盘区
D2：深洗后无真实资金回流
D3：反复回收 POC/AVWAP 但高点下降
D4：二次启动失败后流动性枯竭
D5：叙事衰退导致结构失效
D6：盘口/池子深度恶化导致无法执行
```

---

### 3.5 E 类：定义争议样本

定义：

```text
不同人/Agent 对第一控盘区、锚点、sweep、二次启动标注不一致。
```

用途：

```text
测试规则是否可被非发明者一致执行。
```

细分：

```text
E1：第一控盘区边界争议
E2：POC 区间选择争议
E3：AVWAP 锚点选择争议
E4：sweep 是否有效争议
E5：二次启动是否确认争议
E6：周期选择争议
```

---

### 3.6 F 类：不可执行样本

定义：

```text
图形上看符合，但实盘/链上/MT5 执行不可行或严重滑点。
```

用途：

```text
防止理论收益无法商业落地。
```

细分：

```text
F1：K线入场价实际不可成交
F2：滑点吞噬 1R 以上
F3：池子深度不足
F4：点差/手续费过高
F5：链上交易失败或 gas 异常
F6：实盘延迟导致信号失效
```

## 5. 反证判定标准

### 5.1 直接反证

如果出现以下情况，核心规则需降级或修改：

```text
1. B 类样本数量接近或超过 A 类，但无法被事前过滤。
2. C 类样本收益贡献显著高于 A 类，说明模型过度保守。
3. D 类样本大量满足核心条件，说明模型无法区分吸筹和派发。
4. E 类样本标注一致性低，说明规则主观性太强。
5. F 类样本扣除成本后普遍失效，说明商业不可执行。
```

---

### 5.2 强反证

以下情况触发强反证：

```text
1. 复合策略 expectancy_R <= 0。
2. 复合策略不优于至少两个简单基准。
3. 复合策略只在少数极端样本有效。
4. 参数轻微变化导致结果从正转负。
5. 非发明者标注一致率低于 70%。
```

强反证后动作：

```text
停止继续添加规则。
回到最简单有效模块。
把当前策略降级为观察器或风险审查器。
```

## 6. 标注一致性测试

每 20 个样本做一次双人/双 Agent 标注测试。

记录：

```text
fcz_boundary_agreement
poc_range_agreement
avwap_anchor_agreement
sweep_grade_agreement
entry_signal_agreement
final_class_agreement
```

通过标准：

```text
核心字段一致率 >= 80%
争议字段一致率 >= 70%
```

如果不通过：

```text
先修定义，不继续回测。
```

## 9. 本版结论

反证样本库是当前策略从“概念深化”进入“成熟度审计”的必要步骤。

当前最重要的问题不是：

```text
再加什么指标？
```

而是：

```text
哪些样本能证明这个策略是错的？
哪些失败不能再靠补规则解释？
哪些漏报说明它商业上不值得？
哪些执行问题说明它只能做观察器？
```

只有反证样本库建立后，策略才有资格进入正式消融实验。
