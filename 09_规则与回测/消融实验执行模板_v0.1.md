# 消融实验执行模板 v0.1

> 版本：v0.1  
> 分线：规则与回测线 / 反证审计线 / 商业诊断线  
> 目的：用统一模板执行 POC-only、AVWAP-only、Sweep-only、复合策略、Wyckoff严格版、SMC增强版等消融实验，判断当前结构策略是否真的优于更简单策略。  
> 状态：执行模板初版 / 待样本库填充 / 待 CSV 与脚本落地。  

---

## 1. 实验目标

本实验不是为了证明当前策略正确，而是为了判断：

```text
当前复合策略是否值得继续开发？
复杂规则是否带来真实边际收益？
哪个模块贡献 edge？
哪些模块只是增加解释性？
```

必须优先接受反假设：

```text
当前复合策略可能只是多个常见概念的叠加，未必比简单策略更好。
```

---

## 2. 实验前置条件

执行消融实验前，必须具备：

```text
1. 反证样本库至少 80 个样本。
2. A/B/C/D 四类样本各不少于 20 个。
3. 每个样本必须有固定 rule_version。
4. 第一控盘区边界、POC 区间、AVWAP 锚点不得在结果后移动。
5. 所有策略组使用同一入场、止损、止盈、成本计算口径。
```

如果不满足：

```text
不得输出“策略有效/无效”的结论，只能输出“数据不足”。
```

## 3. 实验组定义

### 3.1 A组：POC-only

触发条件：

```text
1. first_control_zone_detected = true
2. poc_level 已确定
3. 价格曾跌破 POC
4. 后续收盘重新站上 POC
```

不要求：

```text
AVWAP
Sweep
回撤不破
二次启动
BOS/FVG/OB
```

目的：

```text
检验 POC 站回是否单独有 edge。
```

---

### 3.2 B组：AVWAP-only

子组：

```text
B1：ZoneStart AVWAP 站回
B2：ImpulseStart AVWAP 站回
B3：Reclaim AVWAP 站回
B4：best_respected_avwap_anchor 站回
B5：至少两条 AVWAP 共振站回
```

最小条件：

```text
1. AVWAP 锚点在触发前确定
2. 价格曾跌破该 AVWAP
3. 后续收盘重新站上该 AVWAP
```

目的：

```text
判断 AVWAP edge 来自固定锚点、动态尊重锚点，还是多锚点共振。
```

---

### 3.3 C组：Sweep-only

子组：

```text
C1：wick-only sweep reclaim
C2：close-break then reclaim within 3 bars
C3：close-break then reclaim within 5 bars
C4：sweep reclaim + bullish close strength
C5：sweep reclaim + next pullback holds reclaimed level
```

目的：

```text
判断假突破/扫流动性本身是否有 edge，以及“快速收回”和“保持”哪个更关键。
```

---

### 3.4 D组：当前复合策略宽松版

条件：

```text
1. first_control_zone_detected = true
2. deep_washout_detected 或 sweep_detected = true
3. poc_reclaimed 或 value_area_reentered = true
4. avwap_reclaimed 或 avwap_confluence_count >= 1
5. pullback_holds_anchor = true
6. second_push_confirmed 或 bos_after_reclaim = true
7. death_market_score 不高
8. bull_trap_score 不高
```

目的：

```text
测试当前主策略的基础复合版本。
```

---

### 3.5 E组：POC + AVWAP 接受版

条件：

```text
1. first_control_zone_detected = true
2. poc_reclaimed 或 value_area_reentered = true
3. avwap_reclaimed 或 avwap_confluence_count >= 1
```

不要求：

```text
Sweep
回撤不破
二次启动
```

目的：

```text
检验成本锚接受本身是否已经足够，不一定需要深洗与二次启动。
```

---

### 3.6 F组：Wyckoff 顺序严格版

条件：

```text
1. range_or_control_zone_locked = true
2. prior_impulse_or_prior_trend_confirmed = true
3. spring_or_sweep_below_support = true
4. spring_or_sweep_reclaimed = true
5. sos_or_first_bounce_confirmed = true
6. lps_or_pullback_holds_anchor = true
7. second_push_or_bos_confirmed = true
```

目的：

```text
检验严格事件顺序是否优于宽松复合条件。
```

---

### 3.7 G组：SMC 确认增强版

条件：

```text
D组 或 F组 条件成立
且满足以下至少一项：
1. bos_confirmed_by_close = true
2. fvg_after_reclaim_detected = true
3. order_block_retest_holds = true
```

目的：

```text
判断 SMC 确认是否真的提升质量，还是只导致入场滞后和漏报增加。
```

## 4. 统一执行规则

### 4.1 入场口径

主口径：

```text
触发条件成立后的下一根 K 线开盘价。
```

辅记录：

```text
trigger_close_price
first_pullback_entry_price
breakout_entry_price
```

但主比较只能使用同一个口径。

---

### 4.2 止损口径

统一优先级：

```text
1. sweep low 下方
2. 第一控盘区下沿下方
3. 回撤确认低点下方
4. 固定 ATR 止损
```

禁止：

```text
给复合策略更宽止损，给简单策略更窄止损。
```

---

### 4.3 止盈口径

至少同时计算：

```text
TP1：固定 1R
TP2：固定 2R
TP3：固定 3R
TP4：结构退出，跌破 POC/AVWAP 或二次启动失败
```

必须记录：

```text
MFE_R
MAE_R
time_to_1R_bars
time_to_2R_bars
```

---

### 4.4 成本口径

每组必须使用同一成本模型：

```text
fee_bps
slippage_bps
spread_bps
gas_or_execution_cost
impact_cost_bps
```

输出两套结果：

```text
gross_R_before_cost
net_R_after_cost
```

商业判断只看：

```text
net_R_after_cost
```

每轮实验必须输出以下比较：


## 7. 判定门槛

### 7.1 继续研究门槛

复合策略至少满足：

```text
1. net_expectancy_R_after_cost > 0
2. trade_count >= 30
3. 优于至少 2 个简单基准
4. D 类样本触发率低于简单基准
5. C 类漏报不会吞噬主要收益
```

否则：

```text
停止扩展新规则，只保留为观察框架。
```

---

### 7.2 产品化观察器门槛

满足：

```text
1. 标注一致率 >= 75%
2. 可以稳定识别候选结构
3. 能降低死亡盘误入率
4. 即使不能直接交易，也能提升人工复盘效率
```

则可推进：

```text
MQL5观察指标 / 样本标注器
```

---

### 7.3 自动交易门槛

必须满足：

```text
1. net_expectancy_R_after_cost > 0.3R
2. profit_factor >= 1.5
3. 样本外仍优于至少 2 个简单基准
4. 参数扰动后仍保持正期望
5. 执行成本敏感性测试通过
6. trade_count 足够，不依赖少数极端样本
```

未满足前：

```text
不得推进全自动 EA。
```

## 11. 本版结论

消融实验是判断当前策略是否真正成熟的核心步骤。

如果复合策略不能优于简单基准：

```text
不要继续给策略加概念。
```

如果只有 F/G 严格版有效，但交易极少：

```text
它可能适合作为高质量提醒器，而不是主交易系统。
```

如果 D/E 宽松版已经足够有效：

```text
不需要继续堆 Wyckoff/SMC 复杂确认。
```

如果 B4 或 C5 单独表现优秀：

```text
策略核心可能不是“第一控盘区复合模型”，而是 AVWAP 动态锚点或 sweep 后保持。
```

最终目标不是证明当前构思正确，而是找到最简单、最稳健、最可商业执行的版本。
