# mql5-fcz-reclaim-kb

## 1. 这个 skill 解决什么

本 skill 是本项目的 Agent 工作流真源，用于指导 Agent 在以下任务中保持一致：

```text
1. 吸收 MQL5 / MT5 文章、论坛、指标、EA 源码资料。
2. 更新 meme 第一控盘区深洗回收模型知识库。
3. 维护第一控盘区、深洗、成本锚回收、诱多过滤、二次启动等概念。
4. 推进 MQL5 观察指标需求、字段映射、状态机和 MVP 开发计划。
5. 建立后续样本标注、规则整合、回测设计。
```

本 skill 不是：

```text
自动交易策略
收益预测工具
MQL5 编译器
链上数据抓取器
```

## 3. 标准工作流

### Phase 1：确认任务归属

先判断用户任务属于哪一类：

```text
A. 新资料吸收
B. 旧资料归位
C. 概念定义更新
D. 风险评分更新
E. 指标需求更新
F. 样本标注体系
G. 规则与回测设计
H. Agent 工作台维护
```

不要把所有任务都写进同一个文件。

---

### Phase 2：读取真源与现状

必须先查看：

```text
1. SOURCE_OF_TRUTH.md 指向的对应真源。
2. 知识库索引中的当前主线文件。
3. 变更记录中的最近阶段。
4. 对应目录 README.md。
```

如果任务涉及新增文件，先确认当前是否已有类似文件，避免重复造版本。

## 4. MQL5 资料吸收规则

吸收 MQL5 资料时必须区分：

```text
1. 平台实现细节：只作为工程参考。
2. 指标计算逻辑：可转化为观察指标模块。
3. EA 下单逻辑：除非明确需要，否则不要直接迁移为自动交易。
4. 风险管理类代码：可转化为风险约束与资金管理概念。
5. 论坛观点：只能作为资料观点，不能直接成为策略结论。
```

优先吸收方向：

```text
Market Profile / Volume Profile / POC / VAH / VAL
VWAP / Anchored VWAP / 成本锚
Wyckoff / Accumulation / Spring / Test / LPS
Liquidity Sweep / Liquidity Grab / Reclaim
False Breakout / Fakeout / Bull Trap
Order Block / Inducement / BOS / CHoCH
Risk Management / Position Sizing / Drawdown Control
```

## 6. 样本标注推进规则

样本标注必须服务：

```text
1. 第一控盘区是否成立
2. 深洗 vs 死亡盘
3. POC / AVWAP 是否重新接受
4. 第一次拉起是诱多还是二次启动候选
5. 结构风险评分是否合理
```

样本标注不应一开始依赖完全自动抓取。可以先使用人工或半自动样本。

推荐下一步文件：

```text

```
