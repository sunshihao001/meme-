## 2. 模拟样本总览

### 2.1 SIM_FCZ_B_0001：模型符合但失败

模拟路径：

```text
FCZ_LOCKED
-> FIRST_IMPULSE
-> DEEP_WASHOUT_OR_SWEEP
-> RECLAIM_TEST
-> COST_ANCHOR_ACCEPTED
-> PULLBACK_TEST
-> SECOND_START_CANDIDATE
-> FAILED_RECLAIM
```

模拟含义：

```text
该样本几乎满足完整复合策略，也进入二次启动候选，但后续快速跌破 POC/AVWAP 成本锚。
```

审计问题：

```text
1. 如果模型符合仍失败，是否说明 bull_trap_score 需要更早介入？
2. second_push_confirmed 是否可能太宽松？
3. 是否需要把“二次启动后不能快速跌破成本锚”写入失败条件？
```

字段表现：

```text
entry_rule_triggered = true
triggered_group = D/F
net_realized_R_after_cost = -1.1
win_loss = loss
failure_reason = 二次启动后快速跌破 POC/AVWAP 成本锚
```

### 2.2 SIM_FCZ_C_0001：模型不完全符合但大涨

模拟路径：

```text
FCZ_LOCKED
-> FIRST_IMPULSE
-> NO_PULLBACK_DIRECT_MARKUP
```

模拟含义：

```text
该样本没有明显深洗，没有完整回撤不破链条，也没有触发完整复合策略，但后续直接大涨。
```

审计问题：

```text
1. 当前模型是否过度等待深洗和二次确认？
2. 是否需要保留“强趋势无深洗路径”作为观察字段？
3. 如果 C 类真实样本很多，完整复合策略可能错过主要收益。
```

字段表现：

```text
entry_rule_triggered = false
triggered_group = B4/simple_momentum_only
max_favorable_excursion_R = 5.4
win_loss = missed_big_runner
final_classification = missed_by_full_model
```


## 3. 本次演练发现的字段有效性

### 3.0 新增 A/E/F 模拟样本补充

在 B/C/D 三类反证场景后，本轮补充了 A/E/F 三类模拟样本：

```text
SIM_FCZ_A_0001：模型符合且成功
SIM_FCZ_E_0001：定义争议样本
SIM_FCZ_F_0001：图形成立但不可执行样本
```

它们同样只写入：

```text

```

并保持：

```text
sample_source = synthetic
is_real_sample = false
```

---

### 3.1 字段足够表达的部分

当前字段可以表达：

```text
1. 样本来源 synthetic / real 的区分。
2. B/C/D 三类反证样本。
3. 状态机路径 current_state_path。
4. 触发组别 triggered_group。
5. MFE/MAE/net_R。
6. 失败原因 failure_reason。
7. 是否事前触发 would_rule_have_triggered_ex_ante。
8. 商业可执行性 commercial_actionability_score。
```

### 3.2 发现的字段缺口

建议后续真实样本字段表考虑加入：

```text
current_state_path
sample_source
is_real_sample
state_invalidated_reason
entry_lag_bars
post_signal_break_anchor_bars
```

其中最重要的是：

```text
current_state_path
```

原因：

```text
只记录 final_classification 不够，需要知道样本经过哪些状态后失败或漏报。
```

## 4. 状态机演练结论

### 4.1 状态机能表达 B 类失败

B 类模拟显示：

```text
即使完整状态链走到 SECOND_START_CANDIDATE，也可能失败。
```

因此状态机必须保留：

```text
SECOND_START_CANDIDATE -> FAILED_RECLAIM
```

不能把 second_start_candidate 当成确认交易信号。

---

### 4.2 状态机需要表达 C 类旁路上涨

C 类模拟显示：

```text
有些大涨不会经过完整深洗/回撤不破路径。
```

建议状态机后续保留一个非核心观察路径：

```text
NO_PULLBACK_DIRECT_MARKUP
```

但它只能作为观察字段，不能直接并入当前核心策略。

---

### 4.3 D 类对死亡盘识别很关键

D 类模拟显示：

```text
POC 回收不够，若 AVWAP 未接受、高点下降、多次回收失败，应快速进入 DEATH_MARKET_RISK。
```

这提示：

```text
POC-only 很可能误判死亡盘。
```

后续真实样本要重点验证。

## 5. A/E/F 补充演练结论

### 5.1 SIM_FCZ_A_0001：模型符合且成功

模拟路径：

```text
FCZ_LOCKED
-> FIRST_IMPULSE
-> DEEP_WASHOUT_OR_SWEEP
-> RECLAIM_TEST
-> COST_ANCHOR_ACCEPTED
-> PULLBACK_TEST
-> SECOND_START_CANDIDATE
-> CONFIRMED_OBSERVATION
```

模拟含义：

```text
该样本完整走完状态链，并在二次启动后出现有效扩展。
```

审计价值：

```text
1. 给 B 类失败样本提供理想对照。
2. 检查成功样本是否也需要很长 entry_lag_bars。
3. 检查完整状态链是否会错过最佳赔率。
```

关键提醒：

```text
A 类样本不能用于单独证明策略有效，只能作为对照。
```

---

### 5.2 SIM_FCZ_E_0001：定义争议样本

模拟路径：

```text
FCZ_CANDIDATE
-> FCZ_LOCKED? disputed
-> RECLAIM_TEST? disputed
```

模拟含义：

```text
该样本的第一控盘区边界、AVWAP 锚点和 sweep 是否有效都存在争议，不同标注者可能得出不同状态链。
```

审计价值：

```text
1. 测试“非发明者一致标注”能力。
2. 暴露 FCZ 边界与 AVWAP 锚点的主观性。
3. 提醒 E 类样本未解决前，不应急于做自动识别。
```

关键提醒：

```text
如果 E 类真实样本比例高，策略应优先修定义，而不是继续加规则。
```

---

### 5.3 SIM_FCZ_F_0001：不可执行样本

模拟路径：

```text
FCZ_LOCKED
-> FIRST_IMPULSE
-> DEEP_WASHOUT_OR_SWEEP
-> RECLAIM_TEST
-> COST_ANCHOR_ACCEPTED
-> SECOND_START_CANDIDATE
-> CONFIRMED_OBSERVATION
-> execution_not_actionable
```

模拟含义：

```text
图形结构成立，但流动性深度不足、滑点/冲击成本过大，导致 gross_R 为正但 net_R_after_cost 为负。
```

审计价值：

```text
1. 防止把图形有效误判为商业可执行。
2. 强制记录 estimated_slippage、impact_cost、liquidity_depth。
3. 提醒 meme 市场中 F 类可能非常关键。
```

关键提醒：

```text
如果 F 类真实样本比例高，策略更适合做观察器，而不是交易系统。
```

## 7. 对真实样本填充的建议

真实样本填充时，优先补字段：

```text
current_state_path
state_invalidated_reason
post_signal_break_anchor_bars
entry_lag_bars
sample_source
is_real_sample
```

真实样本优先级：

```text
1. FCZ_B_0001：验证 second_start_candidate 是否经常失败。
2. FCZ_C_0001：验证完整模型是否漏掉强趋势大涨。
3. FCZ_D_0001：验证 death_market_score 是否能事前识别死亡盘。
```

## 8. 本版结论

模拟演练证明：

```text
当前字段和状态机基本能承载 B/C/D 三类审计场景。
```

但也暴露出：

```text
1. 需要把 current_state_path 正式纳入真实样本字段。
2. 需要明确区分 simulated 与 real 样本。
3. second_start_candidate 必须保持为观察状态，不是交易确认。
4. C 类旁路上涨需要记录，但不应立即扩展核心策略。
5. D 类死亡盘需要用真实样本重点验证 POC-only 的误判风险。
```
