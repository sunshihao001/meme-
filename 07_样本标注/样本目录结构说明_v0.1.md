# 样本目录结构说明 v0.1

> 版本：v0.1  
> 分线：样本标注线 / 文件组织线  
> 目的：规范反证样本库的截图、原始数据、单样本笔记与 CSV 文件归位方式。  

---

## 1. 目录结构

建议结构：

```text
07_样本标注/
├─ 样本字段表_v0.1.md
├─ 反证样本库_v0.1.csv
├─ 第一批20个反证对照样本采集计划_v0.1.md
├─ 单样本记录模板_v0.1.md
├─ 样本目录结构说明_v0.1.md
├─ 样本记录/
│  ├─ FCZ_B_0001.md
│  ├─ FCZ_C_0001.md
│  └─ ...
├─ screenshots/
│  ├─ FCZ_B_0001.png
│  ├─ FCZ_C_0001.png
│  └─ ...
└─ raw_data/
   ├─ FCZ_B_0001.csv
   ├─ FCZ_C_0001.csv
   └─ ...
```

---

## 2. 文件命名规则

### 2.1 样本编号

```text
FCZ_B_0001：模型符合但失败
FCZ_C_0001：模型不完全符合但大涨
FCZ_D_0001：看似符合但死亡盘
FCZ_A_0001：模型符合且成功
FCZ_E_0001：定义争议
FCZ_F_0001：不可执行
```

### 2.2 配套文件

同一个 sample_id 应对应：

```text
样本记录/FCZ_B_0001.md
screenshots/FCZ_B_0001.png
raw_data/FCZ_B_0001.csv
```

如果有多张截图：

```text
screenshots/FCZ_B_0001_before.png
screenshots/FCZ_B_0001_trigger.png
screenshots/FCZ_B_0001_after.png
```

## 3. 填样本流程

```text
1. 从 反证样本库_v0.1.csv 选择一个 pending 槽位。
2. 保存图表截图到 screenshots/。
3. 保存原始K线或导出数据到 raw_data/。
4. 复制 单样本记录模板_v0.1.md 到 样本记录/sample_id.md。
5. 按事前规则填写结构事实。
6. 填写结果、失败原因、审计结论。
7. 回填 CSV 对应行。
8. 若样本引出新规则，不直接修改核心策略，先进入审计字段。
```

---

## 4. 审计要求

每个样本必须明确：

```text
1. 触发时是否按事前规则成立。
2. 是否事后移动了第一控盘区边界。
3. 是否事后换了 AVWAP 锚点。
4. 是否需要新增规则解释失败。
5. 新增规则是否可能误伤成功样本。
```

---

## 5. 不允许行为

```text
1. 只填成功样本。
2. 失败后删除样本。
3. 失败后重新定义第一控盘区。
4. 失败后换锚点但不记录。
5. 未保存截图或原始数据就下结论。
6. 把观察字段直接升级为核心规则。
```
