## 2. 三个样本的寻找标准

### 2.1 FCZ_B_0001：模型符合但失败

优先寻找：

```text
1. 有明显第一控盘区。
2. 有第一波拉升。
3. 有深洗/Sweep 或明显回撤。
4. 价格重新进入 Value Area 或站回 POC/AVWAP。
5. 出现回撤不破或二次启动候选。
6. 但后续失败：跌破成本锚、诱多砸回、死亡盘缓跌或收益不足。
```

这个样本要回答：

```text
模型符合为什么还失败？
失败能否事前识别？
需要新增规则吗？
新增规则会不会误杀成功样本？
```

---

### 2.2 FCZ_C_0001：模型不完全符合但大涨

优先寻找：

```text
1. 不满足完整复合策略。
2. 例如没有明显 sweep、没有 POC 回收、没有 AVWAP 回收、没有回撤不破。
3. 但后续大涨或出现高 R 扩展。
4. 更简单的 POC-only、AVWAP-only、Sweep-only 或 momentum 规则可能更早捕捉。
```

这个样本要回答：

```text
当前模型是否过度保守？
错过这个大涨是否能接受？
更简单规则是否更有效？
```

---

### 2.3 FCZ_D_0001：看似符合但死亡盘

优先寻找：

```text
1. 早期看起来像第一控盘区。
2. 后续似乎有深洗或 POC/AVWAP 回收。
3. 但实际是高位派发、流动性衰退、反复回收失败或高点下降。
4. 最终进入死亡盘或长期无有效推进。
```

这个样本要回答：

```text
death_market_score 是否能事前识别？
当前模型是否误把派发当吸筹？
是否需要 no_trade 一票否决？
```

## 3. 推荐数据来源

### 3.1 可用来源

```text
1. TradingView 图表截图
2. Dexscreener / GeckoTerminal meme 图表
3. 交易所 K线导出 CSV
4. MT5 历史数据
5. 已有项目截图或历史复盘图
```

### 3.2 每个样本至少需要

```text
1. 一张触发前截图
2. 一张触发时截图
3. 一张触发后结果截图
4. 如果可能，导出 OHLCV CSV
```

命名：

```text
screenshots/FCZ_B_0001_before.png
screenshots/FCZ_B_0001_trigger.png
screenshots/FCZ_B_0001_after.png
raw_data/FCZ_B_0001.csv
```

## 4. 填写顺序

每个样本按这个顺序填：

```text
Step 1：把截图/数据放入 screenshots/ 和 raw_data/。
Step 2：打开 样本记录/FCZ_x_0001.md。
Step 3：填写基础信息和图表链接。
Step 4：按状态机标注 current_state 经过哪些阶段。
Step 5：填写 FCZ / POC / AVWAP / Sweep / 二次启动字段。
Step 6：填写 MFE / MAE / net_R。
Step 7：填写审计问题。
Step 8：同步更新 反证样本库_v0.1.csv。
```

## 5. 状态机标注要求

每个样本都要明确是否经过：

```text
STATE_FCZ_CANDIDATE
STATE_FCZ_LOCKED
STATE_FIRST_IMPULSE
STATE_DEEP_WASHOUT_OR_SWEEP
STATE_RECLAIM_TEST
STATE_COST_ANCHOR_ACCEPTED
STATE_PULLBACK_TEST
STATE_SECOND_START_CANDIDATE
STATE_CONFIRMED_OBSERVATION
STATE_FAILED_RECLAIM
STATE_DEATH_MARKET_RISK
STATE_NO_TRADE
```

如果某一状态无法判断：

```text
不要强行判断；标记为 unknown，并写入 notes。
```
