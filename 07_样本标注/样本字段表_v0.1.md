# 样本字段表 v0.1

> 版本：v0.1  
> 分线：样本标注线 / 规则与回测线 / 指标导出线  
> 目的：统一“第一控盘区成本中枢回收模型”的反证样本库、消融实验、MQL5观察指标 CSV/JSON 导出字段，确保后续样本可审计、可回测、可商业诊断。  
> 状态：字段表初版 / 对齐反证样本库与消融实验模板。  

---

## 1. 字段表原则

### 1.1 先事实，后评分，再结论

字段顺序必须遵守：

```text
基础事实 -> 结构事实 -> 触发事实 -> 结果事实 -> 成本事实 -> 审计结论
```

禁止：

```text
先给 final_classification，再反向补证据。
```

---

### 1.2 每个样本必须绑定规则版本

必须记录：

```text
rule_version
reviewer
review_date
```

否则后续无法判断：

```text
这个样本是按哪个版本的策略判定的？
规则修改后是否偷换了历史结论？
```

---

### 1.3 失败样本不得事后重标

如果触发当时符合规则，后续失败，必须保留：

```text
would_rule_have_triggered_ex_ante = true
```

不得因为失败而把它改成“不符合”。

## 13. 字段维护规则

新增字段必须回答：

```text
1. 是否解决已知失败问题？
2. 是否能事前标注？
3. 是否能被非发明者一致标注？
4. 是否进入 CSV 最小字段集，还是只进入扩展字段？
5. 是否会增加过拟合风险？
```

未通过审计的字段只能进入：

```text
观察字段
```

不能直接进入核心规则。
