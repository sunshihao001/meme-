> 位置：`07_样本标注/样本标注模板_v0.1.md`  
> 分线：样本标注 / 反证审计 / 规则与回测  
> 状态：v0.1，模板初版  
> 用途：为后续人工样本标注、反证样本库、半自动识别、回测原型提供统一字段与填写规范。

## 1. 本文目的

本文件定义“第一控盘区成本中枢回收模型”的样本标注模板。

目标不是直接回测，而是先确保每个样本都能按统一结构记录：

```text
事实
结构
触发
执行
结果
反证意义
审计结论
```

如果样本标注不统一，后续规则识别、状态机校验、反证实验、消融实验和回测都不可靠。

---

## 2. 标注原则

### 2.1 先事实，后解释

必须先记录：

```text
价格事实
成交量事实
POC / VAH / VAL 事实
AVWAP 锚点事实
触发时间事实
后续收益 / 回撤事实
执行约束事实
```

然后才允许写策略解释。

---

### 2.2 事前可见原则

所有触发判断必须回答：

```text
would_rule_have_triggered_ex_ante
```

即：在当时那根 K 线或当时状态下，规则是否已经可见？

禁止使用未来 K 线反推当时可买。

---

### 2.3 失败不得剔除

如果样本在触发时符合规则，后续失败，必须记录失败，不得事后改写为不符合。

必须记录：

```text
was_boundary_moved_after_outcome
was_anchor_changed_after_outcome
is_failure_explained_by_new_rule
```

---

## 3. 样本分类

样本使用以下主类：

```text
A：模型符合且成功
B：模型符合但失败
C：模型不完全符合但大涨
D：看似符合但死亡盘
E：图形成立但不可执行
F：数据不足 / 争议样本
```

建议子类：

```text
B1：回收后快速跌破成本锚
B2：二次启动后失败
B3：POC / AVWAP 站回但无量推进
B4：进入死亡盘缓跌
B5：诱多拉升后砸回区间
B6：执行成本后正 R 变负

C1：没有明显 sweep，但直接二次启动
C2：没有 POC 回收，但 AVWAP 有效
C3：没有 AVWAP 回收，但 POC 有效
C4：没有回撤不破，直接强拉
C5：第一控盘区不明显，但后续大涨
C6：符合更简单策略但不符合复合策略

D1：高位派发伪装成第一控盘区
D2：深洗后无真实资金回流
D3：反复回收 POC / AVWAP 但高点下降
D4：二次启动失败后流动性枯竭
D5：叙事衰退导致结构失效
D6：盘口 / 池子深度恶化导致无法执行

E1：买点在涨停 / 急拉后无法成交
E2：滑点过大
E3：止损距离过远
E4：需要盘中未来确认
E5：流动性太差
E6：持仓周期过长或资金效率过低
```

## 4. 字段模板

### 4.1 基础信息字段

```text
sample_id：唯一样本编号，例如 FCZ_A_0001
sample_class：A/B/C/D/E/F
sample_subclass：如 B1、C4、D3
symbol：标的代码或合约 / token 名称
chain_or_market：链 / 市场，例如 Solana、ETH、A股、美股、期货
exchange_or_dex：交易所或 DEX
timeframe：样本周期，例如 5m、15m、1h、4h、1d
sample_start_time：样本起始时间
sample_end_time：样本结束时间
reviewer：标注者
review_date：标注日期
rule_version：使用的规则版本
sample_source：real / synthetic / imported / manual
is_real_sample：true / false
```

### 4.2 图表与数据来源字段

```text
source_chart_url：图表链接
raw_data_path：原始数据路径
screenshot_path：截图路径
data_vendor：数据来源
data_granularity：数据粒度
data_quality：high / medium / low / unknown
missing_data_notes：缺失数据说明
```

### 4.3 第一控盘区字段

```text
first_control_zone_detected：true / false / unknown
fcz_high：第一控盘区上沿
fcz_low：第一控盘区下沿
fcz_mid：第一控盘区中位
fcz_start_time：区间开始时间
fcz_end_time：区间结束时间
fcz_bars：区间 K 线数
fcz_width_pct：区间宽度百分比
fcz_volume_share：区间成交量占比
fcz_confidence：high / medium / low / disputed
fcz_rejection_reason：若未识别，说明原因
```

### 4.4 成本中枢字段

```text
cost_center_method：POC / ValueArea / AVWAP / HVN / StructureRange / Mixed / Unknown
poc_level：POC 价格
vah_level：Value Area High
val_level：Value Area Low
hvn_zone：高成交节点区域
cost_center_high：成本中枢上沿
cost_center_low：成本中枢下沿
cost_center_mid：成本中枢中位
cost_center_confidence：high / medium / low / disputed
cost_center_notes：备注
```

### 4.5 AVWAP 锚点字段

```text
avwap_zone_start：从第一控盘区起点锚定的 AVWAP
avwap_impulse_start：从第一波启动点锚定的 AVWAP
avwap_reclaim：从重新站回点锚定的 AVWAP
avwap_pullback_low：从深洗低点锚定的 AVWAP
best_respected_avwap_anchor：最受尊重的锚点
anchor_known_at_trigger：true / false / unknown
was_anchor_changed_after_outcome：true / false / unknown
anchor_change_notes：如发生变更，说明原因
```

### 4.6 第一波与深洗字段

```text
first_impulse_done：true / false / unknown
first_impulse_start_time：第一波开始时间
first_impulse_high：第一波高点
first_impulse_return_pct：第一波涨幅
retracement_ratio：回撤比例
deep_washout_detected：true / false / unknown
washout_low：深洗低点
washout_depth_pct：深洗幅度
washout_duration_bars：深洗持续 K 线数
sweep_detected：true / false / unknown
sweep_grade：none / weak / medium / strong
sweep_reclaim_bars：扫流动性后收回所需 K 线数
```

### 4.7 重新接受字段

```text
value_area_reentered：true / false / unknown
poc_reclaimed：true / false / unknown
avwap_reclaimed：true / false / unknown
reclaim_time：重新站回时间
reclaim_close_above_anchor_bars：成本锚上方收盘 K 线数
acceptance_grade：none / weak / medium / strong
pullback_holds_anchor：true / false / unknown
pullback_low_after_reclaim：回撤低点
failed_reclaim：true / false / unknown
failed_reclaim_reason：失败原因
```

### 4.8 二次启动 / 诱多字段

```text
first_bounce_detected：true / false / unknown
second_push_confirmed：true / false / unknown
second_push_time：二次启动时间
second_push_breaks_prior_high：true / false / unknown
bos_after_reclaim：true / false / unknown
fvg_after_reclaim_detected：true / false / unknown
order_block_retest_holds：true / false / unknown
bull_trap_detected：true / false / unknown
bull_trap_reason：诱多原因
```

### 4.9 触发与交易执行字段

```text
entry_rule_triggered：true / false / unknown
triggered_group：结构触发 / 回收触发 / 二次启动触发 / 风险过滤 / 其他
entry_time：入场时间
entry_price：入场价格
stop_price：止损价格
initial_risk_pct：初始风险百分比
slippage_estimate_pct：预估滑点
liquidity_depth_at_entry：入场时流动性深度
is_actionable_entry：true / false / unknown
non_actionable_reason：不可执行原因
```

### 4.10 结果字段

```text
max_favorable_excursion_R：最大有利波动 R
max_adverse_excursion_R：最大不利波动 R
gross_realized_R：毛收益 R
net_realized_R_after_cost：扣成本后净收益 R
max_drawdown_after_entry_R：入场后最大回撤 R
holding_period_bars：持仓 K 线数
win_loss：win / loss / breakeven / unknown
failure_reason：失败原因
final_classification：最终分类
```

### 4.11 风险评分字段

```text
death_market_score：死亡盘风险评分
bull_trap_score：诱多风险评分
failed_reclaim_score：失败回收评分
distribution_score：派发风险评分
liquidity_trap_score：流动性陷阱评分
narrative_decay_score：叙事衰退评分
commercial_actionability_score：商业可执行性评分
```

### 4.12 反证审计字段

```text
challenged_hypothesis：挑战的假设 H1-H7
would_rule_have_triggered_ex_ante：true / false / unknown
was_boundary_moved_after_outcome：true / false / unknown
was_anchor_changed_after_outcome：true / false / unknown
is_failure_explained_by_new_rule：true / false / unknown
new_rule_needed：true / false / unknown
can_simpler_rule_explain_it：true / false / unknown
audit_conclusion：规则缺陷 / 执行缺陷 / 市场环境缺陷 / 不可避免亏损 / 过度保守 / 标注争议
next_action：保留规则 / 降级规则 / 新增观察字段 / 暂停深化 / 进入消融实验
notes：标注者备注
```

## 7. 标注完成检查清单

每个样本完成前检查：

```text
1. 是否填写 sample_id、sample_class、sample_subclass？
2. 是否记录 rule_version？
3. 是否记录数据来源和截图 / 原始数据路径？
4. 是否记录第一控盘区边界？
5. 是否记录成本中枢表达方式？
6. 是否记录 AVWAP 锚点是否事前可知？
7. 是否记录深洗、回收、二次启动状态？
8. 是否记录 entry、stop、MFE、MAE、net R？
9. 是否判断买点可执行性？
10. 是否回答 would_rule_have_triggered_ex_ante？
11. 是否记录边界 / 锚点是否事后移动？
12. 是否给出反证审计结论？
13. 是否同步 CSV？
```

## 8. 本版结论

本模板用于把样本从“看图讨论”转化为“可审计数据”。

后续进入回测或自动化前，至少应完成：

```text
1. A/B/C/D/E/F 类样本各自的最小样本集。
2. CSV 字段与 Markdown 单样本记录的一致性。
3. 状态机路径校验。
4. 反证实验字段补齐。
5. Owner 对成功 / 失败 / 可执行性的阈值决策。
```
