# 第一批20个反证对照样本采集计划 v0.1

> 版本：v0.1  
> 分线：样本标注线 / 反证审计线 / 消融实验线  
> 目的：启动第一批 20 个反证/对照样本采集，不再优先寻找成功案例，而是优先寻找能挑战当前策略成熟度的 B/C/D 类样本。  
> 状态：采集计划初版 / CSV 已预留样本槽位 / 待人工或 Agent 根据真实图表填充。  

---

## 1. 本批样本目标

第一批不追求完整 80 个样本，而是先做 20 个“策略审计启动样本”。

目标不是证明策略正确，而是尽快暴露：

```text
1. 模型符合但失败的真实频率。
2. 模型不完全符合但大涨的漏报风险。
3. 看似符合但死亡盘的误判风险。
4. 定义是否会出现大量争议。
5. 图形上可行但商业上不可执行的情况。
```

---

## 2. 样本配额

第一批 20 个样本建议配额：

```text
B 类：模型符合但失败        6 个
C 类：模型不完全符合但大涨  6 个
D 类：看似符合但死亡盘      4 个
A 类：模型符合且成功        2 个
E 类：定义争议样本          1 个
F 类：不可执行样本          1 个
```

为什么 B/C/D 优先：

```text
B 类挑战“符合就有效”。
C 类挑战“模型是否过度保守”。
D 类挑战“模型能否区分吸筹和派发”。
```

A 类只保留 2 个，是为了避免第一批又变成成功案例收藏。

## 5. B 类采集标准：模型符合但失败

B 类样本必须满足：

```text
1. 触发时看起来符合当前模型的核心条件。
2. 事前规则会触发观察或候选。
3. 后续没有形成有效收益，或跌破关键成本锚，或诱多失败。
```

优先找：

```text
B1：回收后快速跌破成本锚
B2：二次启动后失败
B3：POC/AVWAP 站回但无量推进
B4：进入死亡盘缓跌
B5：诱多拉升后砸回区间
B6：执行成本后正 R 变负
```

每个 B 类样本必须回答：

```text
失败是否能在事前识别？
如果不能识别，是否说明策略核心假设弱？
如果需要新增规则，该规则是否会误杀成功样本？
```

---

## 6. C 类采集标准：不完全符合但大涨

C 类样本必须满足：

```text
1. 未完整满足当前复合策略。
2. 后续出现显著上涨或高 R 扩展。
3. 至少一个简单基准可能提前捕捉。
```

优先找：

```text
C1：没有明显 sweep，但直接二次启动
C2：没有 POC 回收，但 AVWAP 有效
C3：没有 AVWAP 回收，但 POC 有效
C4：没有回撤不破，直接强拉
C5：第一控盘区不明显，但后续大涨
C6：符合更简单策略但不符合复合策略
```

每个 C 类样本必须回答：

```text
当前模型是否过度保守？
错过该样本是否影响总体收益？
是否存在更简单有效规则？
```

---

## 7. D 类采集标准：看似符合但死亡盘

D 类样本必须满足：

```text
1. 早期看起来像第一控盘区、深洗或回收。
2. 后续进入死亡盘、尾仓派发、流动性衰减或持续下跌。
```

优先找：

```text
D1：高位派发伪装成第一控盘区
D2：深洗后无真实资金回流
D3：反复回收 POC/AVWAP 但高点下降
D4：二次启动失败后流动性枯竭
```

每个 D 类样本必须回答：

```text
死亡盘风险是否能由 death_market_score 事前识别？
是否说明当前模型对派发/吸筹区分不足？
是否应加强 no_trade 一票否决？
```

## 9. 采集流程

每个样本按照以下流程：

```text
Step 1：确定样本类别 A/B/C/D/E/F。
Step 2：截取图表或保存原始数据。
Step 3：按事前规则标注 FCZ、POC、AVWAP、Sweep、二次启动。
Step 4：填写 CSV 最小字段。
Step 5：记录 MFE/MAE 和净 R。
Step 6：填写是否事后移动边界/锚点。
Step 7：写 notes，明确这个样本挑战了哪个假设。
```

## 10. 第一批完成标准

完成 20 个样本后，输出第一版审计摘要：

```text
B 类失败主要原因 Top 3
C 类漏报主要原因 Top 3
D 类死亡盘误判主要原因 Top 3
哪些字段最难一致标注
哪些规则最容易事后解释
是否应继续完整策略研究
是否应降级为观察器
```

如果第一批样本显示：

```text
B 类和 D 类触发率很高，且无法事前过滤
```

则暂停继续加规则，优先修正核心定义。

---

## 11. 本版结论

第一批 20 个样本是策略审计的启动批次。

它不是为了“找证据证明策略好”，而是为了回答：

```text
这个结构策略最容易在哪里错？
哪些错不能再靠补规则解释？
它更适合做交易策略，还是观察/审计工具？
```
