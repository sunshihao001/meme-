# 模拟样本演练规则 v0.1

> 版本：v0.1  
> 分线：样本标注线 / 反证审计线 / 方法论边界  
> 目的：允许在没有真实图表/原始数据前，根据 B/C/D 类特征做 synthetic/mock 演练，用于测试字段、状态机、模板和审计流程；但明确模拟样本不能作为策略有效性证据。  
> 状态：模拟演练边界规则。  

---

## 1. 模拟演练的允许范围

模拟样本可以用于：

```text
1. 测试样本字段是否够用。
2. 测试状态机是否能表达 B/C/D 类情形。
3. 测试 CSV 与 Markdown 记录流程。
4. 训练如何提出审计问题。
5. 提前发现字段缺口和模板问题。
```

模拟样本不能用于：

```text
1. 证明策略有效。
2. 计算真实胜率。
3. 计算真实盈亏比。
4. 宣称某个规则优于基准。
5. 替代真实样本、回测或样本外验证。
```

---

## 2. 命名规则

所有模拟样本必须带 `SIM_` 前缀：

```text
SIM_FCZ_B_0001
SIM_FCZ_C_0001
SIM_FCZ_D_0001
```

不得写入真实样本主 CSV：

```text
07_样本标注/反证样本库_v0.1.csv
```

必须单独写入：

```text
07_样本标注/模拟反证样本库_v0.1.csv
```

---

## 3. 标记规则

模拟样本必须明确写：

```text
sample_source = synthetic
is_real_sample = false
```

notes 中必须包含：

```text
模拟样本，仅用于流程演练，不作为策略有效性证据。
```

---

## 4. 设计要求

每个模拟样本必须服务一个审计目标：

```text
SIM_FCZ_B_0001：演练“模型符合但失败”如何记录。
SIM_FCZ_C_0001：演练“模型不完全符合但大涨”如何记录。
SIM_FCZ_D_0001：演练“看似符合但死亡盘”如何记录。
```

模拟数值可以合理构造，但必须：

```text
1. 内部逻辑一致。
2. 不伪装成真实行情。
3. 不使用真实币种制造误导。
4. 不写任何虚假来源链接。
```

---

## 5. 演练完成后的输出

模拟演练完成后输出：

```text
07_样本标注/模拟样本状态机演练审计报告_v0.1.md
```

报告只回答：

```text
字段是否够用？
状态机是否表达清楚？
哪些字段容易混淆？
真实样本填充前还需要补什么？
```

不得回答：

```text
策略是否有效。
策略胜率是多少。
策略是否成熟。
```
