> 分线：问题清单 / Owner 决策 / 方法轮输入  
> 状态：v0.1，初始问题池  
> 用途：集中记录“第一控盘区成本中枢回收模型”中不能由 Codex 或任何执行 Agent 自行决定的问题，作为后续 Good Question Brief、spec、plan、tasks 与 Owner Decision Brief 的输入。

---

## 1. 本文目的

本文用于记录当前项目中需要 Owner 决策的问题。

原则：

```text
Codex 可以执行任务，但不能替 Owner 定义策略。
Agent 可以整理、审计、提出默认建议，但不能把未确认假设写成已确认结论。
```

所有问题应标记为：

```text
P0：进入下一阶段前必须决策
P1：可以先用暂定默认值推进，但后续必须确认
P2：可以延后，不影响当前阶段文档沉淀
```

决策状态：

```text
UNKNOWN：尚未决策
PROPOSED：已有建议默认值，待确认
DECIDED：Owner 已确认
DEFERRED：Owner 明确延后
REJECTED：Owner 明确否决
```

```text
问题：多个成本锚冲突时如何判断？

影响：后视偏差、样本标注、规则实现
```

### Q-P0-006：重新接受最低确认标准

```text
问题：重新接受最少需要哪些条件？
当前建议：站回成本锚 + 停留 / 换手 + 回撤不破 + 二次推进；其中“强接受”必须满足完整结构。
状态：PROPOSED
影响：核心规则、样本分类、触发规则
Owner 决策：UNKNOWN
```

### Q-P0-007：普通反抽和重新接受的分界

```text
问题：什么时候只能算普通反抽，不能算重新接受？
当前建议：只有触碰或短暂站回、没有停留 / 回撤不破 / 二次推进的，统一归为普通反抽或弱接受，不作为核心触发。
状态：PROPOSED
影响：诱多过滤、失败样本归类
Owner 决策：UNKNOWN
```

### Q-P0-008：二次启动确认标准

```text
问题：二次启动是否必须突破局部前高、出现 BOS、放量或站稳？
当前建议：初版至少要求重新接受后的二次推进，并记录是否突破前高 / BOS / 放量；是否作为硬条件待消融实验决定。
状态：PROPOSED
影响：买点定义、回测触发、诱多过滤
Owner 决策：UNKNOWN
```
